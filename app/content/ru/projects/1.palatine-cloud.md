## О проекте

Palatine Cloud — коммерческая облачная платформа, которую я спроектировал и разработал с нуля. Платформа предоставляет бизнесу API-сервисы для полного цикла обработки речи: от распознавания речи (ASR) и разделения спикеров до анализа тональности, интеллектуальной суммаризации через локальные LLM и синтеза речи (TTS).

Ключевая особенность — все ML-модели, включая большие языковые модели, работают локально на собственных GPU-серверах. Это обеспечивает конфиденциальность данных клиентов, предсказуемые затраты и полный контроль над качеством инференса. Платформа обрабатывает **10 миллионов аудиоминут ежедневно**, поддерживает **100+ языков** с автоматическим определением, принимает **23+ аудио/видео формата** и обеспечивает **uptime 99.982%** на 4 географически распределённых дата-центрах Tier III.

Проект полностью монетизирован: реализована система тарифов, приём платежей, автоматическая фискализация — платформа приносит стабильный доход.

## Задача

Создать с нуля коммерческую облачную платформу, способную обрабатывать большие объёмы речевых данных в реальном времени. Платформа должна масштабироваться под нагрузку, обеспечивать высокую надёжность и качество распознавания, предоставлять OpenAI-совместимый API для лёгкой интеграции, включать локальные LLM для суммаризации и аналитики, а также поддерживать полный бизнес-цикл — от регистрации пользователя до оплаты и фискализации.

## Архитектура

Платформа построена на **микросервисной архитектуре из 10 независимых сервисов**, развёрнутых на нескольких выделенных GPU-серверах. Сервисы взаимодействуют через HTTP и gRPC, объединены единым API Gateway и полным стеком мониторинга.

### Сервис распознавания речи и диаризации
Ядро платформы — двухуровневая архитектура: уровень оркестрации (аутентификация, биллинг, маршрутизация запросов) и уровень ML-инференса. Модели обслуживаются через **NVIDIA Triton Inference Server** с gRPC API, sequence batching и stateful inference, а оркестрация GPU-ресурсов выполняется через **Ray Serve** с автоматическим масштабированием реплик под текущую нагрузку.

Стриминговая транскрипция работает в реальном времени: клиент отправляет аудиопоток через WebSocket, сервис декодирует и ресемплирует сигнал, передаёт на Triton через gRPC и возвращает результат по мере распознавания. Поддерживается 15+ аудиокодеков (PCM, Opus, FLAC, MP3, AAC и др.) с частотами дискретизации от 8 до 48 кГц. Достигнута **точность распознавания 92.9% (WER 7.1%)** при обработке файлов до 10+ часов с латентностью ~25 секунд на 30-минутную запись.

### Локальные LLM — суммаризация и аналитика
Платформа использует **локально развёрнутые большие языковые модели** для интеллектуальной обработки распознанного текста. Суммаризация работает с библиотекой настраиваемых промптов и поддерживает режим thinking для сложных аналитических задач. Реализован OpenAI-совместимый эндпоинт chat completions — клиенты могут использовать стандартный OpenAI SDK для работы с локальными моделями. Все данные остаются на серверах платформы, что критично для корпоративных клиентов.

### API Gateway
Единая точка входа для всех клиентских запросов. Маршрутизация описывается декларативно, загружается динамически при старте. Поддерживает несколько типов аутентификации, проксирование ко всем внутренним сервисам и составные запросы — цепочки вызовов с подстановкой результатов промежуточных шагов. Фронтирован reverse proxy с TLS-терминацией и автоматическим получением сертификатов.

### Платёжный шлюз
Построен по принципам **Clean Architecture** с чётким разделением на слои: сущности, бизнес-логика, интерфейсы и адаптеры. Поддерживает маршрутизацию платежей по нескольким мерчантам с учётом тарифа пользователя, обработку webhook-уведомлений с проверкой подлинности, автоматическое формирование фискальных чеков по 54-ФЗ, промокоды и идемпотентную обработку событий.

### Self-hosted база данных и аутентификация
Развёрнут полный стек Supabase из 12+ контейнеров: аутентификация, real-time подписки, хранилище файлов, серверлесс-функции, REST API и пулинг соединений. Кастомные SQL-миграции с политиками безопасности на уровне строк (RLS).

### Мониторинг и наблюдаемость
Полный стек наблюдаемости с мультинодной архитектурой: дашборды и визуализация метрик, сбор метрик со всех серверов (включая метрики Ray Serve и GPU), централизованное хранилище логов от всех Docker-контейнеров, агенты сбора на удалённых хостах.

### Синтез речи (TTS)
Контейнеризированный сервис нейросетевого синтеза речи с GPU-инференсом, поддержкой клонирования голоса по референсному аудио и OpenAI-совместимым API.

### Личный кабинет, админ-панель и документация
Бэкенд личного кабинета пользователя с rate limiting, административная панель управления тарифами и промокодами, а также **публичная API-документация**, автоматически генерируемая из OpenAPI-спецификаций сервисов и развёрнутая на платформе Mintlify с интерактивным API Playground.

## Стек технологий

**Backend:** Python 3.10–3.11, FastAPI, Uvicorn, Pydantic, httpx, WebSocket

**ML/AI:** NVIDIA Triton Inference Server (gRPC, sequence batching), Ray Serve (автоскейлинг), CUDA 12.6, Fish Speech (TTS), локальные LLM (Ollama), CTC Decoder

**Данные:** PostgreSQL 15, Supabase (self-hosted), Redis, Alembic

**Инфраструктура:** Docker / Docker Compose, GitLab CI/CD (lint → test → deploy), Traefik (TLS, HSTS), Grafana, Prometheus, Loki, Alloy

**Интеграции:** YooKassa (54-ФЗ), Supabase Auth (JWT), OpenAI-совместимый API, Mintlify (автогенерация документации)
