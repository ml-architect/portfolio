## О проекте

Внутренняя информационная система МФТИ нуждалась в быстром и точном поиске по растущей базе статей и комментариев. Существующий процесс работы с контентом через табличные файлы перестал справляться: поиск нужного материала среди сотен документов занимал слишком много времени, а совместная работа над комментариями была неудобной.

Я спроектировал и разработал REST API-сервис, который решил эти проблемы — обеспечил мгновенный полнотекстовый поиск с учётом морфологии русского и английского языков, удобное управление статьями и комментариями через единый программный интерфейс, а также возможность миграции накопленных данных из Excel без потери информации.

## Функционал

Сервис предоставляет 18 эндпоинтов, покрывающих полный жизненный цикл работы с контентом. Для статей реализованы операции создания, редактирования, удаления, а также несколько вариантов поиска — по заголовкам, содержимому и отдельным фрагментам текста. Комментарии привязаны к конкретным статьям и поддерживают аналогичный набор операций с поиском по тексту и дате.

Отдельный блок функционала связан с авторами: система автоматически агрегирует список всех авторов из базы и позволяет просматривать публикации конкретного автора с пагинацией. Для удобства миграции реализован импорт данных из Excel-файлов — можно загрузить таблицу со статьями и комментариями, и сервис автоматически разнесёт данные по обоим хранилищам.

Все эндпоинты возвращают ответы в унифицированном формате и автоматически документируются через Swagger UI, что упрощает интеграцию для фронтенд-команды.

## Архитектура

В основе сервиса лежит гибридная архитектура с двумя хранилищами данных, каждое из которых отвечает за свою задачу. Elasticsearch обеспечивает полнотекстовый поиск с морфологическим анализом — при индексации текстов применяются специализированные анализаторы для русского и английского языков, что позволяет находить статьи по любым словоформам. PostgreSQL хранит структурированные данные и обеспечивает реляционные связи между статьями, комментариями и авторами.

При каждой операции записи данные синхронно сохраняются в оба хранилища, что гарантирует консистентность между поисковым индексом и основной базой. Сервис построен по принципам 12-factor app: конфигурация вынесена в переменные окружения, соединения с базами данных корректно закрываются при остановке приложения, а весь стек из трёх сервисов (API, Elasticsearch, PostgreSQL) разворачивается одной командой через Docker Compose.

API-слой реализован на FastAPI с асинхронной обработкой запросов, валидацией входных данных через Pydantic-модели и HTTP Basic-аутентификацией для защищённых операций.

## Ключевые достижения

- **Время отклика поиска < 50 мс** — полнотекстовый поиск по 500+ статей с морфологическим анализом двух языков работает практически мгновенно благодаря Elasticsearch
- **Миграция данных из Excel без потерь** — массовый импорт сотен статей с комментариями из табличных файлов в структурированное хранилище
- **Деплой всего стека одной командой** — полная контейнеризация трёх сервисов через Docker Compose с автоматической инициализацией БД и поисковых индексов
