## О проекте

Внутренняя система МФТИ работала с контентом через табличные файлы — поиск среди сотен статей был медленным, а совместная работа над комментариями неудобной. Я спроектировал REST API с мгновенным полнотекстовым поиском (морфология RU/EN), единым интерфейсом управления статьями и комментариями, а также импортом накопленных данных из Excel.

## Функционал

18 эндпоинтов покрывают полный жизненный цикл контента: CRUD статей и комментариев, поиск по заголовкам, содержимому и фрагментам текста, фильтрация по дате. Комментарии привязаны к статьям. Отдельный блок — агрегация авторов и их публикаций с пагинацией.

Реализован массовый импорт из Excel: загруженная таблица автоматически разносится по обоим хранилищам. Все ответы в унифицированном формате, автодокументация через Swagger UI.

## Архитектура

Гибридное хранилище: **Elasticsearch** отвечает за полнотекстовый поиск с морфологическими анализаторами (RU/EN), **PostgreSQL** — за реляционные связи и структурированные данные. При записи данные синхронно попадают в оба хранилища, что гарантирует консистентность.

Сервис следует принципам 12-factor app: конфигурация через переменные окружения, graceful shutdown соединений, весь стек (API + Elasticsearch + PostgreSQL) поднимается одной командой через Docker Compose с автоинициализацией БД и индексов.

API-слой — FastAPI с асинхронной обработкой, валидацией через Pydantic v2 и HTTP Basic-аутентификацией.

## Ключевые достижения

- **Поиск < 50 мс** — полнотекстовый поиск по 500+ статей с морфологией двух языков
- **Миграция из Excel без потерь** — массовый импорт сотен статей с комментариями в структурированное хранилище
- **Деплой одной командой** — контейнеризация трёх сервисов с автоинициализацией БД и поисковых индексов
