## О проекте

DGX Spark Benchmark — сервис для комплексного бенчмаркинга производительности LLM-инференса на **NVIDIA DGX Spark** — одном из первых компактных устройств на архитектуре Grace Blackwell с 128 ГБ объединённой памяти.

Результаты этого бенчмарка легли в основу [статьи на Хабр](https://habr.com/ru/articles/994058/) — подробного разбора реальных возможностей DGX Spark, где маркетинговые обещания NVIDIA были проверены объективными измерениями.

## Задача

Разработать полностью воспроизводимый инструмент, который автоматически прогоняет модели через набор реалистичных сценариев, собирает метрики производительности и генерирует готовые интерактивные графики для технической публикации.

## Как устроен сервис

### Многомерное тестирование

Сервис выполняет полный перебор всех комбинаций параметров: каждая модель тестируется на каждом датасете при каждом уровне параллельности (от 1 до 100 одновременных запросов). Это позволяет построить полную картину поведения модели — от отзывчивости на единичных запросах до точки насыщения пропускной способности.

### Три сценария нагрузки

Для тестирования подготовлены три датасета, отражающих типовые production-сценарии:

- **Короткий чат** — имитация интерактивного диалога с короткими запросами, ключевая метрика — время до первого токена
- **RAG со средним контекстом** — типичная задача корпоративного поиска по базе знаний с контекстом на несколько тысяч токенов
- **RAG с длинным контекстом** — анализ больших документов с контекстом до 73 000 токенов, стресс-тест для подсистемы памяти

### Метрики

Для каждого прогона собираются четыре ключевые метрики с разбивкой по перцентилям (среднее, медиана, P99):

- **TTFT** — время до первого токена, критично для пользовательского опыта
- **TPOT** — время генерации одного токена, определяет скорость «печати»
- **Throughput** — общая пропускная способность в токенах в секунду
- **E2E Latency** — полное время обработки запроса

### Сравнение квантизаций

Каждая модель тестировалась в нескольких форматах точности — BF16, FP8, AWQ 4-bit и GPTQ-Int4 — что позволило выявить оптимальные режимы квантизации для каждого сценария. Одной из главных находок стала MoE-архитектура в GPTQ-Int4: качество модели на 30 миллиардов параметров при вычислительных затратах модели на 3 миллиарда.

## Автоматическая визуализация

Отдельный модуль визуализации автоматически строит интерактивные графики на базе Plotly по результатам всех прогонов. Для каждой модели генерируется набор из семи типов графиков: зависимость пропускной способности от параллельности, анализ TTFT и TPOT, распределение латентности по перцентилям, столбчатые диаграммы и комбинированная матрица метрик. Для сравнения нескольких моделей строятся те же графики с разбивкой по панелям. Графики экспортируются в HTML и PNG — именно они были использованы в итоговой статье на Хабр.

## Инфраструктура

Весь сервис контейнеризован: vLLM-серверы с моделями и оркестратор бенчмарка запускаются через Docker Compose. Сервис автоматически ожидает готовности инференс-серверов, прогоняет все тесты и сохраняет структурированные результаты в JSON. Воспроизвести бенчмарк на другом оборудовании можно без какой-либо ручной настройки.

## Стек технологий

**Инференс:** vLLM (оптимизированный движок для LLM-инференса)

**Анализ данных:** pandas, NumPy

**Визуализация:** Plotly (интерактивные графики), Kaleido (экспорт в PNG)

**Инфраструктура:** Docker, Docker Compose

## Ключевые результаты

- **Один из первых публичных бенчмарков DGX Spark** — объективная оценка нового устройства, ценный материал для ML-сообщества
- **Статья на Хабр** — результаты бенчмарка стали основой [подробной публикации](https://habr.com/ru/articles/994058/) с анализом реальных возможностей DGX Spark
- **Обнаружение эффекта плато** — throughput перестаёт расти после 50 параллельных запросов из-за ограничений пропускной способности памяти LPDDR5X (273 ГБ/с)
- **Автоматическая визуализация** — интерактивные графики Plotly генерировались автоматически и были напрямую вставлены в статью
- **Полная воспроизводимость** — контейнеризованный пайплайн, запуск без ручной настройки
