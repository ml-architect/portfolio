## О проекте

Mediaclass API — бэкенд десктопного приложения «МЕДИАКЛАСС ИИ», разработанный по заказу [DIGIS](https://mediaclass.digis.ru/) в рамках работы в AI/ML-интеграторе Palatine. Продукт транскрибирует аудио- и видеозаписи с разделением по спикерам. Основная аудитория — школы и вузы, которым нужна расшифровка лекций, уроков и совещаний с идентификацией каждого говорящего.

Сервис разворачивается на компьютере клиента и работает как локальный сервер в связке с Electron-приложением — данные не покидают периметр организации. Клиенты без мощного GPU могут прозрачно переключиться на облачную обработку одним параметром в запросе.

## Задача

Спроектировать автономный бэкенд-сервис для on-premise развёртывания с качественным распознаванием речи, диаризацией спикеров, гибридным переключением между локальным GPU-инференсом и облаком, а также собственной системой лицензирования с привязкой к оборудованию.

## Архитектура

### Гибридные ML-провайдеры

Единый пайплайн транскрибации работает в двух режимах через контрактные интерфейсы: локальный инференс на GPU и облачный через Palatine Cloud API. Добавление нового провайдера не затрагивает существующий код.

### Распознавание и диаризация

Сервис обрабатывает все основные аудио- и видеоформаты (MP3, WAV, OGG, FLAC, MP4, MOV, AVI и др.), автоматически извлекая аудиодорожку из видео. Транскрибация выдаёт текст с временными метками на уровне сегментов. Модели диаризации (сегментация, голосовые отпечатки, кластеризация) поставляются вместе с приложением и работают офлайн. Результат — JSON, где каждый фрагмент текста привязан к конкретному спикеру.

### Система лицензирования

Кроссплатформенная привязка к оборудованию (CPU, RAM, GPU) через уникальный аппаратный хеш с серверной верификацией. Поддерживаются Windows, Linux и macOS — каждая с собственным механизмом идентификации. Обеспечена обратная совместимость при изменении способа сбора аппаратной информации.

### Управление результатами

Результаты хранятся в структурированных директориях с пагинацией и сортировкой. Перед редактированием автоматически создаётся резервная копия (медиафайл + транскрипция). Поддерживается обрезка записи с удалением произвольного временного сегмента, склейкой и повторным распознаванием. При повреждении данных система восстанавливает их из бэкапа.

### Отказоустойчивость

Автоопределение GPU при запуске с fallback на CPU, ротация облачных токенов с автопереключением при сбоях, retry с экспоненциальным backoff, самовосстановление повреждённых директорий из резервных копий.

## Стек технологий

**Backend:** Python 3.10/3.11, FastAPI, Uvicorn, Pydantic v2

**ML/AI:** PyTorch, torchaudio, scikit-learn, SciPy

**Медиа:** FFmpeg, Pydub, MoviePy, SoundFile

**Инфраструктура:** Docker, Conda, PyArmor (шифрование кода), Loguru

## Ключевые достижения

- **Полностью автономный сервис** — on-premise развёртывание без зависимости от интернета при наличии GPU, конфиденциальность данных гарантирована
- **Гибридная ML-архитектура** — бесшовное переключение между локальным инференсом и облаком через контрактные провайдеры
- **Кроссплатформенное лицензирование** — привязка к оборудованию с серверной верификацией и обратной совместимостью
- **Коммерческая защита кода** — шифрование исходников для безопасной поставки конечным клиентам
- **Самовосстанавливающийся сервис** — автоопределение GPU, fallback на CPU, ротация токенов и восстановление из бэкапов без администратора
