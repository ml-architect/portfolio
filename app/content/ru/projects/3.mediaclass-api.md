## О проекте

Mediaclass API — бэкенд коммерческого десктопного приложения «МЕДИАКЛАСС ИИ», разработанный по заказу компании [DIGIS](https://mediaclass.digis.ru/) в рамках работы в AI/ML-интеграторе Palatine. Продукт предназначен для автоматического распознавания речи и разделения спикеров в аудио- и видеозаписях. Основная аудитория — образовательные учреждения: школы и вузы, которым необходимо транскрибировать записи лекций, уроков и совещаний с идентификацией каждого говорящего.

Ключевая особенность архитектуры — сервис полностью разворачивается на компьютере клиента и работает как локальный сервер в связке с десктопным Electron-приложением. Это позволяет обрабатывать записи без отправки данных в интернет, что критично для образовательных организаций с требованиями к конфиденциальности. При этом клиенты без мощного GPU могут прозрачно переключиться на облачную обработку — один параметр в запросе.

## Задача

Спроектировать и разработать бэкенд-сервис, который работает автономно на оборудовании клиента, обеспечивает высокое качество распознавания речи с разделением спикеров и поддерживает гибкое переключение между локальным GPU-инференсом и облачной обработкой. Решение должно поставляться в защищённом виде с собственной системой лицензирования, привязанной к аппаратному обеспечению конечного пользователя.

## Архитектура

### Гибридные ML-провайдеры

Центральное архитектурное решение — система провайдеров, абстрагирующая способ обработки речи. Единый пайплайн транскрибации прозрачно работает в двух режимах: локальный инференс на GPU пользователя и облачный режим через Palatine Cloud API. Провайдеры реализованы на контрактных интерфейсах — добавление нового способа обработки не затрагивает существующий код. Для клиентов с мощными видеокартами это означает полностью автономную работу без зависимости от интернета, а для остальных — возможность использовать облачные мощности без изменений в интерфейсе приложения.

### Распознавание и диаризация

Сервис обрабатывает все основные аудио- и видеоформаты (MP3, WAV, OGG, FLAC, MP4, MOV, AVI и др.), автоматически извлекая аудиодорожку из видеофайлов. Транскрибация выдаёт текст с точными временными метками на уровне отдельных сегментов. Диаризация — промышленного уровня: модели сегментации, генерации голосовых отпечатков и кластеризации спикеров поставляются вместе с приложением и работают полностью офлайн. Результат — структурированный JSON, где каждый фрагмент текста привязан к конкретному говорящему.

### Система лицензирования

Реализована собственная система лицензирования с привязкой к аппаратному обеспечению компьютера клиента. При активации лицензионного ключа сервис собирает информацию о процессоре, оперативной памяти и видеокарте, формирует уникальный аппаратный хеш и верифицирует его через выделенный сервер лицензирования. Система работает кроссплатформенно — поддерживаются Windows, Linux и macOS, каждая с собственным механизмом идентификации оборудования. Обеспечена обратная совместимость с ранее активированными лицензиями при изменении способа сбора аппаратной информации.

### Управление результатами и редактирование

Все результаты распознавания хранятся в структурированных директориях с поддержкой пагинации и сортировки. Перед любым редактированием автоматически создаётся резервная копия — медиафайл и транскрипция. Пользователь может обрезать запись с удалением произвольного временного сегмента и автоматической склейкой оставшихся частей, после чего распознавание перезапускается. При обнаружении повреждённых данных система автоматически восстанавливает их из бэкапа.

### Отказоустойчивость

Сервис спроектирован для автономной работы без администрирования: автоматическое определение GPU при запуске с переключением на CPU при его отсутствии, ротация облачных токенов с автоматическим переключением при сбоях, retry-стратегии с экспоненциальным backoff для сетевых запросов, самовосстановление повреждённых директорий из резервных копий.

## Стек технологий

**Backend:** Python 3.10/3.11, FastAPI, Uvicorn, Pydantic v2

**ML/AI:** PyTorch, torchaudio, scikit-learn, SciPy

**Медиа:** FFmpeg, Pydub, MoviePy, SoundFile

**Инфраструктура:** Docker, Conda, PyArmor (шифрование кода), Loguru

## Ключевые достижения

- **Полностью автономный сервис** — бэкенд разворачивается на хосте клиента и работает без зависимости от интернета при наличии GPU, обеспечивая конфиденциальность данных образовательных учреждений
- **Гибридная ML-архитектура** — контрактные провайдеры позволяют бесшовно переключаться между локальным GPU-инференсом и облачной обработкой одним параметром в запросе
- **Собственная система лицензирования** — кроссплатформенная привязка к аппаратному обеспечению с серверной верификацией и обратной совместимостью при обновлениях
- **Коммерческая защита кода** — шифрование исходного кода для безопасной поставки продукта конечным клиентам
- **Самовосстанавливающийся сервис** — автоопределение GPU, fallback на CPU, ротация облачных токенов и восстановление данных из бэкапов без участия администратора
