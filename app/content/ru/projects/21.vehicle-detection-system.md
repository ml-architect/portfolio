## О проекте

Высоконагруженная production-платформа компьютерного зрения для детекции транспорта и распознавания номерных знаков. Обрабатывает изображения с **6000 камер** в реальном времени — от детекции автомобиля до извлечения текста номера.

Платформа включает **inference-сервер** на NVIDIA Triton с TensorRT-оптимизацией и **асинхронную клиентскую библиотеку**. ML-модели были обучены отдельным разработчиком — моя зона ответственности: развёртывание, оптимизация инференса, конфигурация GPU-кластера и разработка клиентской библиотеки. Инференс выполняется на кластере из **6× NVIDIA RTX A6000**.

## Задача

Создать production-ready платформу инференса для обработки изображений с тысяч камер. Система должна обеспечивать детекцию транспорта, распознавание номерных знаков с минимальной латентностью и предоставлять асинхронный API для интеграции с внешними системами.

## Архитектура

### Inference-сервер (Triton + TensorRT)

Ядро системы — **NVIDIA Triton Inference Server** с 6 моделями, распределёнными по GPU. Модели оптимизированы в **TensorRT `.plan`**. Каждая модель развёрнута с настроенными instance groups: детекция транспорта и номеров — на GPU 1–3, распознавание текста — на GPU 0, сегментация — на GPU 5. Динамический batching обеспечивает утилизацию GPU при переменной нагрузке.

### Ensemble-пайплайн распознавания номеров

Распознавание номерных знаков реализовано как **Triton Ensemble** — цепочка из трёх моделей: препроцессинг (выравнивание 32×100, PyTorch), извлечение признаков (512-мерные эмбеддинги, TensorRT) и attention-декодирование текста (PyTorch). Ensemble выполняется как единый запрос к Triton.

### Модели компьютерного зрения

- **Детекция транспорта** — YOLO (TensorRT), 4 класса: легковой, грузовой, автобус, фургон
- **Детекция номерных знаков** — WPOD-Net (TensorRT), перспективная коррекция через аффинные преобразования
- **Распознавание текста номера** — Ensemble из 3 модулей с attention-декодером
- **Сегментация транспорта** — Instance segmentation (TensorRT), маски + bounding boxes

### Асинхронная клиентская библиотека

Типизированная Python-библиотека на основе **асинхронного gRPC-стриминга**. Архитектура на обобщённом базовом классе — каждая модель наследует единый интерфейс с препроцессингом, постобработкой и инференсом, возвращая AsyncIterator результатов. Все ответы строго типизированы.

### Постобработка и фильтрация

NMS для удаления дубликатов, пороговая фильтрация по confidence, перспективная коррекция номеров через аффинные преобразования и IoU-фильтрация через Shapely для выбора лучшего номера в области транспортного средства.

### Web-интерфейс для тестирования

Streamlit-приложение для интерактивного тестирования: загрузка изображения, выбор задачи, визуализация результатов с bounding boxes, масками и распознанным текстом.

## Стек технологий

**ML Inference:** NVIDIA Triton Inference Server, TensorRT, PyTorch 1.13, CUDA 11.7, Triton Ensemble

**Computer Vision:** OpenCV, YOLO, WPOD-Net, Instance Segmentation, NMS

**Backend:** Python 3.10, asyncio, gRPC (tritonclient.grpc.aio), NumPy, Pandas, Shapely, Pillow

**Инфраструктура:** Docker, Docker Compose, 6× NVIDIA RTX A6000, Poetry, GitLab CI/CD

**Тестирование:** Streamlit, pytest, pytest-asyncio, mypy (strict), Black

## Ключевые достижения

- **6000 камер в реальном времени** — стабильная обработка потоков на GPU-кластере из 6× RTX A6000
- **Латентность <50 мс** — TensorRT-оптимизация и распределение instance groups по GPU
- **Ensemble-пайплайн** — трёхступенчатое распознавание номеров как единый запрос к Triton
- **Multi-GPU инференс** — 6 моделей распределены по 6 GPU с динамическим batching
- **Async-first клиент** — gRPC-стриминг с типизированными результатами и generic-паттерном
- **Два подхода к детекции номеров** — WPOD-Net и segmenter с IoU-фильтрацией, выбор лучшего через Shapely
- **Production-grade код** — mypy strict, pytest, Poetry, GitLab CI/CD
