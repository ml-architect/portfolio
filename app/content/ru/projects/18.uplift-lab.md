## О проекте

В рамках образовательной платформы [Лаборатории интеллектуального анализа РАНХиГС](https://lia.ranepa.ru/) я спроектировал Uplift Lab — интерактивную веб-лабораторию с концепцией «Аналитик vs ML». Студент вручную сегментирует аудиторию маркетинговой кампании через систему фильтров, а затем сравнивает свой результат с ML-моделями. Delta-метрики наглядно демонстрируют превосходство алгоритмического подхода над ручной аналитикой.

Данные — бенчмарк MineThatData E-Mail Analytics (Kevin Hillstrom): 42 000+ записей клиентов с разделением на контрольную и тестовую группы.

## Функционал

Лаборатория начинается с EDA: интерактивные графики показывают распределение клиентов по каналам покупки, ценовым сегментам, давности заказа и другим признакам.

Затем студент сегментирует аудиторию через шесть фильтров: канал покупки, тип клиента, категория товаров, ценовой сегмент, география, давность заказа. По сути — решает ту же задачу, что и маркетолог в реальной компании. Валидация на каждом шаге предотвращает ошибки.

После применения фильтров рассчитываются метрики ручной сегментации (uplift@k, Qini AUC, взвешенный средний uplift) и те же метрики для ML-моделей. Результат — delta-метрики с абсолютной разницей между подходами. На графиках Qini и Uplift curves красный маркер фиксирует позицию ручной аналитики на фоне ML-кривых.

## Архитектура

### ML-пайплайн

Обучено 12 комбинаций моделей: три алгоритма (CatBoost, XGBoost, Random Forest) в четырёх подходах к uplift-моделированию через scikit-uplift. Подходы различаются стратегией работы с контрольной и тестовой группами — от единой модели на объединённых данных до пары зависимых моделей с DDR. Лучший результат: CatBoost + SoloModel — uplift@30% = 0.0776.

Обучение CatBoost и XGBoost — с GPU-ускорением. Воспроизводимость обеспечена фиксированными random seeds и сохранёнными индексами разбиения выборки.

### Мгновенный отклик без GPU

Модели обучаются офлайн, предсказания сохраняются в CSV. Приложение при запуске загружает готовые результаты — мгновенный отклик интерфейса без GPU-зависимости на сервере.

### CI/CD

Приложение развёрнуто на Hugging Face Spaces (Streamlit SDK). GitHub Actions при каждом пуше проверяет размеры файлов и синхронизирует репозиторий с Hugging Face Hub. Бинарные артефакты хранятся через Git LFS.
