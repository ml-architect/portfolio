## О проекте

Образовательная платформа [Лаборатории интеллектуального анализа РАНХиГС](https://lia.ranepa.ru/) включает курсы по анализу данных, в рамках которых студенты изучают методы оценки бизнес-гипотез. A/B-тестирование — один из ключевых инструментов в арсенале аналитика, но освоить его по лекциям сложно: нужно понимать, как формулировать гипотезы, рассчитывать размер выборки, интерпретировать p-value и избегать типичных ошибок вроде подглядывания в результаты до завершения эксперимента.

Я спроектировал и разработал A/B-test Lab — интерактивную веб-лабораторию, которая проводит студента через полный цикл A/B-тестирования шаг за шагом: от постановки цели эксперимента до статистического анализа результатов. Приложение использовалось в реальном учебном процессе на курсах РАНХиГС и развёрнуто на Hugging Face Spaces.

## Функционал

Лаборатория реализует шестиступенчатый сценарий, в котором каждый шаг открывается только после корректного прохождения предыдущего — студент не может «перескочить» к результатам, не разобравшись в методологии.

На первых двух шагах пользователь определяет цель эксперимента и выбирает метрику. Приложение предлагает варианты ответов: правильный выбор продвигает вперёд, а ошибочные — сопровождаются пояснениями, почему этот вариант не подходит для A/B-теста. Такой подход превращает лабораторию в интерактивный тренажёр, а не просто калькулятор.

Далее студент формулирует статистическую гипотезу — выбирает между односторонним и двусторонним тестом — и настраивает параметры эксперимента: уровень значимости, длительность теста, объём ежедневного трафика и размер тестовой выборки. На этом этапе лаборатория наглядно объясняет проблему раннего подглядывания в результаты (peeking problem) и почему размер выборки необходимо зафиксировать до начала эксперимента.

После настройки параметров система генерирует синтетический датасет, имитирующий реальный A/B-тест: пользователей случайным образом распределяет по группам с различными конверсиями, а временные метки равномерно распределяются по выбранному периоду. Интерактивная гистограмма показывает распределение конверсий между группами.

На финальном шаге лаборатория проводит полный статистический анализ: рассчитывает конверсии обеих групп, определяет абсолютную и относительную разницу, вычисляет z-статистику и p-value, а затем выносит вердикт о статистической значимости. Результаты представлены в виде таблицы метрик с цветовой индикацией и столбчатой диаграммы сравнения групп.

Ключевая визуализация лаборатории — график эволюции p-value во времени. Система пересчитывает статистическую значимость на каждом промежутке данных, показывая, как p-value колеблется на малых выборках и стабилизируется по мере накопления наблюдений. Горизонтальная пороговая линия наглядно демонстрирует, почему остановка эксперимента в момент случайного «пересечения порога» приводит к ложным выводам — именно ту проблему peeking, о которой лаборатория предупреждала ранее.

## Архитектура

### Двухмодульная структура

Приложение разделено на два слоя: презентационный и вычислительный. Презентационный модуль управляет пошаговым сценарием через систему логических условий — каждый шаг контролирует доступ к следующему. Вычислительный модуль инкапсулирует всю статистическую логику и визуализации, предоставляя чистый интерфейс для расчётов.

### Статистический движок

В основе лаборатории лежит Z-тест для пропорций — стандартный метод проверки различий между двумя группами в A/B-тестировании. Движок реализует полную цепочку вычислений: конверсия групп, стандартная ошибка пропорции, стандартная ошибка разности, z-статистика и p-value с учётом выбранного типа гипотезы. Для визуализации используются три библиотеки: Altair для сравнительных столбчатых диаграмм, Plotly для интерактивных гистограмм распределений и линейных графиков эволюции p-value.

### CI/CD и развёртывание

Приложение развёрнуто на Hugging Face Spaces со Streamlit SDK. Автоматический пайплайн на GitHub Actions при каждом пуше проверяет размеры файлов на совместимость с платформой и синхронизирует репозиторий с Hugging Face Hub. Данные хранятся через Git LFS.

## Ключевые достижения

- **Образовательный инструмент для РАНХиГС** — интерактивная лаборатория, позволяющая студентам пройти полный цикл A/B-тестирования на практике, от формулирования гипотезы до интерпретации результатов
- **Наглядная демонстрация peeking problem** — график эволюции p-value во времени показывает, почему нельзя останавливать эксперимент при первом «значимом» результате, формируя правильные привычки проведения экспериментов
- **Интерактивный тренажёр, а не калькулятор** — пошаговый сценарий с проверкой ответов обеспечивает осмысленное прохождение каждого этапа методологии
- **Автоматическое развёртывание** — CI/CD через GitHub Actions обеспечивает непрерывную доставку обновлений на Hugging Face Spaces
