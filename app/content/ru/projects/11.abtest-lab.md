## О проекте

[Лаборатория интеллектуального анализа РАНХиГС](https://lia.ranepa.ru/) проводит курсы по анализу данных, где студенты осваивают A/B-тестирование. Освоить его по лекциям сложно: нужно уметь формулировать гипотезы, рассчитывать выборку, интерпретировать p-value и избегать ошибок вроде раннего подглядывания в результаты.

A/B-test Lab — интерактивная веб-лаборатория, которая проводит студента через полный цикл A/B-тестирования: от постановки цели до статистического анализа. Использовалась в учебном процессе РАНХиГС, развёрнута на Hugging Face Spaces.

## Функционал

Лаборатория реализует шестиступенчатый сценарий с последовательным доступом — каждый шаг открывается после корректного прохождения предыдущего.

**Шаги 1–2: цель и метрика.** Студент выбирает цель эксперимента и метрику из предложенных вариантов. Ошибочные ответы сопровождаются пояснениями — лаборатория работает как тренажёр, а не калькулятор.

**Шаг 3: гипотеза и параметры.** Выбор между односторонним и двусторонним тестом, настройка уровня значимости, длительности, объёма трафика и размера выборки. Здесь же лаборатория объясняет peeking problem и необходимость фиксировать выборку до начала эксперимента.

**Шаг 4: генерация данных.** Система создаёт синтетический датасет: случайное распределение пользователей по группам с разными конверсиями, временные метки по выбранному периоду. Интерактивная гистограмма показывает распределение конверсий.

**Шаги 5–6: анализ и визуализация.** Расчёт конверсий групп, абсолютной и относительной разницы, z-статистики и p-value с вердиктом о значимости. Результаты — таблица метрик с цветовой индикацией и столбчатая диаграмма.

Ключевая визуализация — график эволюции p-value во времени. Система пересчитывает значимость на каждом промежутке данных, показывая колебания p-value на малых выборках и стабилизацию при накоплении наблюдений. Пороговая линия демонстрирует, почему остановка эксперимента при случайном «пересечении порога» ведёт к ложным выводам.

## Архитектура

### Двухмодульная структура

Презентационный слой управляет пошаговым сценарием через систему логических условий. Вычислительный слой инкапсулирует статистическую логику и визуализации, предоставляя чистый интерфейс для расчётов.

### Статистический движок

Z-тест для пропорций с полной цепочкой вычислений: конверсия групп, стандартная ошибка пропорции, стандартная ошибка разности, z-статистика и p-value с учётом типа гипотезы. Визуализация: Altair (столбчатые диаграммы), Plotly (гистограммы распределений и графики эволюции p-value).

### CI/CD и развёртывание

Hugging Face Spaces со Streamlit SDK. GitHub Actions при каждом пуше проверяет размеры файлов и синхронизирует репозиторий с HF Hub. Данные — через Git LFS.

## Ключевые достижения

- **Образовательный инструмент для РАНХиГС** — полный цикл A/B-тестирования на практике, от гипотезы до интерпретации результатов
- **Демонстрация peeking problem** — график эволюции p-value показывает, почему нельзя останавливать эксперимент при первом «значимом» результате
- **Тренажёр, а не калькулятор** — пошаговый сценарий с проверкой ответов на каждом этапе методологии
- **CI/CD** — автоматическая доставка обновлений на Hugging Face Spaces через GitHub Actions

**Веб-интерфейс:** Streamlit, Plotly, Altair
