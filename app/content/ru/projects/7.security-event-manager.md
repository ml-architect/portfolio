## О проекте

Security Event Manager (SEM) — платформа автоматизации SOC, развёрнутая в 8 центрах федеральных округов России. Каждый инстанс обрабатывает тысячи SIEM-событий в сутки, замыкая полный цикл реагирования: детекция угроз через ML и Sigma-правила, классификация, эскалация инцидентов в TheHive. Локальная LLM помогает аналитикам исследовать события без отправки данных за периметр.

## Задача

Аналитики SOC вручную разбирали поток событий из SIEM — основное время уходило на рутинную классификацию, значительная часть инцидентов оставалась незамеченной. Требовалась платформа для автоматизации анализа, матчинга с правилами детекции и классификации атак. Критическое требование — все данные, включая AI-ассистента, остаются внутри контура безопасности.

## Архитектура

Двухконтейнерная система с отдельным AI-сервером.

**Web Container** — FastAPI REST API (17 роутеров) + Vue 3 SPA. Async SQLite для конфигурации и правил, psycopg2 — для чтения событий из PostgreSQL SIEM. Параллельный сбор данных из 5 источников через `asyncio.gather` для быстрой загрузки дашборда.

**Back Container** — автономный процессор с циклами по 30–40 минут. Забирает события из SIEM, прогоняет через CatBoost и Sigma-движок, группирует по IP, присваивает severity, создаёт алерты в TheHive. Конфигурацию получает от Web Container по HTTP API (retry with backoff при старте).

**Ollama** — локальный LLM-сервер (Qwen3, DeepSeek-R1) для AI-ассистента аналитиков: интерпретация событий, анализ подозрительной активности. Конвертер Git2Markdown загружает код из репозиториев в контекст LLM.

### Sigma Rules Engine

Кастомный движок с нуля. Recursive descent парсер с полным синтаксисом Sigma: AND/OR/NOT, wildcard, кванторы ("1 of", "all of"), модификаторы (contains, startswith, endswith, re, cidr). Расширен фильтрацией по сенсорам и группам СОА. 11 тестовых сценариев. UI для создания, редактирования и импорта правил в YAML.

### ML-детекция

Двухуровневый CatBoost: первая модель определяет факт атаки, вторая классифицирует тип (DNS-атаки, троянская активность, майнерское ВПО и др.). В связке с Sigma-правилами и тегированием обеспечивает многослойную детекцию.

### Управление правилами и вайтлистами

Многоуровневые правила тегирования (до 4 уровней вложенности, логика AND/OR) для сложных цепочек событий. Вайтлисты трёх типов (сенсоры, сигнатуры, IP) с таймерами истечения и уведомлениями. Аналитика правил автоматически выявляет мёртвые (без срабатываний), шумящие (аномально частые) и деградирующие (падение > 50%) правила.

## Стек технологий

**Backend:** Python 3.11, FastAPI 0.109, aiosqlite, psycopg2, Alembic, HMAC-SHA256, bcrypt

**Frontend:** Vue 3.4, TypeScript 5.3, Vite, Pinia, Tailwind CSS 3.4, Radix Vue, shadcn/vue

**ML/AI:** CatBoost 1.2.7, Ollama (Qwen3, DeepSeek-R1), pySigma

**Инфраструктура:** Docker (multi-stage), Docker Compose, GitLab CI/CD

**Интеграции:** TheHive, SIEM PostgreSQL

## Ключевые достижения

- **x3 к обнаружению инцидентов** — ML + Sigma выявляют втрое больше инцидентов по сравнению с ручным анализом
- **Сокращение времени обработки на 70%** — аналитики фокусируются на расследовании вместо сортировки событий
- **8 инстансов по всей России** — каждый обрабатывает тысячи событий в сутки от региональных SIEM
- **Многослойная детекция** — ML + Sigma + тегирование практически исключили пропуск значимых событий
- **Локальная LLM** — AI-ассистент на Ollama работает внутри контура безопасности
- **Самоочищающаяся rule-база** — автоматическое выявление неэффективных правил без ручного аудита
