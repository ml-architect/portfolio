## О проекте

Security Event Manager (SEM) — платформа автоматизации работы SOC-центров (Security Operations Center), развёрнутая в 8 центрах федеральных округов России. Каждый инстанс обрабатывает тысячи событий безопасности в сутки от региональных SIEM-систем, автоматически выявляя компьютерные атаки и инциденты.

Система замыкает полный цикл реагирования: непрерывный анализ потока SIEM-событий, детекция угроз с помощью ML-моделей и Sigma-правил, автоматическая классификация и эскалация инцидентов в TheHive. Встроенная локальная LLM помогает аналитикам исследовать события и принимать решения — без отправки данных за периметр.

## Задача

До внедрения SEM аналитики SOC вручную разбирали поток событий из SIEM, тратя основное время на рутинную классификацию и не успевая обрабатывать весь объём данных. Значительная часть компьютерных инцидентов оставалась незамеченной из-за человеческого фактора и ограниченного времени.

Требовалось создать платформу, которая автоматизирует рутинные операции — анализ тысяч событий, матчинг с правилами детекции, классификацию типов атак — и позволит аналитикам сосредоточиться на расследовании реальных угроз. Критическое требование: все данные, включая работу AI-ассистента, должны оставаться внутри контура безопасности организации.

## Архитектура

SEM построен как двухконтейнерная микросервисная система с отдельным AI-сервером.

**Web Container** — FastAPI REST API (17 роутеров) с Vue 3 SPA, обслуживающий интерфейс аналитика. Async SQLite хранит конфигурацию, правила, вайтлисты и пользователей. Через psycopg2 подключается к PostgreSQL SIEM для чтения событий и дашборд-аналитики. Параллельный сбор данных из 5 источников через `asyncio.gather` обеспечивает быструю загрузку дашборда.

**Back Container** — автономный процессор, работающий в циклах по 30–40 минут. В каждом цикле он забирает новые события из SIEM, прогоняет их через CatBoost-модели и Sigma-движок, группирует по IP-адресам, присваивает severity и создаёт алерты в TheHive. Процессор получает конфигурацию от Web Container через HTTP API и автоматически ожидает его готовности при старте (retry with backoff, 30 попыток).

**Ollama** — локальный LLM-сервер с моделями Qwen3 и DeepSeek-R1. Обеспечивает AI-ассистента для аналитиков: помощь в интерпретации событий, анализ подозрительной активности, работа с документацией. Конвертер Git2Markdown загружает код из репозиториев прямо в контекст LLM.

### Sigma Rules Engine

Кастомный движок, написанный с нуля. Recursive descent парсер поддерживает полный синтаксис Sigma: логические операторы AND/OR/NOT, wildcard-паттерны, кванторы ("1 of", "all of"), модификаторы (contains, startswith, endswith, re, cidr). Расширен фильтрацией по сенсорам с поддержкой групп СОА. Покрыт 11 тестовыми сценариями. Через UI аналитик может создавать, редактировать и импортировать правила в формате YAML.

### ML-детекция

Двухуровневая система на CatBoost: первая модель определяет факт атаки, вторая классифицирует её тип (DNS-атаки, троянская активность, майнерское ВПО и др.). Модели загружаются при старте и применяются к каждому формируемому инциденту. В связке с Sigma-правилами и системой тегирования ML-детекция обеспечивает многослойную защиту, где разные механизмы дополняют друг друга.

### Управление правилами и вайтлистами

Система поддерживает многоуровневые правила тегирования (до 4 уровней вложенности с логикой AND/OR) для описания сложных цепочек событий. Вайтлисты трёх типов (сенсоры, сигнатуры, IP-адреса) с таймерами истечения и автоматическими уведомлениями не дают подавлять легитимные алерты дольше необходимого. Аналитика правил автоматически выявляет мёртвые (без срабатываний), шумящие (аномально частые) и деградирующие (падение > 50%) правила.

## Стек технологий

**Backend:** Python 3.11, FastAPI 0.109, aiosqlite, psycopg2, Alembic, HMAC-SHA256, bcrypt

**Frontend:** Vue 3.4, TypeScript 5.3, Vite, Pinia, Tailwind CSS 3.4, Radix Vue, shadcn/vue

**ML/AI:** CatBoost 1.2.7, Ollama (Qwen3, DeepSeek-R1), pySigma

**Инфраструктура:** Docker (multi-stage), Docker Compose, GitLab CI/CD

**Интеграции:** TheHive, SIEM PostgreSQL

## Ключевые достижения

- **x3 к обнаружению инцидентов** — автоматическая детекция через ML и Sigma-правила выявляет втрое больше компьютерных инцидентов по сравнению с ручным анализом
- **Сокращение времени обработки на 70%** — аналитик тратит время на расследование, а не на первичную сортировку потока событий
- **8 инстансов по всей России** — сервис развёрнут в центрах федеральных округов, каждый обрабатывает тысячи событий в сутки от региональных SIEM
- **Снижение пропущенных угроз** — многослойная детекция (ML + Sigma + правила тегирования) практически исключила ситуации, когда значимые события проходили незамеченными
- **Локальная LLM без утечки данных** — AI-ассистент на Ollama работает в контуре безопасности, помогая аналитикам без риска компрометации
- **Самоочищающаяся rule-база** — автоматическое выявление неэффективных правил поддерживает качество детекции без ручного аудита
