## О проекте

Contract Risk Analyzer — production-ready AI-система для автоматизированного анализа рисков в международных контрактах на морские грузоперевозки зерна. Разработана для **ТД «Зерно Заволжья»** — одного из крупнейших зерновых трейдеров России.

Трейдеры ежедневно обрабатывают десятки контрактов на купле-продажу и перевозку зерновых грузов по международным маршрутам. Каждый контракт содержит десятки юридических формулировок, влияющих на финансовые риски: условия оплаты, демерредж, лайтайм, качество груза, фрахт, страхование, арбитражные оговорки (GAFTA/FOSFA). Ручной анализ занимал часы и был подвержен человеческим ошибкам.

Система автоматизирует процесс: пользователь загружает DOCX-контракт через Telegram-бота, AI анализирует текст по 111 рисковым и обязательным фразам с учётом контекста (продукт, страна, базис Incoterms, порт) и выдаёт структурированный отчёт с уровнями риска, рекомендациями и цитатами из договора.

## Задача

Создать AI-систему, способную заменить ручной анализ контрактов юристами и трейдерами. Провести полный ML-цикл: от исследования и бенчмаркинга моделей через обучение собственной LLM до production-деплоя. Обеспечить точность анализа через двухступенчатую верификацию, а также возможность перехода с коммерческого GPT-4o API на self-hosted модель для конфиденциальности данных заказчика.

## Архитектура

### Contract Risk Analyze API

Ядро системы — REST API на FastAPI с двухступенчатым LLM-пайплайном. Первая LLM (GPT-4o) анализирует контракт по каждой из 111 рисковых фраз, вторая LLM верифицирует ответ и может перевернуть решение (паттерн **LLM-as-a-Judge**). Каждая фраза проверяется контекстно-зависимо — с учётом продукта (wheat, sunflower oil), страны назначения, базиса поставки (CIF, CFR, FOB, CIFFO) и порта отгрузки.

Обработка контрактов — асинхронная через BackgroundTasks с polling статуса и прогрессом. Результат включает: наличие фразы, chain-of-thought, рекомендации, цитату из договора, подтверждение верификации.

### Contract Risk Analyze Bot

Telegram-бот на aiogram 3 — интерфейс для сотрудников ТДЗЗ. Загрузка DOCX-контрактов, polling статуса задач с 2-секундным интервалом, генерация Excel-отчётов (openpyxl) со всеми результатами анализа.

### Research & Model Training

Полный R&D-цикл:
- **Бенчмарки** — сравнение GPT-4o, Llama 3.3, Qwen2.5 32B, Quepasa SaaS RAG, LightRAG с метриками balanced_accuracy на 4 реальных контрактах
- **RAG-эксперименты** — Graph-based RAG (LightRAG) с мультиязычными эмбеддингами (multilingual-e5-large-instruct, 1024-dim) и гибридным поиском
- **Генерация данных** — пайплайн на Distilabel: 59 контрактов × 111 фраз = 5883 обучающих примера
- **Fine-tuning** — серия моделей Qwen2.5 (0.5B–32B) с QLoRA через Unsloth, RoPE Scaling для контекста 18000 токенов, публикация на HuggingFace Hub в GGUF-формате

## Стек технологий

**Backend:** Python 3.12, FastAPI, LangChain (цепочки, JsonOutputParser, @chain), Pydantic v2, aiogram 3, aiohttp, openpyxl

**ML/AI:** OpenAI GPT-4o, Unsloth (2x ускорение fine-tuning), QLoRA (4bit NF4, rank 32–256), TRL (SFTTrainer), Transformers, Qwen2.5, Distilabel, LightRAG, Ollama, SetFit, PyTorch + CUDA 12.1

**Данные:** python-docx, markitdown, Pandas, scikit-learn (balanced_accuracy_score), HuggingFace Hub, Git LFS

**Инфраструктура:** Docker Compose, GitLab CI/CD, Makefile, pre-commit hooks (Ruff), HTTP Basic Auth

## Ключевые достижения

- **Полный ML-цикл** — от R&D и бенчмаркинга 6+ моделей через генерацию данных и fine-tuning до production-деплоя. Систематический подход к выбору архитектуры, подтверждённый метриками
- **LLM-as-a-Judge** — двухступенчатая верификация, где вторая модель проверяет и может перевернуть решение первой, значительно повышая точность анализа
- **Domain-specific NLP** — 111 рисковых фраз с 5 уровнями риска, 200+ отраслевых терминов (GAFTA, FOSFA, Incoterms), контекстно-зависимая проверка по 4 параметрам
- **Fine-tuning для замены GPT-4o** — собственная модель на Qwen2.5 32B с QLoRA, обеспечивающая конфиденциальность данных заказчика и контроль стоимости инференса
- **Генерация обучающих данных** — пайплайн на Distilabel для создания 5883 примеров из реальных контрактов с автоматической разметкой
- **Модель на HuggingFace** — обученная модель опубликована в GGUF-формате для self-hosted деплоя через Ollama
