## О проекте

Production-ready AI-система анализа рисков в международных контрактах на морские грузоперевозки зерна. Разработана для **ТД «Зерно Заволжья»** — одного из лидеров зернового экспорта России.

Юристы и трейдеры компании ежедневно обрабатывают десятки контрактов, каждый из которых содержит формулировки с прямым влиянием на финансовые риски: демерредж, лайтайм, фрахт, страхование, арбитражные оговорки. Ручной анализ одного контракта занимал 2–3 часа и был подвержен ошибкам — пропущенная формулировка могла стоить сотни тысяч долларов.

Система автоматизирует процесс: сотрудник загружает контракт через Telegram-бота, AI-модель проверяет текст по 111 рисковым формулировкам с учётом контекста сделки (тип продукта, страна назначения, базис Incoterms, порт отгрузки) и формирует Excel-отчёт с уровнями риска, рекомендациями и цитатами из договора.

## Задача

Провести полный ML-цикл (R&D → бенчмарк → fine-tuning → production) для замены ручного анализа контрактов. Обеспечить высокую точность через двухступенчатую LLM-верификацию и заложить переход с коммерческого API на self-hosted модель для конфиденциальности данных заказчика.

## Архитектура

Микросервисная архитектура из трёх компонентов в общей Docker-сети: аналитическое API, Telegram-бот и исследовательский модуль.

### Аналитическое API

Асинхронный REST API с двухступенчатым LLM-пайплайном (**LLM-as-a-Judge**). На первом этапе основная модель анализирует контракт по каждой из 111 формулировок, генерируя chain-of-thought рассуждения, рекомендации и цитаты. На втором этапе независимая модель-верификатор получает исходный контракт и первоначальный анализ, подтверждая или опровергая решение. Это значительно снижает и ложноположительные, и ложноотрицательные срабатывания.

Каждая формулировка проверяется контекстно-зависимо — с учётом типа товара, страны назначения, базиса поставки и порта отгрузки. Например, ограничение на тип судов является риском только при определённом базисе и портах.

Результат по каждой формулировке: факт наличия, уровень риска, цепочка рассуждений, рекомендация, цитата и результат верификации.

### Telegram-бот

Интерфейс для сотрудников: загрузка контракта, отслеживание прогресса в реальном времени, получение Excel-отчёта для передачи между отделами.

### Research & Model Training

**Бенчмаркинг.** Систематическое сравнение коммерческих LLM, открытых моделей различных масштабов и RAG-решений (SaaS-платформы, собственный графовый RAG с мультиязычными эмбеддингами и гибридным поиском). Оценка по balanced accuracy на реальных контрактах с экспертной разметкой. Двухступенчатый LLM-пайплайн показал лучшее соотношение точности и надёжности.

**Генерация обучающих данных.** 59 реальных контрактов x 111 формулировок = ~6000 обучающих примеров. Каждый пример включал системную инструкцию, описание формулировки, отраслевой словарь, текст контракта и структурированный JSON-ответ.

**Fine-tuning.** Серия экспериментов с открытыми LLM от <1B до 32B параметров. QLoRA (4-bit NF4, rank 32–256) позволил обучать крупные модели на доступном GPU. Обучение только на ответной части, масштабирование контекста до 18 000 токенов. Итоговая модель сопоставима по качеству с коммерческим API при полном контроле над данными.

## Стек технологий

**Backend:** Python 3.12, FastAPI, LangChain (цепочки, JsonOutputParser, @chain), Pydantic v2, aiogram 3, aiohttp, openpyxl

**ML/AI:** OpenAI GPT-4o, Unsloth (2x ускорение fine-tuning), QLoRA (4bit NF4, rank 32–256), TRL (SFTTrainer), Transformers, Qwen2.5, Distilabel, LightRAG, Ollama, SetFit, PyTorch + CUDA 12.1

**Данные:** python-docx, markitdown, Pandas, scikit-learn (balanced_accuracy_score), HuggingFace Hub, Git LFS

**Инфраструктура:** Docker Compose, GitLab CI/CD, Makefile, pre-commit hooks (Ruff), HTTP Basic Auth

## Ключевые достижения

- **Ускорение в 12–15 раз** — с 2–3 часов до 10–15 минут на контракт; пропускная способность отдела выросла с 3–5 до 20–30 контрактов в день
- **Полнота проверки** — все 111 формулировок проверяются без исключения; двухступенчатая верификация (LLM-as-a-Judge) снизила пропуски рисков и ложные срабатывания
- **Стандартизация** — единый Excel-формат отчётов для юридического, торгового и логистического отделов устранил разночтения и ускорил согласование сделок
- **Полный ML-цикл** — R&D → бенчмарк 6 подходов → генерация данных → fine-tuning → production. Каждое решение подтверждено метриками на реальных контрактах
- **Domain-specific NLP** — 111 рисковых формулировок (5 уровней риска), 200+ отраслевых терминов, контекстно-зависимая проверка по 4 параметрам сделки
- **Self-hosted LLM** — fine-tuning собственной модели обеспечил конфиденциальность контрактных данных и прогнозируемую стоимость инференса
