## О проекте

Findata BOT — production-ready финтех-система в формате Telegram-бота, созданная для инвесторов на российском облигационном рынке. Система решает ключевую проблему: инвестору в облигации необходимо ежедневно отслеживать десятки компаний-эмитентов на нескольких разрозненных площадках — порталы раскрытия информации, рейтинговые агентства, биржевые данные. Это отнимает часы времени и создаёт риск пропустить критически важное событие — от изменения рейтинга до неисполнения обязательств.

Findata BOT полностью автоматизирует этот процесс. Система агрегирует данные из шести официальных источников — e-disclosure.ru, Национальный расчётный депозитарий (НРД), рейтинговые агентства АКРА, НРА и Эксперт РА, а также isin.ru — и доставляет персонализированные уведомления с учётом конкретных облигаций в портфеле пользователя. Инвестор получает только релевантную информацию, отфильтрованную по международным идентификаторам ценных бумаг (ISIN) и ключевым фразам.

Проект полностью монетизирован: реализованы тарифные планы с пробным периодом, автоматический приём платежей с фискализацией, управление подписками и администрирование.

## Задача

Создать надёжную финтех-платформу, которая превращает ручной мониторинг разрозненных площадок в автоматическую доставку персонализированных уведомлений. Платформа должна охватывать все ключевые источники информации для инвестора в облигации, интегрироваться с брокерскими API для автоматической привязки портфеля, обеспечивать безопасность финансовых данных и поддерживать полный бизнес-цикл — от регистрации пользователя до оплаты и управления подпиской.

## Функционал

### Мониторинг корпоративных раскрытий
Пользователь подписывается на интересующие компании-эмитенты, а система автоматически отслеживает новости и события на портале раскрытия информации. Уведомления фильтруются по конкретным ISIN-кодам облигаций и триггер-фразам — инвестор получает только то, что касается его портфеля. Отдельно отслеживаются четыре типа финансовой отчётности: устав, годовые отчёты, бухгалтерская отчётность и отчётность по МСФО. При обнаружении критичных событий — таких как неисполнение обязательств — система немедленно предупреждает пользователя.

### Рейтинговые уведомления
Параллельно система мониторит три ведущих российских рейтинговых агентства — АКРА, НРА и Эксперт РА — а также Национальный расчётный депозитарий. Рейтинговые действия и события доставляются в двух режимах: точечно — по совпадению ISIN-кодов облигаций в портфеле пользователя, и широковещательно — для событий, важных всем инвесторам.

### Мониторинг новых облигаций
Система отслеживает появление новых облигаций на рынке: собирает свежие ISIN-коды за скользящее окно в 30 дней и проверяет их доступность для покупки у брокера. Как только новая облигация становится доступна — пользователь получает уведомление.

### Привязка брокерского портфеля
Интеграция с T-Bank Invest API позволяет автоматически подключить брокерский счёт. Система безопасно сохраняет API-токен в зашифрованном виде и регулярно синхронизирует портфель облигаций в ночном окне, минимизируя нагрузку на внешний API. На основе актуального состава портфеля автоматически создаются, обновляются и деактивируются подписки на соответствующих эмитентов — инвестору не нужно вручную настраивать мониторинг.

### Монетизация
Реализована полноценная подписочная модель с несколькими тарифными планами: от бесплатного с ограничением по количеству компаний до премиального с полным доступом ко всем функциям. Предусмотрен пробный период для новых пользователей, автоматические напоминания об истечении подписки и плавный переход на бесплатный тариф без потери настроек.

## Архитектура

### Двухпроцессная модель
Система разделена на два независимых процесса, работающих параллельно. Первый отвечает за взаимодействие с пользователем через Telegram — обработку команд, навигацию по меню, настройку подписок и управление аккаунтом. Второй — набор фоновых задач, каждая из которых работает по своему расписанию: мониторинг источников данных, рассылка уведомлений, синхронизация портфелей, обработка платежей и управление подписками. Такое разделение позволяет бот-интерфейсу оставаться отзывчивым вне зависимости от нагрузки на фоновые процессы.

### Расширяемая система источников данных
Архитектура источников данных построена на модульном принципе с единым реестром. Каждый источник — это независимый модуль, реализующий общий интерфейс: получение данных, парсинг, нормализация событий и доставка уведомлений. Добавление нового источника сводится к реализации одного модуля и его регистрации — остальная инфраструктура (планирование, дедупликация, доставка) подхватывает его автоматически. Это позволяет быстро расширять охват системы при появлении новых площадок или изменении существующих.

### Управление подписками
Подписки спроектированы с учётом жизненного цикла инвестора. При привязке брокерского портфеля подписки создаются автоматически на основе текущего состава облигаций. При смене тарифа подписки не удаляются, а мягко деактивируются — пользователь может вернуть их одним действием при повышении тарифа. При ночной синхронизации портфеля система обнаруживает изменения и корректирует подписки: добавляет новые и деактивирует те, чьих облигаций в портфеле больше нет.

### Двухуровневый доступ к данным
Для работы с базой данных применяется осознанное разделение: декларативное описание схемы и миграции — через ORM, а все операции в реальном времени — через низкоуровневые асинхронные запросы. Это обеспечивает удобство эволюции схемы и максимальную производительность при обработке данных.

## Безопасность и надёжность

Безопасность финансовых данных — один из приоритетов системы. API-токены брокерских счетов и контактные данные пользователей хранятся в базе данных в зашифрованном виде — даже при компрометации базы злоумышленник не получит доступ к этим данным. Все пользовательские вводы — ссылки, идентификаторы, коды ценных бумаг — проходят строгую валидацию перед обработкой. Доступ к административным функциям ограничен белым списком.

Для надёжности работы реализовано несколько механизмов. Система дедупликации предотвращает повторную отправку одного и того же уведомления при любых сценариях — сбоях, перезапусках, параллельных обработках. Все фоновые процессы поддерживают корректное завершение работы без потери данных. Настроено структурированное логирование с раздельными журналами для каждого компонента, автоматической ротацией и компрессией — это позволяет быстро диагностировать проблемы в продакшене. Приём платежей реализован через официальный платёжный шлюз с автоматической фискализацией по 54-ФЗ, обеспечивая полное соответствие законодательству.

## Стек технологий

**Backend:** Python, aiogram 3 (FSM), asyncpg, SQLAlchemy, Alembic, httpx, Pydantic, loguru

**Парсинг:** BeautifulSoup4 + lxml, Selenium + Chromium, fake-useragent

**Финтех:** T-Bank Invest SDK (gRPC), YooKassa SDK (фискализация по 54-ФЗ), Fernet (шифрование)

**Инфраструктура:** Docker Compose, PostgreSQL (JSONB, partial indexes), uv

## Ключевые достижения

- **Реальная польза для инвестиций** — на основе своевременных уведомлений из системы удалось увеличить доходность личного облигационного портфеля на 10%, оперативно реагируя на рейтинговые события и корпоративные раскрытия
- **Экономия времени инвестора** — система заменяет ежедневный ручной мониторинг шести площадок, экономя несколько часов в неделю и исключая человеческий фактор — ни одно важное событие не будет пропущено
- **Своевременные инвестиционные решения** — мгновенные уведомления о критичных событиях (неисполнение обязательств, изменение рейтинга, появление новых бумаг) позволяют принимать решения в момент, когда это действительно имеет значение, а не постфактум
- **Глубокая интеграция с финансовой инфраструктурой РФ** — e-disclosure.ru, НРД, АКРА, НРА, Эксперт РА, isin.ru, T-Bank Invest API — полный охват ключевых источников для инвестора в облигации
- **Автоматизация управления портфелем** — привязка брокерского счёта, ночная синхронизация и автоматическое создание подписок избавляют от рутинной ручной настройки
- **Полноценная монетизация** — от приёма платежей с фискализацией по 54-ФЗ до автоматического управления жизненным циклом подписки
- **Расширяемая архитектура** — модульная система источников данных позволяет подключать новые площадки без изменения существующего кода
- **Безопасность финансовых данных** — шифрование токенов и персональных данных в базе, строгая валидация ввода, разграничение прав доступа
