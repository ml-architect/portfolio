## О проекте

Аналитики SOC регулярно решают задачи, которые хорошо ложатся на ML — классификация инцидентов, поиск аномалий, прогнозирование. Но в большинстве SOC-команд нет выделенных ML-инженеров.

MultiAutoML — корпоративная no-code платформа, позволяющая аналитикам SOC обучать модели через браузерный интерфейс. Загрузка табличного файла, выбор целевой переменной и типа задачи — система автоматически выполняет предобработку, подбор алгоритмов и гиперпараметров, а затем отдаёт результаты с интерактивными графиками и метриками. Использовалась в продакшене командой из 5-15 аналитиков.

## Задача

Дать аналитикам ИБ возможность самостоятельно проверять гипотезы на данных — без привлечения ML-инженера и без написания кода. Платформа должна поддерживать несколько типов задач, сравнение конфигураций моделей и легко развёртываться во внутренней инфраструктуре.

## Функционал

Три типа задач: бинарная классификация (F1, Accuracy, ROC-AUC, Precision, Recall, PR-AUC), мультиклассовая классификация (взвешенные метрики), регрессия (MAE, MSE, R^2, MAPE, SMAPE).

Гибкая работа с данными: загрузка одного файла с автосплитом или готовых train/test-файлов. Выбор целевой переменной из списка столбцов, исключение нерелевантных признаков, настройка таймаута обучения (10 сек - 2 часа).

Визуализация результатов: сводная таблица метрик, интерактивные графики confusion matrix, ROC-AUC и Precision-Recall кривых. Встроены три демо-датасета и пошаговое руководство.

## Архитектура

### Мультимодельная оркестрация

Вместо обучения одной модели система запускает несколько AutoML-моделей с разными конфигурациями и сравнивает результаты на единых данных. Оркестратор последовательно обучает каждую модель (с прогресс-баром), собирает предсказания и метрики, формирует сравнительную таблицу — аналитик выбирает лучшую модель на основе объективных показателей.

### Унифицированный интерфейс поверх AutoML

Абстрактный слой оборачивает LightAutoML (Сбер) в scikit-learn-подобный интерфейс: fit, predict, evaluate. Слой скрывает особенности API фреймворка — обрабатывает различные форматы выходных данных для бинарной и мультиклассовой классификации, управляет ролями признаков. Добавление нового AutoML-бэкенда сводится к реализации адаптера с тем же контрактом — без изменений в UI или оркестраторе.

### Контейнеризация

Docker Compose разворачивает весь стек одной командой на порту 80 — единое окружение без проблем совместимости зависимостей.

## Стек технологий

**ML:** Python, LightAutoML (Сбер), scikit-learn, NumPy

**Веб-интерфейс:** Streamlit, Plotly

**Данные:** pandas

**Инфраструктура:** Docker, Docker Compose

## Ключевые достижения

- **Демократизация ML в SOC** — цикл проверки гипотез сократился с дней до минут, аналитики обучают и оценивают модели самостоятельно
- **Продакшен-использование** — инструмент применялся командой из 5-15 аналитиков SOC для прикладных задач ИБ
- **Снижение зависимости от ML-экспертизы** — для типовых задач классификации и регрессии команда SOC перестала привлекать внешних ML-специалистов
- **Прозрачное сравнение моделей** — мультимодельная оркестрация исключает субъективный выбор: несколько конфигураций оцениваются на одних данных
- **Расширяемая архитектура** — новые ML-бэкенды подключаются через адаптер без изменения интерфейса и логики
- **Быстрое развёртывание** — контейнеризация + русскоязычный интерфейс со встроенной документацией, без дополнительного обучения
