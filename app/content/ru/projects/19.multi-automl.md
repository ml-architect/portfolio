## О проекте

В работе центра мониторинга информационной безопасности (SOC) аналитики регулярно сталкиваются с задачами, которые хорошо ложатся на машинное обучение — классификация, поиск закономерностей, прогнозирование. Однако в большинстве SOC-команд нет выделенных ML-инженеров: аналитики безопасности отлично разбираются в угрозах и работе с данными, но не занимаются написанием ML-пайплайнов на Python.

MultiAutoML — внутренняя корпоративная платформа, которая решает эту проблему. Сервис позволяет аналитикам SOC самостоятельно обучать модели машинного обучения на своих данных без единой строчки кода — через браузерный интерфейс. Достаточно загрузить табличный файл, указать целевую переменную, выбрать тип задачи и нажать кнопку. Система автоматически выполнит предобработку признаков, подбор алгоритмов и гиперпараметров, обучит модель и представит результаты с интерактивными графиками и таблицей метрик.

Платформа была развёрнута в продакшене и использовалась командой из 5–15 аналитиков SOC для решения прикладных задач информационной безопасности.

## Задача

Снизить порог входа в машинное обучение для аналитиков информационной безопасности. Вместо привлечения ML-инженера или освоения Python-экосистемы специалист SOC должен иметь возможность самостоятельно проверить гипотезу на данных — загрузить выборку, получить обученную модель с оценкой качества и принять решение о её применимости. При этом платформа должна поддерживать несколько типов задач, позволять сравнивать различные конфигурации моделей и легко развёртываться во внутренней инфраструктуре.

## Функционал

Платформа покрывает три основных типа задач машинного обучения. Бинарная классификация — для задач вида «да/нет» с полным набором метрик: F1, Accuracy, ROC-AUC, Precision, Recall, PR-AUC. Мультиклассовая классификация — для разделения данных по нескольким категориям с взвешенными метриками. Регрессия — для числовых прогнозов с метриками MAE, MSE, R², MAPE и SMAPE.

Работа с данными гибко настраивается: можно загрузить один файл с автоматическим разделением на обучающую и тестовую выборки, либо предоставить готовые train/test-файлы отдельно. Аналитик выбирает целевую переменную из списка столбцов, может исключить нерелевантные признаки и задать таймаут обучения — от 10 секунд до 2 часов в зависимости от объёма данных.

Результаты обучения представлены наглядно: сводная таблица метрик для сравнения моделей, интерактивные графики confusion matrix, ROC-AUC и Precision-Recall кривых с возможностью масштабирования и детального анализа. Для быстрого знакомства с платформой встроены три демонстрационных датасета — по одному на каждый тип задачи — и пошаговое руководство пользователя прямо в интерфейсе.

## Архитектура

### Мультимодельная оркестрация

Ключевое архитектурное решение — паттерн мультимодельного оркестратора. Вместо обучения одной модели система позволяет одновременно запускать несколько AutoML-моделей с разными конфигурациями и сравнивать их результаты на единых данных. Оркестратор последовательно обучает каждую модель с отображением прогресс-бара, собирает предсказания и метрики, после чего формирует сравнительную таблицу результатов. Такой подход позволяет аналитику не принимать решение «вслепую», а выбрать лучшую модель на основе объективных метрик.

### Унифицированный интерфейс поверх AutoML

Внутри платформы реализован абстрактный слой, который оборачивает фреймворк LightAutoML (разработка Сбера) в стандартный scikit-learn-подобный интерфейс — обучение, предсказание, оценка. Этот слой нивелирует особенности API фреймворка: автоматически обрабатывает различные форматы выходных данных для бинарной и мультиклассовой классификации, корректно управляет ролями признаков и целевых переменных. Архитектура спроектирована так, что добавление нового AutoML-бэкенда не требует изменений в интерфейсе или логике оркестратора — достаточно реализовать адаптер с тем же контрактом.

### Контейнеризация

Приложение полностью контейнеризировано через Docker и Docker Compose. Весь стек — от ML-фреймворка до веб-интерфейса — разворачивается одной командой на порту 80. Это обеспечивает одинаковое окружение для всех аналитиков и устраняет проблемы совместимости зависимостей, что критично для инструмента, используемого нетехнической командой.

**Веб-интерфейс:** Streamlit, Plotly

**Инфраструктура:** Docker, Docker Compose

## Ключевые достижения

- **Демократизация ML в SOC** — аналитики информационной безопасности получили возможность самостоятельно обучать и оценивать модели без привлечения ML-инженеров, что сократило цикл проверки гипотез с дней до минут
- **Повышение эффективности команды** — инструмент использовался командой из 5–15 аналитиков SOC в продакшене, позволяя каждому специалисту самостоятельно решать прикладные задачи ML
- **Снижение зависимости от ML-экспертизы** — для типовых задач классификации и регрессии команда SOC перестала зависеть от внешних ML-специалистов, высвободив их время для более сложных исследовательских задач
- **Прозрачное сравнение моделей** — мультимодельная оркестрация позволяет объективно оценить несколько конфигураций на одних данных, исключая субъективный выбор «лучшей» модели
- **Расширяемая архитектура** — модульный дизайн с абстрактным слоем поверх AutoML позволяет подключать новые ML-бэкенды без изменения интерфейса и логики приложения
- **Минимальный порог развёртывания** — полная контейнеризация и русскоязычный интерфейс со встроенной документацией обеспечили быстрое внедрение без дополнительного обучения пользователей
