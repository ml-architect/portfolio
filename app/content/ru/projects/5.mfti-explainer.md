## О проекте

Explainer — интеллектуальная RAG-система (Retrieval-Augmented Generation) для объяснения учебного контента школьникам 1–11 классов. Разработана по заказу МФТИ (Московский физико-технический институт).

Ключевая идея — создание «объяснительного AI», который не заменяет учителя, а помогает ученику разобраться в материале. Система настроена не решать задачи за ученика, а подводить к пониманию, используя адаптированный под возраст язык и строго следуя загруженным учебникам.

## Задача

Создать RAG-платформу, которая принимает вопросы учеников и генерирует понятные ответы, опираясь исключительно на проиндексированные учебные материалы. Критическое требование: архитектура должна позволять замену LLM-провайдера (Ollama → российский API) без изменения внешнего API.

## Архитектура

### RAG-пайплайн

Ingestion → Chunking → Embedding → PGVector → Retrieval → LLM Generation.
Мультиязычная модель эмбеддингов `intfloat/multilingual-e5-large-instruct` (1024-dim). IVFFlat индекс с cosine distance для приближённого поиска. Двухэтапный fallback: сначала по предмету + классу, затем только по предмету.

### Pluggable LLM Provider

Архитектура через n8n workflows позволяет заменить Ollama на любой LLM-провайдер без изменения HTTP API — критическое требование для production-размещения.

### Инфраструктура

7+ Docker-контейнеров: n8n + n8n-worker + PostgreSQL + PGVector + Ollama + Redis + web-panel. GitLab CI/CD с автоматическим деплоем. Healthcheck для каждого сервиса.

## Стек технологий

**AI/ML:** Ollama, intfloat/multilingual-e5-large-instruct (f16), PGVector, IVFFlat

**Оркестрация:** n8n 2.4.6, n8n-worker, Redis

**Frontend:** Vue 3, TypeScript, Vite, Tailwind CSS, Radix Vue, Nginx

**SDK:** Python 3.11+, Pydantic, Requests

**Данные:** PostgreSQL 15, PGVector (PostgreSQL 17)

**Инфраструктура:** Docker Compose, GitLab CI/CD, multi-stage Docker builds

## Ключевые достижения

- **Педагогический AI-промпт** — система адаптирует ответы под возраст ученика, отказывается решать задачи, не «выдумывает» ответы
- **Pluggable LLM** — замена провайдера без изменения API, что критично для развёртывания у заказчика
- **Полнофункциональный Python SDK** — типизированный клиент с dataclass-моделями, иерархией исключений и Session pooling
- **Vue 3 панель мониторинга** — real-time статус сервисов, статистика индекса, автообновление
- **Инкрементальная индексация** — дедупликация документов по хешу, поддержка PDF, DOCX, TXT, CSV, JSON, ePUB
