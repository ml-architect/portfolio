## О проекте

Explainer — интеллектуальная RAG-система (Retrieval-Augmented Generation) для объяснения учебного контента школьникам 1–11 классов, разработанная по заказу МФТИ (Московский физико-технический институт).

Сервис принимает вопросы учеников и генерирует развёрнутые, понятные ответы, опираясь исключительно на загруженные учебные материалы. Принципиальное отличие от обычных чат-ботов — педагогический подход: система не решает задачи за ученика, а подводит к пониманию, адаптируя язык ответа под возраст и уровень подготовки. При отсутствии нужной информации в базе знаний система честно сообщает об этом, не генерируя вымышленные ответы.

Помимо ядра с RAG-пайплайном, проект включает web-панель мониторинга для администраторов, а также документированную Python-библиотеку для интеграции сервиса в сторонние образовательные платформы.

## Задача

Создать RAG-платформу, которая принимает вопросы учеников и генерирует понятные ответы, основанные строго на проиндексированных учебных материалах. Критическое бизнес-требование — модульная архитектура с возможностью замены LLM-провайдера без изменения внешнего API, что необходимо для production-размещения на серверах заказчика с использованием российского языкового сервиса.

## Архитектура

### RAG-пайплайн

Процесс обработки запросов выстроен по классической RAG-схеме: загрузка документов, разбиение на фрагменты, генерация векторных представлений, сохранение в векторное хранилище, семантический поиск и генерация ответа языковой моделью.

При поиске релевантных фрагментов активно используется метаинформация документов — предмет, класс и тема. Это позволяет сузить область поиска и повысить точность ответов: система сначала ищет фрагменты, точно соответствующие предмету и классу ученика, и только если подходящих результатов недостаточно — автоматически расширяет область поиска. Каждый ответ сопровождается ссылками на источники с оценкой релевантности, что обеспечивает прозрачность и возможность проверки.

Система поддерживает историю диалога, позволяя ученику задавать уточняющие вопросы в рамках одной сессии, а также предоставляет статистику использования токенов для мониторинга расходов.

### Pluggable LLM Provider

Архитектура сервиса спроектирована с чётким разделением слоёв: оркестрация бизнес-логики отделена от конкретного LLM-провайдера. Это позволяет заменить языковую модель на любую другую — включая российские LLM-сервисы — без изменения HTTP API и без доработки клиентского кода. Для заказчика это означает бесшовный переход на production-инфраструктуру.

### Python SDK

Для удобства интеграции с внешними системами разработана полноценная Python-библиотека — типизированный клиент с подробной документацией. SDK покрывает все возможности сервиса: отправку вопросов, загрузку учебных материалов, управление индексом и мониторинг состояния. Библиотека включает типизированные модели данных, структурированную иерархию исключений, поддержку пулинга соединений и контекстного менеджера. Документация SDK включает quickstart-руководство, полное описание API, справочник моделей данных и перечень исключений.

### Инфраструктура

Сервис полностью контейнеризирован и состоит из 7+ Docker-контейнеров: оркестратор бизнес-логики, фоновый обработчик задач, реляционная база данных, векторное хранилище, LLM-сервис, очередь задач и web-панель мониторинга. Каждый контейнер оснащён healthcheck-ом и корректной цепочкой зависимостей. Настроен CI/CD-пайплайн с автоматическим деплоем на production-сервер.

## Ключевые достижения

- **Педагогический AI** — система адаптирует ответы под возраст ученика, принципиально отказывается решать задачи и не генерирует вымышленные ответы при отсутствии информации в базе знаний
- **Модульная архитектура** — замена LLM-провайдера без изменения API и клиентского кода, что обеспечило бесшовное развёртывание у заказчика
- **Умный контекстный поиск** — использование метаинформации (предмет, класс, тема) для уточнения семантического поиска с автоматическим расширением области при недостатке результатов
- **Python SDK** — документированная клиентская библиотека с типизацией, иерархией исключений и quickstart-руководством для быстрой интеграции
- **Vue 3 панель мониторинга** — real-time статус сервисов, визуализация статистики индекса, автоматическое обновление
- **Инкрементальная индексация** — дедупликация документов по хешу, поддержка множества форматов (PDF, DOCX, TXT, CSV, JSON, ePUB)
