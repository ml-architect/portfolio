## О проекте

Explainer — RAG-система для объяснения учебного контента школьникам 1–11 классов, разработанная по заказу МФТИ. Система не решает задачи за ученика, а подводит к пониманию, адаптируя язык под возраст. При отсутствии информации в базе знаний — честно сообщает об этом, не генерируя вымышленные ответы.

Помимо RAG-ядра, проект включает web-панель мониторинга и документированную Python-библиотеку для интеграции в сторонние образовательные платформы.

## Задача

Создать RAG-платформу с педагогическим подходом к генерации ответов на основе проиндексированных учебных материалов. Критическое требование — модульная архитектура с заменой LLM-провайдера без изменения внешнего API, что необходимо для production-размещения на серверах заказчика с российским языковым сервисом.

## Архитектура

### RAG-пайплайн

Классическая RAG-схема: загрузка документов → чанкинг → эмбеддинги → векторное хранилище → семантический поиск → генерация ответа LLM.

При поиске используется метаинформация документов — предмет, класс, тема. Система сначала ищет фрагменты, точно соответствующие предмету и классу ученика, и при недостатке результатов автоматически расширяет область поиска. Каждый ответ сопровождается ссылками на источники с оценкой релевантности.

Поддерживается история диалога для уточняющих вопросов и статистика использования токенов для мониторинга расходов.

### Pluggable LLM Provider

Бизнес-логика отделена от конкретного LLM-провайдера. Замена языковой модели — включая российские LLM-сервисы — не требует изменения HTTP API и доработки клиентского кода, что обеспечивает бесшовный переход на production-инфраструктуру заказчика.

### Python SDK

Типизированная Python-библиотека для интеграции с внешними системами. Покрывает все возможности сервиса: отправку вопросов, загрузку материалов, управление индексом, мониторинг. Включает модели данных, иерархию исключений, пулинг соединений, контекстный менеджер и документацию с quickstart-руководством.

### Инфраструктура

7+ Docker-контейнеров: оркестратор, фоновый обработчик задач, PostgreSQL, PGVector, LLM-сервис, Redis (очередь задач) и web-панель мониторинга. Каждый контейнер с healthcheck и цепочкой зависимостей. CI/CD-пайплайн с автоматическим деплоем на production.

## Ключевые достижения

- **Педагогический AI** — адаптация ответов под возраст, отказ от решения задач за ученика, отсутствие галлюцинаций благодаря strict-grounding на базе знаний
- **Pluggable LLM** — замена провайдера без изменения API, бесшовное развёртывание у заказчика
- **Контекстный поиск** — фильтрация по метаинформации (предмет, класс, тема) с автоматическим расширением области при недостатке результатов
- **Python SDK** — типизированная библиотека с документацией для быстрой интеграции
- **Vue 3 панель мониторинга** — real-time статус сервисов, визуализация статистики индекса
- **Инкрементальная индексация** — дедупликация по хешу, поддержка PDF, DOCX, TXT, CSV, JSON, ePUB
