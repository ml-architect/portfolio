## О проекте

Сервис интеллектуальной модерации текстовых сообщений, разработанный для образовательной платформы [Лаборатории интеллектуального анализа РАНХиГС](https://lia.ranepa.ru/). Встроен в платформу как интерактивная лабораторная работа для студентов, изучающих методы обработки естественного языка.

На торговых онлайн-площадках пользователи нередко пытаются обойти комиссию сервиса, обмениваясь контактными данными в замаскированном виде — заменяют буквы цифрами, вставляют пробелы внутри слов, записывают числа словами. Задача сервиса — научиться распознавать такие уловки и показать студентам, как разные подходы к NLP справляются с этой задачей. Приложение позволяет в реальном времени вводить произвольные сообщения и мгновенно видеть вердикт трёх различных алгоритмов — от простых правил до нейросетей. Тут же можно загрузить случайное сообщение из реального датасета и проверить, как модели работают на «живых» примерах.

Помимо практической части, сервис содержит полноценный образовательный блок: каждый алгоритм сопровождается теоретическим описанием с формулами и визуальными аналогиями, а в завершение студенты проходят квиз из 8 вопросов с порогом прохождения 7 из 8 — полный цикл «теория → практика → проверка знаний» в одном приложении.

## Задача

Разработать интерактивный образовательный инструмент, который наглядно демонстрирует эволюцию подходов к модерации контента: от ручных правил до нейросетевых моделей. Сервис должен решать реальную бизнес-задачу — обнаружение замаскированных контактных данных в пользовательских сообщениях — и при этом быть понятным для студентов, не имеющих глубокого опыта в машинном обучении.

## Архитектура

Сервис построен на чётком разделении ответственности: веб-интерфейс отвечает за взаимодействие с пользователем и образовательный контент, а вся ML-логика вынесена в отдельный модуль. При запуске приложение загружает обученные модели и данные, после чего готово к работе.

Ядро системы — три независимых классификатора, каждый из которых представляет свой уровень сложности анализа текста:

**Уровень 1 — Регулярные выражения.** Самый простой и быстрый подход, работающий по строгим правилам. Система ищет в тексте характерные паттерны: URL-адреса различных форматов (включая IPv4 и IPv6), email-адреса и телефонные номера в любой записи — с пробелами, дефисами, скобками. Этот метод отлично справляется с явно записанными контактами, но бессилен против маскировки.

**Уровень 2 — Частотный анализ текста с градиентным бустингом.** Текст проходит многоступенчатую предобработку: очистку от спецсимволов и пунктуации, нормализацию пробелов и лемматизацию (приведение слов к базовой форме). Затем подготовленный текст преобразуется в числовое представление методом TF-IDF — алгоритмом, который оценивает важность каждого слова относительно всего корпуса сообщений. Полученный вектор признаков подаётся на вход классификатору CatBoost, обученному различать сообщения с контактными данными и без них.

**Уровень 3 — Нейросетевой анализ с BERT.** Самый мощный подход, основанный на русскоязычной нейросети DeepPavlov/rubert-base-cased-conversational — модели, специально обученной на разговорных текстах. Сообщение токенизируется, дополняется до фиксированной длины и пропускается через нейросеть, которая формирует плотное числовое представление (эмбеддинг), отражающее семантику текста. Этот эмбеддинг передаётся тому же классификатору CatBoost. Ключевое решение — использование подхода feature extraction вместо полного дообучения нейросети: модель BERT используется только для извлечения признаков, что кратно снижает требования к вычислительным ресурсам без существенной потери качества.

Такая трёхуровневая архитектура позволяет студентам наглядно увидеть, как растёт качество распознавания по мере усложнения алгоритма, и понять, в каких случаях простые методы достаточны, а когда необходим нейросетевой подход.

### Деплой и CI/CD

Приложение развёрнуто на платформе Hugging Face Spaces и встроено в образовательную платформу ЛИА РАНХиГС. Обученные модели и датасеты хранятся в репозитории через Git LFS. Настроен автоматический CI/CD-пайплайн: при каждом обновлении кода GitHub Actions проверяет размер файлов и синхронизирует репозиторий с Hugging Face Spaces — новая версия приложения становится доступна студентам без ручного вмешательства.

## Ключевые достижения

- **Три уровня анализа на одной задаче** — от регулярных выражений через частотный анализ до нейросети, с наглядной демонстрацией сильных и слабых сторон каждого подхода
- **Русскоязычная NLP-модель** — выбрана и интегрирована специализированная BERT-модель для русского языка, обученная на разговорных данных, что оптимально для модерации пользовательских сообщений
- **Оптимизация ресурсов** — CatBoost поверх BERT-эмбеддингов вместо дообучения всей нейросети, что кратно снижает вычислительные требования
- **Полный образовательный цикл** — теория с формулами и визуальными аналогиями, интерактивная практика и квиз с порогом прохождения в одном приложении
- **Автоматический деплой** — CI/CD-пайплайн через GitHub Actions с автоматической публикацией на Hugging Face Spaces
