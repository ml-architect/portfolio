## О проекте

Интерактивная лабораторная работа для [Лаборатории интеллектуального анализа РАНХиГС](https://lia.ranepa.ru/), встроенная в образовательную платформу. Студенты изучают методы NLP на реальной задаче — обнаружение замаскированных контактных данных в сообщениях торговых площадок (замена букв цифрами, пробелы внутри слов, числа словами).

Приложение позволяет вводить произвольные сообщения и мгновенно видеть вердикт трёх алгоритмов — от regex до BERT. Можно загрузить случайное сообщение из реального датасета. Каждый алгоритм сопровождается теорией с формулами, а в финале — квиз из 8 вопросов (порог: 7/8). Полный цикл «теория → практика → проверка знаний».

## Задача

Разработать интерактивный инструмент, демонстрирующий эволюцию подходов к модерации контента — от ручных правил до нейросетей — на реальной бизнес-задаче, понятный студентам без глубокого опыта в ML.

## Архитектура

Веб-интерфейс отвечает за UX и образовательный контент, ML-логика вынесена в отдельный модуль. Ядро — три независимых классификатора нарастающей сложности:

**Уровень 1 — Regex.** Поиск паттернов URL (включая IPv4/IPv6), email и телефонов в любой записи (пробелы, дефисы, скобки). Надёжен для явных контактов, бессилен против маскировки.

**Уровень 2 — TF-IDF + CatBoost.** Предобработка текста (очистка, нормализация, лемматизация), затем TF-IDF-векторизация — оценка важности слов относительно корпуса. Полученный вектор подаётся на CatBoost-классификатор.

**Уровень 3 — BERT + CatBoost.** Русскоязычная модель DeepPavlov/rubert-base-cased-conversational (обучена на разговорных текстах). Сообщение токенизируется, пропускается через нейросеть, полученный эмбеддинг подаётся на CatBoost. Подход feature extraction вместо полного дообучения — кратно снижает вычислительные требования без существенной потери качества.

Трёхуровневая архитектура наглядно показывает рост качества распознавания по мере усложнения алгоритма.

### Деплой и CI/CD

Развёрнуто на Hugging Face Spaces и встроено в платформу ЛИА РАНХиГС. Модели и датасеты хранятся через Git LFS. CI/CD: GitHub Actions проверяет размер файлов и синхронизирует репозиторий с HF Spaces при каждом пуше.

## Ключевые достижения

- **Три уровня анализа** — Regex → TF-IDF + CatBoost → BERT + CatBoost с наглядным сравнением сильных и слабых сторон
- **Русскоязычная BERT** — DeepPavlov/rubert-base-cased-conversational, оптимальная для модерации разговорных сообщений
- **Feature extraction вместо fine-tuning** — CatBoost поверх BERT-эмбеддингов, кратное снижение вычислительных требований
- **Образовательный цикл** — теория с формулами, интерактивная практика, квиз (7/8) в одном приложении
- **CI/CD** — GitHub Actions → Hugging Face Spaces
