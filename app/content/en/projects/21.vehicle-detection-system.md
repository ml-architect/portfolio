## About the Project

A high-load production computer vision platform for vehicle detection and license plate recognition. Processes images from **6,000 cameras** in real time — from vehicle detection to license plate text extraction.

The platform includes an **inference server** on NVIDIA Triton with TensorRT optimization and an **asynchronous client library**. The ML models were trained by a separate developer — my area of responsibility: deployment, inference optimization, GPU cluster configuration, and client library development. Inference runs on a cluster of **6× NVIDIA RTX A6000**.

## Objective

Build a production-ready inference platform for processing images from thousands of cameras. The system must provide vehicle detection and license plate recognition with minimal latency and offer an asynchronous API for integration with external systems.

## Architecture

### Inference Server (Triton + TensorRT)

The system's core is **NVIDIA Triton Inference Server** with 6 models distributed across GPUs. Models are optimized to **TensorRT `.plan`** format. Each model is deployed with configured instance groups: vehicle and plate detection on GPUs 1–3, text recognition on GPU 0, segmentation on GPU 5. Dynamic batching ensures GPU utilization under variable load.

### License Plate Recognition Ensemble Pipeline

License plate recognition is implemented as a **Triton Ensemble** — a chain of three models: preprocessing (alignment to 32×100, PyTorch), feature extraction (512-dimensional embeddings, TensorRT), and attention-based text decoding (PyTorch). The ensemble executes as a single request to Triton.

### Computer Vision Models

- **Vehicle Detection** — YOLO (TensorRT), 4 classes: car, truck, bus, van
- **License Plate Detection** — WPOD-Net (TensorRT), perspective correction via affine transforms
- **Plate Text Recognition** — Ensemble of 3 modules with an attention decoder
- **Vehicle Segmentation** — Instance segmentation (TensorRT), masks + bounding boxes

### Asynchronous Client Library

A typed Python library based on **asynchronous gRPC streaming**. Architecture built on a generic base class — each model inherits a unified interface with preprocessing, post-processing, and inference, returning an AsyncIterator of results. All responses are strictly typed.

### Post-processing and Filtering

NMS for duplicate removal, confidence thresholding, license plate perspective correction via affine transforms, and IoU filtering via Shapely to select the best plate within a vehicle region.

### Web Interface for Testing

A Streamlit application for interactive testing: image upload, task selection, result visualization with bounding boxes, masks, and recognized text.

## Tech Stack

**ML Inference:** NVIDIA Triton Inference Server, TensorRT, PyTorch 1.13, CUDA 11.7, Triton Ensemble

**Computer Vision:** OpenCV, YOLO, WPOD-Net, Instance Segmentation, NMS

**Backend:** Python 3.10, asyncio, gRPC (tritonclient.grpc.aio), NumPy, Pandas, Shapely, Pillow

**Infrastructure:** Docker, Docker Compose, 6× NVIDIA RTX A6000, Poetry, GitLab CI/CD

**Testing:** Streamlit, pytest, pytest-asyncio, mypy (strict), Black

## Key Achievements

- **6,000 cameras in real time** — stable stream processing on a GPU cluster of 6× RTX A6000
- **<50 ms latency** — TensorRT optimization and instance group distribution across GPUs
- **Ensemble pipeline** — three-stage license plate recognition as a single Triton request
- **Multi-GPU inference** — 6 models distributed across 6 GPUs with dynamic batching
- **Async-first client** — gRPC streaming with typed results and a generic pattern
- **Dual plate detection approach** — WPOD-Net and segmenter with IoU filtering, best result selection via Shapely
- **Production-grade code** — mypy strict, pytest, Poetry, GitLab CI/CD
