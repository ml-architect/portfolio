## About the Project

Findata BOT is a production-ready Telegram bot for investors in the Russian bond market. It aggregates data from six sources — e-disclosure.ru, NSD, the credit rating agencies ACRA, NRA, and Expert RA, and isin.ru — and delivers personalized notifications filtered by the ISIN codes of bonds in the user's portfolio. Fully monetized: pricing plans, trial period, payment processing with fiscal compliance per 54-FZ.

## Objective

Transform the daily manual monitoring of six disparate platforms into automatic notification delivery. Integrate with a brokerage API for portfolio linking, ensure financial data security, and support the full business cycle — from registration to payment.

## Features

### Corporate Disclosure Monitoring
Subscription to issuers with filtering by ISIN codes and trigger phrases. Four types of reporting are tracked (charter, annual report, accounting statements, IFRS). Critical events — such as default on obligations — are delivered immediately.

### Rating Notifications
Monitoring of ACRA, NRA, Expert RA, and NSD. Two delivery modes: targeted — matching ISINs from the portfolio, and broadcast — for events important to all investors.

### New Bond Monitoring
Collection of fresh ISINs within a rolling 30-day window with broker availability checks. A notification is sent as soon as a bond becomes available for purchase.

### Brokerage Portfolio Linking
Integration with the T-Bank Invest API: encrypted token storage, nightly portfolio synchronization. Based on the bond holdings, subscriptions to issuers are automatically created, updated, and deactivated.

### Monetization
Subscription model: from a free tier with a company limit to a premium tier with full access. Trial period, expiration reminders, smooth downgrade without losing settings.

## Architecture

### Two-Process Model
The first process handles the Telegram interface (commands, menus, settings). The second is a set of scheduled background tasks: source monitoring, notification delivery, portfolio synchronization, payment processing. This separation ensures bot responsiveness regardless of background load.

### Extensible Data Source System
Each source is an independent module with a unified interface: data retrieval, parsing, normalization, delivery. Adding a new source comes down to implementing a module and registering it in the registry — scheduling, deduplication, and delivery are picked up automatically.

### Subscription Management
When a portfolio is linked, subscriptions are automatically created based on the bond holdings. On plan downgrade, they are soft-deactivated without deletion. Nightly synchronization detects portfolio changes and adjusts subscriptions accordingly.

### Two-Level Data Access
Schema and migrations are managed through SQLAlchemy ORM + Alembic. All runtime operations use asyncpg directly for maximum performance.

## Security and Reliability

- **Encryption** — API tokens and personal data are stored in the database in encrypted form (Fernet); if the database is compromised, the data remains protected
- **Validation** — strict checking of all user inputs (links, ISINs, identifiers); admin access via whitelist
- **Deduplication** — prevention of duplicate notifications during failures, restarts, and parallel processing
- **Graceful shutdown** — proper termination of background processes without data loss
- **Logging** — structured per-component logs with rotation and compression (loguru)
- **Payments** — YooKassa with automatic fiscal compliance per 54-FZ

## Tech Stack

**Backend:** Python, aiogram 3 (FSM), asyncpg, SQLAlchemy, Alembic, httpx, Pydantic, loguru

**Parsing:** BeautifulSoup4 + lxml, Selenium + Chromium, fake-useragent

**Fintech:** T-Bank Invest SDK (gRPC), YooKassa SDK (fiscal compliance per 54-FZ), Fernet (encryption)

**Infrastructure:** Docker Compose, PostgreSQL (JSONB, partial indexes), uv
