## About the Project

Findata BO is a production-ready fintech system in the form of a Telegram bot for investors in the Russian bond market. It aggregates data from 6+ official sources (e-disclosure.ru, NSD, ACRA, NRA, Expert RA, isin.ru) and delivers personalized notifications based on the user's bond ISIN codes.

## Objective

Solve a critical investor problem: timely tracking of dozens of issuer companies across disparate platforms. Automate monitoring by transforming manual work into automatic delivery of relevant notifications — from disclosure events to new bond listings on the exchange. With full monetization.

## Architecture

### Two-Process Model

**Bot Process** — Telegram interactions via aiogram 3, FSM for dialogs, 6 routers.

**Worker Process** — 6 parallel workers via `asyncio.gather()`:
- CompanyWorker (30s) — subscription monitoring
- BroadcastWorker (30s) — global ISIN-based notifications
- TbankIsinWorker — monitoring of new bond listings
- SubscriptionWorker (1h) — subscription management
- TbankSyncWorker (night window 01:00–05:00) — delta portfolio sync
- PaymentWorker (30s, 24/7) — YooKassa status checks

### Source Registry

Extensible architecture: `NewsSource` (ABC + Generic) with 5 abstract methods, `NewsEvent` dataclass, `SourceRegistry` singleton. Adding a new source requires implementing the ABC class and registering it.

### Two-Level Data Access

SQLAlchemy ORM — only for Alembic migrations. asyncpg raw queries — for all runtime operations (maximum performance).

## Tech Stack

**Backend:** Python 3.11, aiogram 3 (FSM), asyncpg, SQLAlchemy 2.0, Alembic (20 migrations), httpx, Pydantic, loguru

**Parsing:** BeautifulSoup4 + lxml, Selenium + Chromium, fake-useragent

**Fintech:** T-Bank Invest SDK (gRPC), YooKassa SDK (54-FZ), cryptography (Fernet)

**Infrastructure:** Docker Compose, PostgreSQL 15 (16 tables, JSONB, partial indexes), uv

## Key Achievements

- **Deep integration with Russian financial infrastructure** — e-disclosure, NSD, ACRA, NRA, Expert RA, T-Bank, isin.ru
- **gRPC integration with T-Bank Invest API** — delta portfolio sync, issuer identification via asset_uid/borrow_name
- **Fiscal compliance per 54-FZ** — full payment processing with electronic receipts via YooKassa
- **Event-driven deduplication** — event_key prevents duplicate notifications
- **Fernet encryption** — T-Bank API tokens and contact data are encrypted in the database
- **20 Alembic migrations** — evolution from a simple bot to a fintech platform
