## About the Project

The [RANEPA Intelligent Analysis Laboratory](https://lia.ranepa.ru/) offers data analysis courses where students learn A/B testing. Mastering it from lectures alone is difficult: you need to be able to formulate hypotheses, calculate sample sizes, interpret p-values, and avoid mistakes like early peeking at results.

A/B-test Lab is an interactive web laboratory that guides students through the full A/B testing cycle: from defining the objective to statistical analysis. It was used in the RANEPA curriculum and is deployed on Hugging Face Spaces.

## Features

The laboratory implements a six-step scenario with sequential access — each step unlocks only after the previous one is completed correctly.

**Steps 1–2: objective and metric.** The student selects the experiment's objective and a metric from the provided options. Incorrect answers are accompanied by explanations — the laboratory functions as a trainer, not a calculator.

**Step 3: hypothesis and parameters.** Choosing between a one-tailed and two-tailed test, configuring the significance level, duration, traffic volume, and sample size. At this stage, the laboratory also explains the peeking problem and the necessity of fixing the sample size before the experiment begins.

**Step 4: data generation.** The system creates a synthetic dataset: random assignment of users to groups with different conversion rates, timestamps distributed across the selected period. An interactive histogram shows the distribution of conversions.

**Steps 5–6: analysis and visualization.** Calculation of group conversion rates, absolute and relative difference, z-statistic, and p-value with a verdict on significance. The results are presented as a metrics table with color coding and a bar chart.

The key visualization is a chart showing the evolution of p-value over time. The system recalculates significance at each data interval, showing how the p-value fluctuates on small samples and stabilizes as observations accumulate. The threshold line demonstrates why stopping an experiment at a random "threshold crossing" leads to false conclusions.

## Architecture

### Two-Module Structure

The presentation layer manages the step-by-step scenario through a system of logical conditions. The computation layer encapsulates the statistical logic and visualizations, providing a clean interface for calculations.

### Statistical Engine

Z-test for proportions with the full computation chain: group conversion rates, standard error of the proportion, standard error of the difference, z-statistic, and p-value accounting for the hypothesis type. Visualization: Altair (bar charts), Plotly (distribution histograms and p-value evolution charts).

### CI/CD and Deployment

Hugging Face Spaces with the Streamlit SDK. GitHub Actions checks file sizes on every push and synchronizes the repository with HF Hub. Data is stored via Git LFS.

## Key Achievements

- **Educational tool for RANEPA** — full A/B testing cycle in practice, from hypothesis to result interpretation
- **Peeking problem demonstration** — the p-value evolution chart shows why an experiment should not be stopped at the first "significant" result
- **Trainer, not a calculator** — step-by-step scenario with answer validation at each stage of the methodology
- **CI/CD** — automated delivery of updates to Hugging Face Spaces via GitHub Actions
