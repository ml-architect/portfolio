## About the Project

The educational platform of the [RANEPA Intelligent Analysis Laboratory](https://lia.ranepa.ru/) offers data analysis courses where students learn methods for evaluating business hypotheses. A/B testing is one of the key tools in an analyst's arsenal, but mastering it from lectures alone is difficult: you need to understand how to formulate hypotheses, calculate sample sizes, interpret p-values, and avoid common mistakes like peeking at results before an experiment is complete.

I designed and developed A/B-test Lab — an interactive web laboratory that guides students through the full A/B testing cycle step by step: from defining the experiment's objective to performing statistical analysis of the results. The application was used in real coursework at RANEPA and is deployed on Hugging Face Spaces.

## Features

The laboratory implements a six-step scenario where each step unlocks only after the previous one is completed correctly — students cannot skip ahead to the results without understanding the methodology.

In the first two steps, the user defines the experiment's objective and selects a metric. The application presents multiple-choice options: the correct choice advances the student forward, while incorrect ones are accompanied by explanations of why that option is unsuitable for an A/B test. This approach turns the laboratory into an interactive trainer rather than just a calculator.

Next, the student formulates a statistical hypothesis — choosing between a one-tailed and two-tailed test — and configures the experiment parameters: significance level, test duration, daily traffic volume, and test sample size. At this stage, the laboratory clearly explains the early peeking problem and why the sample size must be fixed before the experiment begins.

After configuring the parameters, the system generates a synthetic dataset simulating a real A/B test: users are randomly assigned to groups with different conversion rates, and timestamps are uniformly distributed across the selected period. An interactive histogram shows the distribution of conversions between groups.

In the final step, the laboratory performs a full statistical analysis: it calculates the conversion rates of both groups, determines the absolute and relative difference, computes the z-statistic and p-value, and then renders a verdict on statistical significance. The results are presented as a metrics table with color coding and a bar chart comparing the groups.

The laboratory's key visualization is a chart showing the evolution of p-value over time. The system recalculates statistical significance at each data interval, showing how the p-value fluctuates with small samples and stabilizes as observations accumulate. A horizontal threshold line clearly demonstrates why stopping an experiment at the moment of a random "threshold crossing" leads to false conclusions — precisely the peeking problem that the laboratory warned about earlier.

## Architecture

### Two-Module Structure

The application is split into two layers: presentation and computation. The presentation module manages the step-by-step scenario through a system of logical conditions — each step controls access to the next one. The computation module encapsulates all statistical logic and visualizations, providing a clean interface for calculations.

### Statistical Engine

At the core of the laboratory lies the Z-test for proportions — the standard method for testing differences between two groups in A/B testing. The engine implements the full computation chain: group conversion rates, standard error of the proportion, standard error of the difference, z-statistic, and p-value accounting for the selected hypothesis type. Three libraries are used for visualization: Altair for comparative bar charts, Plotly for interactive distribution histograms and p-value evolution line charts.

### CI/CD and Deployment

The application is deployed on Hugging Face Spaces with the Streamlit SDK. An automated GitHub Actions pipeline checks file sizes for platform compatibility on every push and synchronizes the repository with Hugging Face Hub. Data is stored via Git LFS.

## Key Achievements

- **Educational tool for RANEPA** — an interactive laboratory enabling students to go through the full A/B testing cycle in practice, from hypothesis formulation to result interpretation
- **Visual peeking problem demonstration** — the p-value evolution chart over time shows why an experiment should not be stopped at the first "significant" result, building proper experimentation habits
- **Interactive trainer, not a calculator** — the step-by-step scenario with answer validation ensures meaningful engagement with each stage of the methodology
- **Automated deployment** — CI/CD via GitHub Actions ensures continuous delivery of updates to Hugging Face Spaces
