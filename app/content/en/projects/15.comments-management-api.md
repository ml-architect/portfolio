## About the Project

MIPT's internal system managed content through spreadsheet files — searching across hundreds of articles was slow, and collaborative work on comments was cumbersome. I designed a REST API with instant full-text search (RU/EN morphology), a unified interface for managing articles and comments, and import of accumulated data from Excel.

## Features

18 endpoints cover the full content lifecycle: CRUD for articles and comments, search by titles, content, and text fragments, and date filtering. Comments are linked to articles. A separate block handles author aggregation and their publications with pagination.

Bulk import from Excel is implemented: an uploaded spreadsheet is automatically distributed across both storage systems. All responses follow a unified format, with auto-documentation via Swagger UI.

## Architecture

Hybrid storage: **Elasticsearch** handles full-text search with morphological analyzers (RU/EN), while **PostgreSQL** manages relational links and structured data. On write, data is synchronously persisted to both stores, ensuring consistency.

The service follows 12-factor app principles: configuration via environment variables, graceful shutdown of connections, and the entire stack (API + Elasticsearch + PostgreSQL) is launched with a single command via Docker Compose with automatic database and index initialization.

The API layer is built on FastAPI with asynchronous processing, Pydantic v2 validation, and HTTP Basic authentication.

## Key Achievements

- **Search < 50 ms** — full-text search across 500+ articles with two-language morphology
- **Lossless migration from Excel** — bulk import of hundreds of articles with comments into structured storage
- **Single-command deployment** — containerization of three services with automatic database and search index initialization
