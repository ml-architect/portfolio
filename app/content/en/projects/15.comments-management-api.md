## About the Project

MIPT's internal information system needed fast and accurate search across a growing collection of articles and comments. The existing workflow based on spreadsheet files was no longer sufficient: finding the right material among hundreds of documents took too long, and collaborative work on comments was inconvenient.

I designed and developed a REST API service that solved these problems -- providing instant full-text search with Russian and English morphological analysis, convenient article and comment management through a unified API, and the ability to migrate accumulated data from Excel without information loss.

## Features

The service provides 18 endpoints covering the full content lifecycle. For articles, operations include creation, editing, deletion, as well as multiple search options -- by titles, content, and individual text fragments. Comments are linked to specific articles and support a similar set of operations with search by text and date.

A separate feature block is related to authors: the system automatically aggregates a list of all authors from the database and allows browsing a specific author's publications with pagination. For convenient migration, data import from Excel files is implemented -- you can upload a spreadsheet with articles and comments, and the service will automatically distribute the data across both storage systems.

All endpoints return responses in a unified format and are automatically documented via Swagger UI, simplifying integration for the frontend team.

## Architecture

At the core of the service lies a hybrid architecture with two data stores, each responsible for its own task. Elasticsearch provides full-text search with morphological analysis -- specialized analyzers for Russian and English languages are applied during text indexing, enabling article discovery by any word form. PostgreSQL stores structured data and provides relational links between articles, comments, and authors.

With each write operation, data is synchronously saved to both stores, ensuring consistency between the search index and the primary database. The service is built following 12-factor app principles: configuration is extracted into environment variables, database connections are properly closed on application shutdown, and the entire stack of three services (API, Elasticsearch, PostgreSQL) is deployed with a single command via Docker Compose.

The API layer is built on FastAPI with asynchronous request handling, input data validation through Pydantic models, and HTTP Basic authentication for protected operations.

## Key Achievements

- **Search response time < 50 ms** -- full-text search across 500+ articles with morphological analysis in two languages works virtually instantly thanks to Elasticsearch
- **Lossless data migration from Excel** -- bulk import of hundreds of articles with comments from spreadsheet files into structured storage
- **Full stack deployment with a single command** -- complete containerization of three services via Docker Compose with automatic database and search index initialization
