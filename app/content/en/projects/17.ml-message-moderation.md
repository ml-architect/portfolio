## About the Project

An intelligent text message moderation service developed for the educational platform of the [RANEPA Intelligent Analysis Laboratory](https://lia.ranepa.ru/). It is integrated into the platform as an interactive lab exercise for students studying natural language processing methods.

On online marketplaces, users often try to bypass service commissions by exchanging contact information in disguised forms -- replacing letters with digits, inserting spaces within words, writing numbers as words. The service's task is to learn to recognize such tricks and show students how different NLP approaches handle this challenge. The application allows users to enter arbitrary messages in real time and instantly see the verdict of three different algorithms -- from simple rules to neural networks. Users can also load a random message from a real dataset and check how the models perform on "live" examples.

Beyond the practical component, the service includes a full educational section: each algorithm is accompanied by a theoretical description with formulas and visual analogies, and at the end, students take a quiz of 8 questions with a passing threshold of 7 out of 8 -- a complete "theory -> practice -> knowledge check" cycle in one application.

## Objective

Develop an interactive educational tool that clearly demonstrates the evolution of content moderation approaches: from manual rules to neural network models. The service must solve a real business problem -- detecting disguised contact information in user messages -- while remaining accessible to students without deep experience in machine learning.

## Architecture

The service is built on a clear separation of concerns: the web interface handles user interaction and educational content, while all ML logic is extracted into a separate module. On startup, the application loads trained models and data, after which it is ready for use.

The core of the system consists of three independent classifiers, each representing its own level of text analysis complexity:

**Level 1 -- Regular Expressions.** The simplest and fastest approach, operating on strict rules. The system searches text for characteristic patterns: URLs in various formats (including IPv4 and IPv6), email addresses, and phone numbers in any notation -- with spaces, hyphens, parentheses. This method handles explicitly written contacts excellently but is powerless against disguise.

**Level 2 -- Frequency-Based Text Analysis with Gradient Boosting.** Text undergoes multi-stage preprocessing: removal of special characters and punctuation, whitespace normalization, and lemmatization (reducing words to their base form). The prepared text is then converted into a numerical representation using TF-IDF -- an algorithm that evaluates the importance of each word relative to the entire message corpus. The resulting feature vector is fed into a CatBoost classifier trained to distinguish messages with contact information from those without.

**Level 3 -- Neural Network Analysis with BERT.** The most powerful approach, based on the Russian-language neural network DeepPavlov/rubert-base-cased-conversational -- a model specifically trained on conversational texts. The message is tokenized, padded to a fixed length, and passed through the neural network, which produces a dense numerical representation (embedding) reflecting the text's semantics. This embedding is passed to the same CatBoost classifier. The key design decision is using the feature extraction approach instead of full fine-tuning: the BERT model is used only for feature extraction, which dramatically reduces computational resource requirements without significant quality loss.

This three-level architecture allows students to clearly see how recognition quality improves as the algorithm grows more complex, and to understand when simple methods are sufficient versus when a neural network approach is necessary.

### Deployment and CI/CD

The application is deployed on Hugging Face Spaces and integrated into the RANEPA Intelligent Analysis Laboratory educational platform. Trained models and datasets are stored in the repository via Git LFS. An automatic CI/CD pipeline is configured: with each code update, GitHub Actions checks file sizes and synchronizes the repository with Hugging Face Spaces -- a new version of the application becomes available to students without manual intervention.

## Tech Stack

**ML/AI:** PyTorch, Hugging Face Transformers, DeepPavlov/rubert-base-cased-conversational, CatBoost, scikit-learn, NLTK

**Web Interface:** Streamlit, Plotly

**Infrastructure:** GitHub Actions, Hugging Face Spaces, Git LFS

## Key Achievements

- **Three levels of analysis on a single task** -- from regular expressions through frequency analysis to neural networks, with a clear demonstration of each approach's strengths and weaknesses
- **Russian-language NLP model** -- a specialized BERT model for Russian, trained on conversational data, was selected and integrated, making it optimal for user message moderation
- **Resource optimization** -- CatBoost on top of BERT embeddings instead of fine-tuning the entire neural network, dramatically reducing computational requirements
- **Complete educational cycle** -- theory with formulas and visual analogies, interactive practice, and a quiz with a passing threshold in one application
- **Automated deployment** -- CI/CD pipeline via GitHub Actions with automatic publishing to Hugging Face Spaces
