## About the Project

Palatine Astro is an AI platform for personalized astrological consultations, developed for an astrology studio. The studio's clients message a Telegram bot, ask questions about their natal chart, partner compatibility, or upcoming events — and receive a detailed personalized response in real time, with a "typing" message effect.

Behind every response is a language model that does not simply generate text but operates as a full-fledged AI agent: it independently decides whether to build a natal or synastry chart, requests missing data from the user, and enriches the response with context from the studio's professional knowledge base. The platform targets a B2C audience with subscription-based monetization — plans by duration and message packages.

## Objective

The astrology studio conducted consultations manually: a client submitted a request, an astrologer built a chart in specialized software, interpreted it, and wrote a text response. The process took from 30 minutes to several hours per consultation, which limited the studio's throughput and created a backlog of waiting clients.

The goal was to create an AI platform that automates routine consultations while maintaining response quality at the level of a professional astrologer. The language model had to not just answer questions but make autonomous decisions — when to build a natal chart, when a synastry chart, and what context to pull from the knowledge base. At the same time, studio administrators needed to be able to update the knowledge base directly through Telegram without involving developers.

## Architecture

The platform is built on a microservice architecture with three key components, each running in separate Docker containers and responsible for its own area of concern.

### Telegram Bot — Interaction Interface

An asynchronous bot on aiogram 3 serves as the single entry point for users. When a client sends a question, the bot instantly displays a placeholder message and begins updating it as the language model's response arrives — at 200 ms intervals, creating a live conversation effect. To correctly handle streaming responses, a custom incremental JSON parser was developed, capable of extracting data from incomplete fragments and working with arbitrary LLM response formats.

The bot also includes an administrative panel: studio staff with the appropriate permissions can upload new materials to the knowledge base directly through Telegram — files are automatically processed, indexed, and become available to the model without restarting services.

### MCP Server — Astrological Toolkit for the LLM

The central component of the platform is an MCP server (Model Context Protocol), developed from scratch on top of the Kerykeion library. The server provides the language model with two tools: building a natal chart by date, time, and place of birth, and building a synastry compatibility chart for two people. Each tool performs the full cycle — from astronomical calculations (planet positions, aspects, houses) to generating a visual SVG diagram in Russian.

The key feature is that the language model itself decides when and which tool to invoke. Each tool's parameters are annotated with typed descriptions that the model "reads" and uses for decision-making. If a user asks about compatibility, the model automatically requests both partners' data and invokes the synastry chart builder. If the question is about career, it builds a natal chart and focuses the interpretation on the professional sphere. Transport is implemented via Server-Sent Events (SSE).

### Orchestration Layer — The Connecting Link

Between the bot and the language model operates an orchestration layer based on n8n, which serves as the "conductor" of the entire request processing chain. When a user's question arrives, the orchestrator performs a semantic search across the vector knowledge base (RAG on pgvector), finds relevant fragments from the studio's professional materials, and adds them to the LLM request context. This allows the model to provide answers grounded in the expertise of a specific astrological school rather than general knowledge.

Request processing is built on a distributed queue with a Redis broker and a dedicated worker process, ensuring reliability when multiple clients make simultaneous requests.

### Infrastructure

The entire platform is packaged into 6 Docker containers with automatic health checks, startup order management, and persistent data storage. The CI/CD pipeline on GitLab provides automatic deployment to dev and prod environments with selective triggering — rebuilds occur only when relevant files change.

## Tech Stack

**Backend:** Python 3.11, aiogram 3, aiohttp (streaming), FastMCP, Kerykeion, Pydantic

**AI/ML:** n8n (AI orchestration), pgvector (RAG, semantic search), MCP (Model Context Protocol)

**Data:** PostgreSQL, pgvector, Redis

**Infrastructure:** Docker Compose (6 services), GitLab CI/CD (dev/prod), uv (Astral)

## Key Achievements

- **Automation of 80% of routine consultations** — the platform took over routine questions (natal chart, compatibility, general forecasts), freeing up to 5 hours per day for the studio's astrologers to work with complex and VIP clients
- **35% increase in paid subscription conversion** — instant responses in Telegram and the live conversation effect significantly boosted engagement: clients who received their first free consultation were more likely to switch to a paid plan
- **24/7 consultation availability** — the bot processes requests around the clock without days off, which expanded the studio's audience reach to clients in other time zones and those who reach out outside business hours
- **Custom MCP server** — developed from scratch on top of the Kerykeion library, enabling the language model to autonomously build natal and synastry charts. A modern approach to AI agents that goes beyond simple prompt chains
- **RAG with dynamic updates** — semantic search across the studio's professional knowledge base provides accurate and contextually relevant answers without model fine-tuning. Administrators update the knowledge base through Telegram without involving developers
- **Production infrastructure** — 6 containers with automatic health checks, distributed task queue, dual-environment CI/CD with automatic deployment
