## About the Project

Palatine Astro is an AI platform for personalized astrological consultations. A client messages the Telegram bot, asks a question about their natal chart, compatibility, or forecast — and receives a detailed real-time response with a "typing" message effect.

Behind the response is an LLM operating as an AI agent: it independently decides which chart to build, requests missing data, and enriches the response with context from the studio's knowledge base. Monetization is subscription-based (plans by duration and message packages).

## Objective

The studio conducted consultations manually: an astrologer built a chart, interpreted it, and wrote a response — taking from 30 minutes to several hours per request. The goal was to automate routine consultations while maintaining quality at the level of a professional astrologer: the LLM had to autonomously select tools (natal/synastry chart, knowledge base context), and administrators had to be able to update the knowledge base via Telegram without involving developers.

## Architecture

A microservice platform consisting of three key components in separate Docker containers.

### Telegram Bot

An asynchronous bot on aiogram 3 — the single entry point. Upon receiving a question, it displays a placeholder and updates it every 200 ms as the LLM response streams in. A custom incremental JSON parser was developed for streaming output, extracting data from incomplete fragments of arbitrary format.

The built-in admin panel allows staff to upload materials to the knowledge base directly through Telegram — files are automatically indexed and available to the model without restarting.

### MCP Server

Developed from scratch on top of Kerykeion. It provides the LLM with two tools via Server-Sent Events: building a natal chart (date, time, place of birth) and a synastry chart (two people). Each tool performs the full cycle — from astronomical calculations (planets, aspects, houses) to generating an SVG diagram in Russian.

The model itself decides when and which tool to invoke, guided by typed parameter descriptions: a compatibility question triggers a synastry chart, a career question triggers a natal chart focused on the professional sphere.

### Orchestration Layer (n8n)

When a question arrives, the orchestrator performs a semantic search across the vector knowledge base (RAG on pgvector) and adds relevant fragments to the LLM request context. This produces answers grounded in the expertise of a specific astrological school. Request processing runs through a distributed queue with a Redis broker and a dedicated worker process.

### Infrastructure

6 Docker containers with health checks, startup order management, and persistent volumes. CI/CD on GitLab with selective triggering — rebuilds only when relevant files change, automatic deployment to dev/prod.

## Tech Stack

**Backend:** Python 3.11, aiogram 3, aiohttp (streaming), FastMCP, Kerykeion, Pydantic

**AI/ML:** n8n (AI orchestration), pgvector (RAG, semantic search), MCP (Model Context Protocol)

**Data:** PostgreSQL, pgvector, Redis

**Infrastructure:** Docker Compose (6 services), GitLab CI/CD (dev/prod), uv (Astral)

## Key Achievements

- **Automation of 80% of routine consultations** — routine questions (natal chart, compatibility, forecasts) are handled without human involvement, freeing up to 5 hours per day for astrologers to focus on complex and VIP clients
- **35% increase in paid subscription conversion** — instant responses with a live conversation effect boosted engagement after the first free consultation
- **24/7 availability** — round-the-clock operation expanded reach to clients in other time zones
- **Custom MCP server** — the LLM autonomously builds natal and synastry charts, going beyond simple prompt chains
- **RAG with dynamic updates** — accurate answers based on the studio's expertise without model fine-tuning; administrators update the knowledge base via Telegram
- **Production infrastructure** — 6 containers with health checks, distributed task queue, dual-environment CI/CD
