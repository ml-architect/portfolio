## About the Project

Explainer is an intelligent RAG system (Retrieval-Augmented Generation) for explaining educational content to students in grades 1 through 11, developed under contract for MIPT (Moscow Institute of Physics and Technology).

The service accepts student questions and generates detailed, comprehensible answers based exclusively on uploaded educational materials. What fundamentally sets it apart from conventional chatbots is its pedagogical approach: the system does not solve problems for the student but guides them toward understanding, adapting the language to the student's age and level. When the knowledge base lacks the required information, the system honestly reports this rather than generating fabricated answers.

In addition to the core RAG pipeline, the project includes a web-based monitoring dashboard for administrators and a well-documented Python library for integrating the service into third-party educational platforms.

## Objective

Build a RAG platform that accepts student questions and generates clear answers based strictly on indexed educational materials. The critical business requirement was a modular architecture with the ability to swap the LLM provider without changing the external API — essential for production deployment on the client's servers using a Russian language service.

## Architecture

### RAG Pipeline

The query processing flow follows a classic RAG scheme: document ingestion, chunking, vector embedding generation, storage in a vector database, semantic search, and answer generation by a language model.

During relevant fragment retrieval, document metadata — subject, grade, and topic — is actively used to narrow the search scope and improve answer accuracy. The system first searches for fragments that precisely match the student's subject and grade, and only if there are insufficient results does it automatically broaden the search. Each answer is accompanied by source references with relevance scores, ensuring transparency and verifiability.

The system supports conversation history, allowing students to ask follow-up questions within a single session, and provides token usage statistics for cost monitoring.

### Pluggable LLM Provider

The service architecture is designed with clear layer separation: business logic orchestration is decoupled from the specific LLM provider. This allows swapping the language model for any other — including Russian LLM services — without changing the HTTP API or modifying client code. For the client, this means a seamless transition to production infrastructure.

### Python SDK

A full-featured Python library was developed for convenient integration with external systems — a typed client with comprehensive documentation. The SDK covers all service capabilities: sending questions, uploading educational materials, managing the index, and monitoring service health. The library includes typed data models, a structured exception hierarchy, connection pooling support, and a context manager. SDK documentation includes a quickstart guide, full API reference, data model reference, and exception catalog.

### Infrastructure

The service is fully containerized, consisting of 7+ Docker containers: a business logic orchestrator, a background task worker, a relational database, a vector store, an LLM service, a task queue, and a monitoring web panel. Each container is equipped with healthchecks and proper dependency chains. A CI/CD pipeline with automatic deployment to the production server is configured.

## Key Achievements

- **Pedagogical AI** — the system adapts responses to the student's age, fundamentally refuses to solve problems, and does not generate fabricated answers when information is missing from the knowledge base
- **Modular architecture** — LLM provider replacement without changing the API or client code, enabling seamless deployment at the client's site
- **Smart contextual search** — leveraging document metadata (subject, grade, topic) to refine semantic search with automatic scope expansion when results are insufficient
- **Python SDK** — documented client library with type safety, exception hierarchy, and a quickstart guide for rapid integration
- **Vue 3 monitoring dashboard** — real-time service status, index statistics visualization, automatic refresh
- **Incremental indexing** — document deduplication by hash, support for multiple formats (PDF, DOCX, TXT, CSV, JSON, ePUB)
