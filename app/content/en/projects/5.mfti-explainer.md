## About the Project

Explainer is an intelligent RAG system (Retrieval-Augmented Generation) for explaining educational content to students in grades 1 through 11. Developed under contract for MIPT (Moscow Institute of Physics and Technology).

The key idea is creating an "explanatory AI" that does not replace the teacher but helps the student understand the material. The system is configured not to solve problems for the student but to guide them toward understanding, using age-appropriate language and strictly following uploaded textbooks.

## Objective

Build a RAG platform that accepts student questions and generates clear answers based exclusively on indexed educational materials. Critical requirement: the architecture must allow LLM provider replacement (Ollama → Russian API) without changing the external API.

## Architecture

### RAG Pipeline

Ingestion → Chunking → Embedding → PGVector → Retrieval → LLM Generation.
Multilingual embedding model `intfloat/multilingual-e5-large-instruct` (1024-dim). IVFFlat index with cosine distance for approximate search. Two-stage fallback: first by subject + grade, then by subject only.

### Pluggable LLM Provider

Architecture via n8n workflows allows replacing Ollama with any LLM provider without changing the HTTP API — a critical requirement for production deployment.

### Infrastructure

7+ Docker containers: n8n + n8n-worker + PostgreSQL + PGVector + Ollama + Redis + web-panel. GitLab CI/CD with automatic deployment. Healthcheck for each service.

## Tech Stack

**AI/ML:** Ollama, intfloat/multilingual-e5-large-instruct (f16), PGVector, IVFFlat

**Orchestration:** n8n 2.4.6, n8n-worker, Redis

**Frontend:** Vue 3, TypeScript, Vite, Tailwind CSS, Radix Vue, Nginx

**SDK:** Python 3.11+, Pydantic, Requests

**Data:** PostgreSQL 15, PGVector (PostgreSQL 17)

**Infrastructure:** Docker Compose, GitLab CI/CD, multi-stage Docker builds

## Key Achievements

- **Pedagogical AI prompt** — the system adapts responses to the student's age, refuses to solve problems, and does not "invent" answers
- **Pluggable LLM** — provider replacement without API changes, critical for client-side deployment
- **Full-featured Python SDK** — typed client with dataclass models, exception hierarchy, and session pooling
- **Vue 3 monitoring dashboard** — real-time service status, index statistics, auto-refresh
- **Incremental indexing** — document deduplication by hash, support for PDF, DOCX, TXT, CSV, JSON, ePUB
