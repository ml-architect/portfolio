## About the Project

Explainer is a RAG system for explaining educational content to schoolchildren in grades 1–11, developed for MIPT. The system does not solve problems for the student but rather guides them toward understanding, adapting the language to their age. When the knowledge base lacks the required information, it honestly reports this rather than generating fabricated answers.

In addition to the RAG core, the project includes a web-based monitoring dashboard and a documented Python library for integration into third-party educational platforms.

## Objective

Build a RAG platform with a pedagogical approach to answer generation based on indexed educational materials. The critical requirement was a modular architecture with LLM provider swapping without changing the external API, which was necessary for production deployment on the client's servers using a Russian language service.

## Architecture

### RAG Pipeline

Classic RAG scheme: document ingestion → chunking → embeddings → vector store → semantic search → LLM answer generation.

During search, document metadata is used — subject, grade, topic. The system first looks for fragments that precisely match the student's subject and grade, and when results are insufficient, it automatically broadens the search scope. Each answer is accompanied by source references with relevance scores.

Conversation history is supported for follow-up questions, along with token usage statistics for cost monitoring.

### Pluggable LLM Provider

Business logic is decoupled from the specific LLM provider. Swapping the language model — including Russian LLM services — does not require changes to the HTTP API or modifications to client code, ensuring a seamless transition to the client's production infrastructure.

### Python SDK

A typed Python library for integration with external systems. It covers all service capabilities: sending questions, uploading materials, managing the index, and monitoring. Includes data models, an exception hierarchy, connection pooling, a context manager, and documentation with a quickstart guide.

### Infrastructure

7+ Docker containers: orchestrator, background task worker, PostgreSQL, PGVector, LLM service, Redis (task queue), and a web-based monitoring dashboard. Each container has healthchecks and dependency chains. A CI/CD pipeline with automatic deployment to production is configured.

## Key Achievements

- **Pedagogical AI** — response adaptation to the student's age, refusal to solve problems for the student, no hallucinations thanks to strict grounding on the knowledge base
- **Pluggable LLM** — provider swap without API changes, seamless deployment at the client's site
- **Contextual search** — filtering by metadata (subject, grade, topic) with automatic scope expansion when results are insufficient
- **Python SDK** — typed library with documentation for rapid integration
- **Vue 3 monitoring dashboard** — real-time service status, index statistics visualization
- **Incremental indexing** — deduplication by hash, support for PDF, DOCX, TXT, CSV, JSON, ePUB
