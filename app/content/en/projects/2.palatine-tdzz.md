## About the Project

Contract Risk Analyzer is a production-ready AI system for automated risk analysis in international maritime grain shipping contracts. Developed for **TD Zerno Zavolzhya** — one of the largest grain traders in Russia.

Traders process dozens of grain purchase and shipping contracts daily across international routes. Each contract contains dozens of legal formulations affecting financial risks: payment terms, demurrage, laytime, cargo quality, freight, insurance, and arbitration clauses (GAFTA/FOSFA). Manual analysis took hours and was prone to human error.

The system automates the process: a user uploads a DOCX contract via a Telegram bot, the AI analyzes the text against 111 risk and mandatory phrases considering context (product, country, Incoterms basis, port) and produces a structured report with risk levels, recommendations, and contract quotes.

## Objective

Build an AI system capable of replacing manual contract analysis by lawyers and traders. Execute the full ML cycle: from research and model benchmarking through training a custom LLM to production deployment. Ensure analysis accuracy via two-stage verification and enable transition from commercial GPT-4o API to a self-hosted model for client data confidentiality.

## Architecture

### Contract Risk Analyze API

The system's core — a REST API on FastAPI with a two-stage LLM pipeline. The first LLM (GPT-4o) analyzes the contract against each of the 111 risk phrases; the second LLM verifies the response and can overturn the decision (**LLM-as-a-Judge** pattern). Each phrase is checked context-dependently — considering the product (wheat, sunflower oil), destination country, delivery basis (CIF, CFR, FOB, CIFFO), and shipping port.

Contract processing is asynchronous via BackgroundTasks with status polling and progress tracking. Results include: phrase presence, chain-of-thought, recommendations, contract quote, and verification confirmation.

### Contract Risk Analyze Bot

Telegram bot on aiogram 3 — interface for TD Zerno Zavolzhya employees. DOCX contract upload, task status polling at 2-second intervals, Excel report generation (openpyxl) with all analysis results.

### Research & Model Training

Full R&D cycle:
- **Benchmarks** — comparison of GPT-4o, Llama 3.3, Qwen2.5 32B, Quepasa SaaS RAG, LightRAG with balanced_accuracy metrics on 4 real contracts
- **RAG experiments** — Graph-based RAG (LightRAG) with multilingual embeddings (multilingual-e5-large-instruct, 1024-dim) and hybrid search
- **Data generation** — pipeline on Distilabel: 59 contracts x 111 phrases = 5,883 training examples
- **Fine-tuning** — series of Qwen2.5 models (0.5B–32B) with QLoRA via Unsloth, RoPE Scaling for 18,000-token context, published on HuggingFace Hub in GGUF format

## Tech Stack

**Backend:** Python 3.12, FastAPI, LangChain (chains, JsonOutputParser, @chain), Pydantic v2, aiogram 3, aiohttp, openpyxl

**ML/AI:** OpenAI GPT-4o, Unsloth (2x fine-tuning speedup), QLoRA (4bit NF4, rank 32–256), TRL (SFTTrainer), Transformers, Qwen2.5, Distilabel, LightRAG, Ollama, SetFit, PyTorch + CUDA 12.1

**Data:** python-docx, markitdown, Pandas, scikit-learn (balanced_accuracy_score), HuggingFace Hub, Git LFS

**Infrastructure:** Docker Compose, GitLab CI/CD, Makefile, pre-commit hooks (Ruff), HTTP Basic Auth

## Key Achievements

- **Full ML cycle** — from R&D and benchmarking 6+ models through data generation and fine-tuning to production deployment. Systematic approach to architecture selection backed by metrics
- **LLM-as-a-Judge** — two-stage verification where the second model reviews and can overturn the first model's decision, significantly improving analysis accuracy
- **Domain-specific NLP** — 111 risk phrases with 5 risk levels, 200+ industry terms (GAFTA, FOSFA, Incoterms), context-dependent checking across 4 parameters
- **Fine-tuning to replace GPT-4o** — custom model based on Qwen2.5 32B with QLoRA, ensuring client data confidentiality and inference cost control
- **Training data generation** — Distilabel pipeline for creating 5,883 examples from real contracts with automated labeling
- **Model on HuggingFace** — trained model published in GGUF format for self-hosted deployment via Ollama
