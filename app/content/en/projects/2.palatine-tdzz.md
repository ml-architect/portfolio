## About the Project

Production-ready AI system for risk analysis in international maritime grain shipping contracts. Developed for **TD Zerno Zavolzhya** — one of Russia's leading grain exporters.

The company's lawyers and traders process dozens of contracts daily, each containing formulations with a direct impact on financial risks: demurrage, laytime, freight, insurance, and arbitration clauses. Manual analysis of a single contract took 2–3 hours and was prone to errors — a missed formulation could cost hundreds of thousands of dollars.

The system automates the process: an employee uploads a contract via a Telegram bot, the AI model checks the text against 111 risk phrases considering the deal context (product type, destination country, Incoterms basis, shipping port) and generates an Excel report with risk levels, recommendations, and quotes from the contract.

## Objective

Execute the full ML cycle (R&D → benchmark → fine-tuning → production) to replace manual contract analysis. Ensure high accuracy through two-stage LLM verification and lay the groundwork for transitioning from a commercial API to a self-hosted model to protect the client's data confidentiality.

## Architecture

Microservice architecture consisting of three components in a shared Docker network: the analytics API, a Telegram bot, and a research module.

### Analytics API

Asynchronous REST API with a two-stage LLM pipeline (**LLM-as-a-Judge**). In the first stage, the primary model analyzes the contract against each of the 111 phrases, generating chain-of-thought reasoning, recommendations, and quotes. In the second stage, an independent verifier model receives the original contract and the initial analysis, confirming or overturning the decision. This significantly reduces both false positives and false negatives.

Each phrase is checked context-dependently — considering the product type, destination country, delivery basis, and shipping port. For example, a restriction on vessel types is a risk only under a specific delivery basis and specific ports.

The result for each phrase includes: presence status, risk level, chain of thought, recommendation, quote, and verification result.

### Telegram Bot

Interface for company employees: contract upload, real-time progress tracking, and receiving an Excel report for sharing between departments.

### Research & Model Training

**Benchmarking.** Systematic comparison of commercial LLMs, open-source models of various scales, and RAG solutions (SaaS platforms, custom graph RAG with multilingual embeddings and hybrid search). Evaluation by balanced accuracy on real contracts with expert annotations. The two-stage LLM pipeline demonstrated the best balance of accuracy and reliability.

**Training data generation.** 59 real contracts x 111 phrases = ~6,000 training examples. Each example included a system instruction, phrase description, industry glossary, contract text, and a structured JSON response.

**Fine-tuning.** A series of experiments with open-source LLMs from <1B to 32B parameters. QLoRA (4-bit NF4, rank 32–256) enabled training large models on accessible GPU hardware. Training was performed exclusively on the response portion, with context window scaling up to 18,000 tokens. The final model achieved quality comparable to the commercial API while providing full control over data.

## Tech Stack

**Backend:** Python 3.12, FastAPI, LangChain (chains, JsonOutputParser, @chain), Pydantic v2, aiogram 3, aiohttp, openpyxl

**ML/AI:** OpenAI GPT-4o, Unsloth (2x fine-tuning speedup), QLoRA (4bit NF4, rank 32–256), TRL (SFTTrainer), Transformers, Qwen2.5, Distilabel, LightRAG, Ollama, SetFit, PyTorch + CUDA 12.1

**Data:** python-docx, markitdown, Pandas, scikit-learn (balanced_accuracy_score), HuggingFace Hub, Git LFS

**Infrastructure:** Docker Compose, GitLab CI/CD, Makefile, pre-commit hooks (Ruff), HTTP Basic Auth

## Key Achievements

- **12–15x speedup** — from 2–3 hours to 10–15 minutes per contract; department throughput increased from 3–5 to 20–30 contracts per day
- **Complete coverage** — all 111 phrases are checked without exception; two-stage verification (LLM-as-a-Judge) reduced missed risks and false positives
- **Standardization** — a unified Excel report format for legal, trading, and logistics departments eliminated discrepancies and accelerated deal approvals
- **Full ML cycle** — R&D → benchmark of 6 approaches → data generation → fine-tuning → production. Every decision is backed by metrics on real contracts
- **Domain-specific NLP** — 111 risk phrases (5 risk levels), 200+ industry terms, context-dependent checking across 4 deal parameters
- **Self-hosted LLM** — fine-tuning a custom model ensured confidentiality of contract data and predictable inference costs
