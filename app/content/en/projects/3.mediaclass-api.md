## About the Project

Mediaclass API is the backend of the commercial desktop application "MEDIACLASS AI" for automatic speech recognition and speaker diarization in audio and video files. A product for enterprise clients — educational institutions, media companies, and organizations that need to transcribe meeting and lecture recordings with speaker identification.

## Objective

Develop an API that operates as a local server with an Electron frontend and provides transparent switching between local GPU inference and the Palatine Cloud service. The solution must be delivered in a protected form with hardware-bound licensing.

## Architecture

### Hybrid ML Providers (Local + Cloud)

The key architectural feature is a Provider pattern based on Protocol interfaces. A unified transcription pipeline transparently works with both local models (Whisper + Pyannote on the user's GPU) and the Palatine Cloud API. Switching requires a single request parameter.

### ML Diarization Pipeline

Production-grade diarization: Pyannote.audio 4.0 with pretrained segmentation models, speaker embeddings, and VBx clustering with PLDA. Models are shipped with the application.

### Fault Tolerance

- Cloud Token Manager with a 3-token fallback chain
- HTTP retry with exponential backoff
- Auto-validation and recovery of corrupted directories from backup
- CPU fallback when GPU is unavailable

## Tech Stack

**Backend:** Python 3.10/3.11, FastAPI, Uvicorn, Pydantic v2, ORJSONResponse

**ML/AI:** OpenAI Whisper (whisper-timestamped), Pyannote.audio 4.0, PyTorch 2.8, torchaudio, scikit-learn, SciPy

**Media:** FFmpeg, Pydub, MoviePy, SoundFile

**Infrastructure:** Docker, Conda, PyArmor (obfuscation), Loguru

**Licensing:** WMI COM / PowerShell (Windows), machine-id (Linux), IOPlatformUUID (macOS)

## Key Achievements

- **Hybrid ML architecture** — Protocol-based providers for seamless switching between local and cloud inference
- **Industrial diarization pipeline** — Pyannote.audio 4.0 with VBx clustering and PLDA
- **Commercial protection** — PyArmor encryption + hardware hash licensing (cross-platform)
- **E2E tests without GPU** — stub classes for torch, numpy, moviepy enable testing without ML dependencies
- **Asynchronous processing** — thread-safe TaskStore with real-time progress tracking
