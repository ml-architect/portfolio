## About the Project

Mediaclass API is the backend of the desktop application "MEDIACLASS AI," developed for [DIGIS](https://mediaclass.digis.ru/) as part of the work at the AI/ML integrator Palatine. The product transcribes audio and video recordings with speaker separation. The primary audience is schools and universities that need transcription of lectures, classes, and meetings with identification of each speaker.

The service is deployed on the client's computer and operates as a local server alongside the Electron application — data never leaves the organization's perimeter. Clients without a powerful GPU can transparently switch to cloud processing with a single request parameter.

## Objective

Design an autonomous backend service for on-premise deployment with high-quality speech recognition, speaker diarization, hybrid switching between local GPU inference and cloud processing, and a custom licensing system with hardware binding.

## Architecture

### Hybrid ML Providers

A unified transcription pipeline operates in two modes via contract-based interfaces: local GPU inference and cloud processing through the Palatine Cloud API. Adding a new provider does not affect the existing code.

### Recognition and Diarization

The service handles all major audio and video formats (MP3, WAV, OGG, FLAC, MP4, MOV, AVI, etc.), automatically extracting the audio track from video. Transcription produces text with timestamps at the segment level. Diarization models (segmentation, voiceprints, clustering) are shipped with the application and work offline. The result is a JSON where each text fragment is linked to a specific speaker.

### Licensing System

Cross-platform hardware binding (CPU, RAM, GPU) via a unique hardware hash with server-side verification. Windows, Linux, and macOS are supported, each with its own identification mechanism. Backward compatibility is maintained when the hardware information collection method changes.

### Result Management

Results are stored in structured directories with pagination and sorting. Before editing, a backup is automatically created (media file + transcription). Recording trimming is supported with removal of an arbitrary time segment, splicing, and re-recognition. In case of data corruption, the system restores it from a backup.

### Fault Tolerance

Automatic GPU detection at startup with fallback to CPU, cloud token rotation with automatic failover on errors, retry with exponential backoff, self-recovery of corrupted directories from backups.

## Tech Stack

**Backend:** Python 3.10/3.11, FastAPI, Uvicorn, Pydantic v2

**ML/AI:** PyTorch, torchaudio, scikit-learn, SciPy

**Media:** FFmpeg, Pydub, MoviePy, SoundFile

**Infrastructure:** Docker, Conda, PyArmor (code encryption), Loguru

## Key Achievements

- **Fully autonomous service** — on-premise deployment with no internet dependency when a GPU is available, data privacy guaranteed
- **Hybrid ML architecture** — seamless switching between local inference and cloud processing via contract-based providers
- **Cross-platform licensing** — hardware binding with server-side verification and backward compatibility
- **Commercial code protection** — source code encryption for secure delivery to end clients
- **Self-healing service** — automatic GPU detection, CPU fallback, token rotation, and recovery from backups without administrator intervention
