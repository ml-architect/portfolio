## About the Project

Mediaclass API is the backend of the commercial desktop application "MEDIACLASS AI," developed for [DIGIS](https://mediaclass.digis.ru/) as part of the work at the AI/ML integrator Palatine. The product is designed for automatic speech recognition and speaker diarization in audio and video recordings. The primary audience is educational institutions — schools and universities that need to transcribe recordings of lectures, classes, and meetings with identification of each speaker.

The key architectural feature is that the service is fully deployed on the client's computer and operates as a local server alongside the desktop Electron application. This allows processing recordings without sending data to the internet, which is critical for educational organizations with privacy requirements. At the same time, clients without a powerful GPU can transparently switch to cloud processing — with a single request parameter.

## Objective

Design and develop a backend service that operates autonomously on the client's hardware, delivers high-quality speech recognition with speaker diarization, and supports flexible switching between local GPU inference and cloud processing. The solution must be delivered in a protected form with a custom licensing system bound to the end user's hardware.

## Architecture

### Hybrid ML Providers

The central architectural decision is a provider system that abstracts the speech processing method. A unified transcription pipeline transparently operates in two modes: local inference on the user's GPU and cloud mode via the Palatine Cloud API. Providers are implemented on contract-based interfaces — adding a new processing method does not affect the existing code. For clients with powerful GPUs, this means fully autonomous operation without internet dependency, while for others it provides the ability to use cloud resources without any changes to the application interface.

### Recognition and Diarization

The service handles all major audio and video formats (MP3, WAV, OGG, FLAC, MP4, MOV, AVI, etc.), automatically extracting the audio track from video files. Transcription produces text with precise timestamps at the individual segment level. Diarization is production-grade: segmentation models, voiceprint generation, and speaker clustering models are shipped with the application and work entirely offline. The result is a structured JSON where each text fragment is linked to a specific speaker.

### Licensing System

A custom licensing system is implemented with hardware binding to the client's computer. Upon license key activation, the service collects information about the CPU, RAM, and GPU, generates a unique hardware hash, and verifies it through a dedicated licensing server. The system works cross-platform — Windows, Linux, and macOS are supported, each with its own hardware identification mechanism. Backward compatibility is maintained with previously activated licenses when the hardware information collection method changes.

### Result Management and Editing

All recognition results are stored in structured directories with pagination and sorting support. Before any editing, a backup is automatically created — both the media file and the transcription. Users can trim a recording by removing an arbitrary time segment with automatic splicing of the remaining parts, after which recognition is re-run. When corrupted data is detected, the system automatically restores it from the backup.

### Fault Tolerance

The service is designed for autonomous operation without administration: automatic GPU detection at startup with fallback to CPU when unavailable, cloud token rotation with automatic failover on errors, retry strategies with exponential backoff for network requests, and self-recovery of corrupted directories from backups.

## Tech Stack

**Backend:** Python 3.10/3.11, FastAPI, Uvicorn, Pydantic v2

**ML/AI:** PyTorch, torchaudio, scikit-learn, SciPy

**Media:** FFmpeg, Pydub, MoviePy, SoundFile

**Infrastructure:** Docker, Conda, PyArmor (code encryption), Loguru

## Key Achievements

- **Fully autonomous service** — the backend is deployed on the client host and operates without internet dependency when a GPU is available, ensuring data privacy for educational institutions
- **Hybrid ML architecture** — contract-based providers enable seamless switching between local GPU inference and cloud processing with a single request parameter
- **Custom licensing system** — cross-platform hardware binding with server-side verification and backward compatibility across updates
- **Commercial code protection** — source code encryption for secure product delivery to end clients
- **Self-healing service** — automatic GPU detection, CPU fallback, cloud token rotation, and data recovery from backups without administrator intervention
