## About the Project

As part of the educational platform of the [RANEPA Intelligent Analysis Laboratory](https://lia.ranepa.ru/), I designed Uplift Lab — an interactive web laboratory built around the "Analyst vs ML" concept. The student manually segments a marketing campaign audience through a filter system and then compares their result with ML models. Delta metrics clearly demonstrate the superiority of the algorithmic approach over manual analytics.

The data is the MineThatData E-Mail Analytics benchmark (Kevin Hillstrom): 42,000+ customer records split into control and treatment groups.

## Features

The laboratory begins with EDA: interactive charts show customer distribution by purchase channel, price segment, order recency, and other features.

The student then segments the audience through six filters: purchase channel, customer type, product category, price segment, geography, and order recency. In essence, they solve the same task as a marketer in a real company. Validation at each step prevents errors.

After applying the filters, manual segmentation metrics are calculated (uplift@k, Qini AUC, weighted average uplift) along with the same metrics for ML models. The result is delta metrics showing the absolute difference between approaches. On Qini and Uplift curve charts, a red marker pins the manual analytics position against the ML curves.

## Architecture

### ML Pipeline

12 model combinations were trained: three algorithms (CatBoost, XGBoost, Random Forest) across four uplift modeling approaches via scikit-uplift. The approaches differ in their strategy for handling control and treatment groups — from a single model on combined data to a pair of dependent models with DDR. Best result: CatBoost + SoloModel — uplift@30% = 0.0776.

CatBoost and XGBoost training used GPU acceleration. Reproducibility is ensured through fixed random seeds and saved split indices.

### Instant Response Without GPU

Models are trained offline, and predictions are saved to CSV. On startup, the application loads pre-computed results — instant interface responsiveness without GPU dependency on the server.

### CI/CD

The application is deployed on Hugging Face Spaces (Streamlit SDK). GitHub Actions on each push checks file sizes and synchronizes the repository with Hugging Face Hub. Binary artifacts are stored via Git LFS.
