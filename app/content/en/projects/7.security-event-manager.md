## About the Project

Security Event Manager (SEM) is a full-featured platform for SOC teams (Security Operations Center). It automatically detects, classifies, and escalates security incidents. It analyzes SIEM events, applies ML models for attack detection, matches against Sigma rules, and creates alerts in TheHive.

## Objective

Build a unified interface for SOC teams that automates routine work: analyzing thousands of SIEM events, matching them against detection rules, classifying attacks, and escalating to TheHive. With an AI assistant running locally without sending data outside the organization's perimeter.

## Architecture

### Two-Container System

**Web Container** — FastAPI + Vue 3 SPA. 17 routers, async SQLite for configuration, integration with PostgreSQL SIEM.

**Back Container** — autonomous processor. CatBoost ML models, Sigma Rules Engine, cyclic SIEM event processing (30–40 min).

**Ollama** — local AI server (Qwen3, DeepSeek-R1) for the chat assistant.

### Sigma Rules Engine

Built from scratch — recursive descent parser supporting AND/OR/NOT, wildcards, quantifiers ("1 of", "all of"), modifiers (contains, cidr, re). 11 built-in tests.

### ML Detection

Two-level CatBoost system: the first model determines whether an attack occurred; the second classifies the attack type. Applied in every processing cycle.

### Rule Analytics

Automatic identification of problematic rules: dead (0 matches), noisy (> avg * 3), declining (drop > 50%).

## Tech Stack

**Backend:** Python 3.11, FastAPI 0.109, aiosqlite, psycopg2, Alembic, HMAC-SHA256, bcrypt

**Frontend:** Vue 3.4, TypeScript 5.3, Vite, Pinia, Tailwind CSS 3.4, Radix Vue, shadcn/vue

**ML/AI:** CatBoost 1.2.7, Ollama (Qwen3, DeepSeek-R1), pySigma

**Infrastructure:** Docker (multi-stage), Docker Compose, GitLab CI/CD

**Integrations:** TheHive, SIEM PostgreSQL

## Key Achievements

- **Sigma Rules Engine from scratch** — full-featured recursive descent parser covered by 11 tests
- **ML attack detection** — two-level CatBoost for real-time detection and classification
- **Rule effectiveness analytics** — automatic identification of dead/noisy/declining rules to maintain rule base quality
- **Local AI** — data never leaves the organization's perimeter, Git2Markdown converter for loading code into context
- **Architecture evolution** — migration from Streamlit to FastAPI + Vue 3 while preserving backward data compatibility
- **RBAC** — granular permissions by section, HMAC-SHA256 (timing-attack safe), bcrypt
