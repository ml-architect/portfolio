## About the Project

An internal corporate tool — a web platform for automated ML model training without writing code. Built on LightAutoML (developed by Sber) and wrapped in an interactive Streamlit interface. Enables data analysts and business users to upload tabular data, select a task type, configure training parameters, and receive a trained model with metric visualizations — all through the browser.

## Objective

Lower the barrier to machine learning for non-ML-engineer employees. Instead of writing Python code and configuring pipelines, users upload a CSV, specify the target variable, and get a trained model with metrics at the click of a button.

## Architecture

### Multi-Model Orchestrator Pattern

The `MultiAutoML` class accepts a list of `BaseModel` instances, sequentially trains each with a progress bar, and collects predictions and metrics. Allows comparing multiple configurations and scaling the system by adding new AutoML backends.

### BaseModel Abstraction

A unified interface (fit, predict, predict_proba, evaluate) on top of LightAutoML, hiding API quirks: no clean `fit` method, probability output for binary classification, OHE format for multiclass.

### Three Task Types

**Binary Classification** — F1, Accuracy, ROC-AUC, Precision, Recall, PR-AUC. Visualization: confusion matrix, ROC-AUC curve, PR curve.

**Multiclass Classification** — F1, Accuracy, Precision, Recall (weighted). Validation: minimum 5 records per class.

**Regression** — MAE, MSE, R2, MAPE, custom SMAPE.

## Tech Stack

**ML:** LightAutoML 0.3.7.3 (Sber), scikit-learn 1.1.3, NumPy 1.23.5

**Web Interface:** Streamlit 1.15.1, Plotly 5.11.0

**Data:** pandas 1.4.3

**Infrastructure:** Docker, Docker Compose (port 80)

## Key Achievements

- **Abstraction over LightAutoML** — `BaseModel` provides a scikit-learn-like interface, smoothing out framework API quirks
- **Extensible multi-model architecture** — `MultiAutoML` allows adding new AutoML backends without changing UI code
- **Russian-language interface** — all UI elements, errors, and documentation in Russian for corporate use
- **Built-in demo data** — 3 downloadable datasets (Titanic, Body Fat, synthetic multiclass) for quick onboarding
- **Custom SMAPE metric** — implementation of Symmetric Mean Absolute Percentage Error, absent in scikit-learn
- **Single-command deployment** — `docker-compose up -d` launches a ready-to-use application
