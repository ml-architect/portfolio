## About the Project

SOC analysts regularly tackle tasks that map well to ML — incident classification, anomaly detection, forecasting. But most SOC teams have no dedicated ML engineers.

MultiAutoML is a corporate no-code platform that enables SOC analysts to train models through a browser interface. Upload a tabular file, select the target variable and task type — the system automatically performs preprocessing, algorithm and hyperparameter selection, then delivers results with interactive charts and metrics. It was used in production by a team of 5–15 analysts.

## Objective

Give information security analysts the ability to independently validate hypotheses on data — without involving an ML engineer and without writing code. The platform must support multiple task types, model configuration comparison, and easy deployment within internal infrastructure.

## Features

Three task types: binary classification (F1, Accuracy, ROC-AUC, Precision, Recall, PR-AUC), multiclass classification (weighted metrics), regression (MAE, MSE, R^2, MAPE, SMAPE).

Flexible data handling: upload a single file with automatic splitting or pre-split train/test files. Select the target variable from a list of columns, exclude irrelevant features, configure training timeout (10 sec – 2 hours).

Result visualization: summary metrics table, interactive confusion matrix charts, ROC-AUC and Precision-Recall curves. Three built-in demo datasets and a step-by-step guide are included.

## Architecture

### Multi-Model Orchestration

Instead of training a single model, the system launches multiple AutoML models with different configurations and compares results on the same data. The orchestrator sequentially trains each model (with a progress bar), collects predictions and metrics, and generates a comparative table — the analyst selects the best model based on objective indicators.

### Unified Interface on Top of AutoML

An abstraction layer wraps LightAutoML (Sber) into a scikit-learn-like interface: fit, predict, evaluate. The layer hides framework API specifics — handles different output formats for binary and multiclass classification, manages feature roles. Adding a new AutoML backend comes down to implementing an adapter with the same contract — without changes to the UI or orchestrator.

### Containerization

Docker Compose deploys the entire stack with a single command on port 80 — a unified environment without dependency compatibility issues.

## Tech Stack

**ML:** Python, LightAutoML (Sber), scikit-learn, NumPy

**Web Interface:** Streamlit, Plotly

**Data:** pandas

**Infrastructure:** Docker, Docker Compose

## Key Achievements

- **Democratizing ML in SOC** — the hypothesis validation cycle was reduced from days to minutes, analysts train and evaluate models independently
- **Production use** — the tool was used by a team of 5–15 SOC analysts for applied information security tasks
- **Reduced dependency on ML expertise** — for standard classification and regression tasks, the SOC team stopped engaging external ML specialists
- **Transparent model comparison** — multi-model orchestration eliminates subjective selection: multiple configurations are evaluated on the same data
- **Extensible architecture** — new ML backends are connected via an adapter without changing the interface or logic
- **Rapid deployment** — containerization plus a Russian-language interface with built-in documentation, no additional training required
