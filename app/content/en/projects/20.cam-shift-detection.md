## About the Project

A REST service for automatic detection of camera angle shifts. It accepts two images — a reference and a check image — and determines whether the camera position has changed along the X-axis (pan), Y-axis (tilt), and scale (zoom).

It addresses the challenge of monitoring fixed surveillance cameras: it detects unauthorized repositioning or rotation of a camera and reports the shift direction.

## Objective

Develop an API service for reliable detection of camera angle changes with false positive filtering from moving objects (people, vehicles). The service must operate autonomously, support GPU acceleration, and be ready for containerized deployment.

## Architecture

### Processing Pipeline

Input images undergo preprocessing: byte stream to NumPy array conversion, resizing to 1497x843, grayscale conversion, and Gaussian blur smoothing (5x5 kernel) for noise suppression.

### Shift Detection via Optical Flow

The system's core is the **Farneback algorithm** for computing dense optical flow between a pair of images. Shift direction is classified based on mean displacements with threshold filtering.

### Moving Object Masking

To eliminate false positives, masking is implemented via **YOLOv8x-seg**. The model detects people, cars, motorcycles, buses, and trucks, replacing mask regions with the median color. Optical flow is computed only from static scene elements.

### REST API

FastAPI with three endpoints: health check (`GET /health`), shift detection (`POST /detect_shift`), and Swagger UI (`GET /doc`). The endpoint accepts two images via multipart/form-data and an optional flag to exclude moving objects.

## Tech Stack

**Backend:** Python 3.10, FastAPI 0.100, Uvicorn, Pydantic

**Computer Vision:** OpenCV 4.8 (Farneback optical flow, Gaussian blur), Pillow

**ML:** PyTorch 2.0.1 (CUDA 11.7), Ultralytics YOLOv8x-seg

**Infrastructure:** Docker, Docker Compose, NVIDIA Container Runtime

## Key Achievements

- **Monitoring automation** — replaces manual camera position checks
- **~85% false positive reduction** — moving object masking via YOLOv8x-seg
- **Sub-second processing** — suitable for periodic real-time monitoring
- **Minimal shift detection** — threshold filtering detects even minor camera displacements
- **Easy integration** — REST API with Swagger UI, plug-in without infrastructure changes
