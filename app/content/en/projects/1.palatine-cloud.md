## About the Project

Palatine Cloud is a commercial cloud platform that I designed and built from scratch. The platform provides businesses with API services for the full speech processing cycle: from automatic speech recognition (ASR) and speaker diarization to sentiment analysis, intelligent summarization via local LLMs, and text-to-speech (TTS).

A key differentiator is that all ML models, including large language models, run locally on dedicated GPU servers. This ensures client data privacy, predictable costs, and full control over inference quality. The platform processes **10 million audio minutes daily**, supports **100+ languages** with automatic detection, accepts **23+ audio/video formats**, and maintains **99.982% uptime** across 4 geographically distributed Tier III data centers.

The project is fully monetized with a pricing system, payment processing, and automatic fiscal compliance — generating stable revenue.

## Objective

Build a commercial cloud platform from the ground up, capable of processing large volumes of speech data in real time. The platform must scale under load, ensure high reliability and recognition quality, provide an OpenAI-compatible API for easy integration, include local LLMs for summarization and analytics, and support the full business cycle — from user registration to payment and fiscal compliance.

## Architecture

The platform is built on a **microservices architecture of 10 independent services** deployed across multiple dedicated GPU servers. Services communicate via HTTP and gRPC, unified by a single API Gateway and a full monitoring stack.

### Speech Recognition and Diarization Service
The platform's core — a two-tier architecture: an orchestration layer (authentication, billing, request routing) and an ML inference layer. Models are served via **NVIDIA Triton Inference Server** with gRPC API, sequence batching, and stateful inference, while GPU resource orchestration is handled by **Ray Serve** with automatic replica scaling based on current load.

Streaming transcription works in real time: the client sends an audio stream over WebSocket, the service decodes and resamples the signal, forwards it to Triton via gRPC, and returns results as they are recognized. Supports 15+ audio codecs (PCM, Opus, FLAC, MP3, AAC, etc.) with sample rates from 8 to 48 kHz. Achieved **92.9% recognition accuracy (WER 7.1%)** while processing files up to 10+ hours with ~25 seconds latency for 30-minute recordings.

### Local LLMs — Summarization and Analytics
The platform runs **locally deployed large language models** for intelligent processing of recognized text. Summarization works with a library of configurable prompts and supports a thinking mode for complex analytical tasks. An OpenAI-compatible chat completions endpoint is implemented — clients can use the standard OpenAI SDK to work with local models. All data stays on the platform's servers, which is critical for enterprise clients.

### API Gateway
A single entry point for all client requests. Routing is described declaratively and loaded dynamically at startup. Supports multiple authentication types, proxying to all internal services, and composite requests — chains of calls with intermediate result substitution. Fronted by a reverse proxy with TLS termination and automatic certificate provisioning.

### Payment Gateway
Built on **Clean Architecture** principles with clear layer separation: entities, business logic, interfaces, and adapters. Supports multi-merchant payment routing based on user plans, webhook notification handling with authenticity verification, automatic fiscal receipt generation per Russian Federal Law 54-FZ, promo codes, and idempotent event processing.

### Self-hosted Database and Authentication
A full Supabase stack deployed with 12+ containers: authentication, real-time subscriptions, file storage, serverless functions, REST API, and connection pooling. Custom SQL migrations with row-level security (RLS) policies.

### Monitoring and Observability
A complete observability stack with multi-node architecture: dashboards and metric visualization, metric collection from all servers (including Ray Serve and GPU metrics), centralized log storage from all Docker containers, and collection agents on remote hosts.

### Text-to-Speech (TTS)
A containerized neural speech synthesis service with GPU inference, voice cloning from reference audio, and an OpenAI-compatible API.

### Dashboard, Admin Panel, and Documentation
User dashboard backend with rate limiting, an admin panel for managing plans and promo codes, and **public API documentation** auto-generated from service OpenAPI specs and deployed on the Mintlify platform with an interactive API Playground.

## Tech Stack

**Backend:** Python 3.10–3.11, FastAPI, Uvicorn, Pydantic, httpx, WebSocket

**ML/AI:** NVIDIA Triton Inference Server (gRPC, sequence batching), Ray Serve (autoscaling), CUDA 12.6, Fish Speech (TTS), local LLMs (Ollama), CTC Decoder

**Data:** PostgreSQL 15, Supabase (self-hosted), Redis, Alembic

**Infrastructure:** Docker / Docker Compose, GitLab CI/CD (lint → test → deploy), Traefik (TLS, HSTS), Grafana, Prometheus, Loki, Alloy

**Integrations:** YooKassa (54-FZ), Supabase Auth (JWT), OpenAI-compatible API, Mintlify (auto-generated docs)

- **High accuracy at scale** — achieved 92.9% recognition accuracy (WER 7.1%) while processing 10M audio minutes per day with 99.982% uptime

- **Real-time streaming transcription** — live speech recognition via WebSocket → Triton gRPC supporting 15+ codecs, 100+ languages, and files up to 10+ hours
- **OpenAI-compatible API** — drop-in replacement for transcription and chat completions, fully compatible with OpenAI SDK for easy client migration
- **Clean Architecture in billing** — full layer separation with multi-merchant routing, fiscal compliance (54-FZ), and idempotent payment processing
- **Auto-generated documentation** — public API docs on Mintlify with automatic generation from OpenAPI specs and an interactive Playground
- **Full monitoring stack** — Grafana + Prometheus + Loki with multi-node metric/log collection and GPU inference monitoring
- **CI/CD with coverage** — GitLab CI pipelines with linting, unit/integration/E2E tests, and coverage reports for all services
