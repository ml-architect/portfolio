## About the Project

Palatine Cloud is a commercial cloud platform designed and built by me from scratch. It provides businesses with an API for the full speech processing cycle: ASR, diarization, sentiment analysis, summarization via local LLMs, and text-to-speech (TTS).

All ML models, including LLMs, run locally on dedicated GPU servers — this ensures data privacy, predictable costs, and full control over inference quality. The platform processes **10M audio minutes/day**, supports **100+ languages** with automatic detection, accepts **23+ audio/video formats**, and maintains **99.982% uptime** across 4 geographically distributed Tier III data centers.

The project is fully monetized: pricing plans, payment processing, automatic fiscal compliance — the platform generates stable revenue.

## Objective

Build a commercial cloud platform for processing large volumes of speech data in real time: scalable, reliable, with high recognition quality, an OpenAI-compatible API, local LLMs, and the full business cycle from registration to payment and fiscal compliance.

## Architecture

The platform is built on a **microservices architecture of 10 services** across multiple dedicated GPU servers. Communication is via HTTP and gRPC, with a single API Gateway and a full monitoring stack.

### Speech Recognition and Diarization
Two-tier architecture: orchestration (authentication, billing, routing) and ML inference. Models are served via **NVIDIA Triton Inference Server** (gRPC, sequence batching, stateful inference), and GPU resource orchestration is handled by **Ray Serve** with automatic replica scaling.

Streaming transcription: the client sends an audio stream over WebSocket, the service decodes and resamples the signal, forwards it to Triton via gRPC, and returns results as they are recognized. Supports 15+ codecs (PCM, Opus, FLAC, MP3, AAC, etc.), sample rates from 8 to 48 kHz. **Accuracy 92.9% (WER 7.1%)**, files up to 10+ hours, ~25 s latency for a 30-minute recording.

### Local LLMs — Summarization and Analytics
**Locally deployed LLMs** for processing recognized text. A library of configurable prompts, thinking mode for complex tasks. OpenAI-compatible chat completions endpoint — clients use the standard OpenAI SDK. All data stays on the platform's servers.

### API Gateway
Single entry point: declarative routing with dynamic loading, multiple authentication types, proxying to internal services, composite requests — call chains with intermediate result substitution. Fronted by a reverse proxy with TLS termination and automatic certificates.

### Payment Gateway
Built on **Clean Architecture**: entities, business logic, interfaces, adapters. Multi-merchant routing based on pricing plans, webhook handling with authenticity verification, fiscal receipts per 54-FZ, promo codes, idempotent event processing.

### Self-hosted Supabase
A full stack of 12+ containers: authentication, real-time subscriptions, file storage, serverless functions, REST API, connection pooling. Custom SQL migrations with RLS policies.

### Monitoring and Observability
Multi-node architecture: dashboards and visualization, metric collection from all servers (including Ray Serve and GPU), centralized logs from all Docker containers, collection agents on remote hosts.

### Text-to-Speech (TTS)
Containerized neural TTS with GPU inference, voice cloning from reference audio, and an OpenAI-compatible API.

### User Dashboard, Admin Panel, and Documentation
User dashboard backend with rate limiting, admin panel for managing plans and promo codes, **public API documentation** on Mintlify — auto-generated from OpenAPI specs with an interactive Playground.

## Tech Stack

**Backend:** Python 3.10–3.11, FastAPI, Uvicorn, Pydantic, httpx, WebSocket

**ML/AI:** NVIDIA Triton Inference Server (gRPC, sequence batching), Ray Serve (autoscaling), CUDA 12.6, Fish Speech (TTS), local LLMs (Ollama), CTC Decoder

**Data:** PostgreSQL 15, Supabase (self-hosted), Redis, Alembic

**Infrastructure:** Docker / Docker Compose, GitLab CI/CD (lint → test → deploy), Traefik (TLS, HSTS), Grafana, Prometheus, Loki, Alloy

**Integrations:** YooKassa (54-FZ), Supabase Auth (JWT), OpenAI-compatible API, Mintlify (auto-generated docs)
