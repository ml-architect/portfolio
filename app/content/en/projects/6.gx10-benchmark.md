## About the Project

A service for benchmarking LLM inference on **NVIDIA DGX Spark** — a compact device based on the Grace Blackwell architecture (128 GB unified memory). The results became the basis for a [Habr article](https://habr.com/ru/articles/994058/), where NVIDIA's marketing claims were verified through objective measurements.

## Objective

Create a reproducible tool: automatically run models through realistic scenarios, collect metrics, and generate interactive charts for publication.

## How the Service Works

### Multi-Dimensional Testing

A full sweep of combinations: model * dataset * concurrency level (1--100 simultaneous requests). This provides a complete picture — from responsiveness on single requests to the throughput saturation point.

### Three Load Scenarios

- **Short chat** — interactive dialogue, key metric — TTFT
- **Medium-context RAG** — knowledge base search, context of several thousand tokens
- **Long-context RAG** — document analysis up to 73,000 tokens, a memory stress test

### Metrics

Four metrics broken down by percentiles (mean, median, P99):

- **TTFT** — time to first token
- **TPOT** — time per output token
- **Throughput** — throughput capacity (tok/s)
- **E2E Latency** — total request processing time

### Quantization Comparison

Models were tested in BF16, FP8, AWQ 4-bit, and GPTQ-Int4 formats. The key finding — MoE architecture in GPTQ-Int4: the quality of a 30B model at the computational cost of a 3B one.

## Automated Visualization

A Plotly-based module generates seven types of charts for each model: throughput vs. concurrency, TTFT/TPOT analysis, latency percentile distribution, bar charts, and a metrics matrix. For model comparison — the same charts with panel breakdowns. Export to HTML and PNG — used directly in the article.

## Infrastructure

A containerized pipeline: vLLM servers and the benchmark orchestrator are launched via Docker Compose. The service waits for inference servers to be ready, runs the tests, and saves results in JSON. Reproducibility on different hardware — without manual configuration.

## Tech Stack

**Inference:** vLLM | **Data:** pandas, NumPy | **Visualization:** Plotly, Kaleido | **Infrastructure:** Docker, Docker Compose

## Key Results

- **One of the first public DGX Spark benchmarks** — objective evaluation of a new device for the ML community
- **Basis for a [Habr article](https://habr.com/ru/articles/994058/)** analyzing the real-world capabilities of DGX Spark
- **Plateau effect discovery** — throughput stops growing beyond 50 concurrent requests due to LPDDR5X bandwidth limitations (273 GB/s)
- **Full reproducibility** — containerized pipeline without manual configuration
