# Добавление статей блога

Руководство по добавлению новых статей в блог портфолио.

## Расположение файлов

```
app/content/
├── ru/blog/    # Русскоязычные статьи
└── en/blog/    # Англоязычные статьи
```

## Именование файлов

Формат: `{НОМЕР}.{slug}.md`

- Номер — порядковый (следующий свободный, текущий: 1)
- Slug — латиница, lowercase, дефис как разделитель
- Пример: `2.fine-tuning-bert.md`

Файл с одинаковым именем должен быть создан в обеих локалях.

## Frontmatter

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `title` | string | Да | Заголовок статьи |
| `description` | string | Да | Краткое описание (1 строка). На списке блога обрезается до 3 строк |
| `date` | string | Да | Дата публикации в формате `YYYY-MM-DD` |
| `tags` | string[] | Да | Массив тегов для категоризации |

## Шаблон статьи

```yaml
---
title: 'Заголовок статьи'
description: 'Краткое описание статьи в одну строку'
date: '2026-03-01'
tags: [ml, python, tutorial]
---
```

### Структура контента

```markdown
## Введение

Вводный абзац: о чём статья, зачем это нужно, для кого.

## Основная часть

### Подсекция 1

Текст с примерами кода, **выделением** и `inline code`.

```python
# Блок кода с подсветкой синтаксиса
def example():
    return "Hello"
```

### Подсекция 2

Маркированные списки:
- **Пункт 1** — описание
- **Пункт 2** — описание

## Результаты / Выводы

Итоги, ключевые выводы, ссылки на ресурсы.
```

## Форматирование

### Заголовки

- `##` (h2) — основные секции статьи
- `###` (h3) — подсекции
- `####` (h4) — вложенные подсекции
- **Не используйте** `#` (h1) — заголовок статьи берётся из `title` в frontmatter

### Поддерживаемые элементы Markdown

| Элемент | Синтаксис | Пример |
|---------|-----------|--------|
| Жирный текст | `**текст**` | **жирный** |
| Курсив | `*текст*` | *курсив* |
| Inline code | `` `код` `` | `код` |
| Ссылка | `[текст](url)` | [пример](https://example.com) |
| Изображение | `![alt](url)` | — |
| Blockquote | `> текст` | — |
| Таблица | Markdown таблицы | — |
| Маркированный список | `- пункт` | — |
| Нумерованный список | `1. пункт` | — |

### Блоки кода

Поддерживаемые языки для подсветки синтаксиса:
- `python`
- `typescript` / `ts`
- `javascript` / `js`
- `bash` / `shell`
- `yaml`
- `json`
- `sql`
- `dockerfile`

Пример:
````markdown
```python
import torch
model = torch.load("model.pt")
```
````

### Стилистические рекомендации

- **Абзацы:** 1-3 предложения, без длинных полотен текста
- **Списки:** используйте bold-префиксы: `- **Термин** — описание`
- **Технические термины:** выделяйте `code` или **bold**
- **Ссылки:** добавляйте на внешние ресурсы, где это уместно
- **Код:** всегда указывайте язык для подсветки

## SEO

Метаданные генерируются автоматически:
- **Title:** `{title} — ML Architect`
- **Description:** значение поля `description` из frontmatter
- **OG-теги:** автоматически из title и description

## Отображение

- **Список блога:** карточки с заголовком, датой, описанием (до 3 строк) и ссылкой "Читать далее"
- **Страница статьи:** заголовок, дата, markdown-контент со стилями `prose prose-invert`
- **Сортировка:** по дате (новые первыми)
- **Формат даты:** полный формат — "1 марта 2026 г." (рус.) / "March 1, 2026" (англ.)

## Чек-лист перед публикацией

- [ ] Файл создан в `app/content/ru/blog/` и `app/content/en/blog/`
- [ ] Имя файла: `{N}.{slug}.md` (N — следующий свободный номер)
- [ ] Frontmatter содержит все обязательные поля (title, description, date, tags)
- [ ] `date` — формат `YYYY-MM-DD`
- [ ] `description` — помещается в 3 строки
- [ ] Заголовки секций — h2 (`##`), не h1
- [ ] Блоки кода содержат указание языка
- [ ] Контент проверен в обеих языковых версиях
