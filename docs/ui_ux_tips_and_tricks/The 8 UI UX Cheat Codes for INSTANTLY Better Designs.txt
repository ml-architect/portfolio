При создании UI/UX дизайна строго следуй этим 8 правилам:

## 1. Кернинг крупного текста

Для текста размером более 70-80px уменьшай межбуквенное расстояние (letter-spacing) на -2% ... -4%. На мелком тексте автоматический кернинг работает нормально, но при масштабировании текст начинает выглядеть разрозненно и нескладно. Тонкая настройка кернинга особенно критична для крупных заголовков на лендингах и hero-секциях.

## 2. Вложенные скругления углов

Когда один скруглённый элемент находится внутри другого, внутренний радиус должен быть меньше внешнего. Формула:

  inner_radius = outer_radius - gap

Например: внешний радиус 30px, расстояние между элементами 10px => внутренний радиус = 20px. Если расстояние превышает внешний радиус — внутренний элемент делай с прямыми углами. Для pill-форм (капсул) это правило не нужно, так как расстояние одинаково по всему периметру. Дополнительно: используй iOS Corner Smoothing (сглаживание углов) для более мягкого, плавного перехода в скругление вместо резкого геометрического.

## 3. Подбор цветовых палитр через HSB

Не угадывай цвета — используй модель HSB (Hue, Saturation, Brightness) для построения гармоничной палитры:

- Начни с базового цвета.
- Для более тёмного оттенка: увеличь Saturation на ~20, уменьши Brightness на ~10.
- Для ещё более тёмного: повтори то же самое от предыдущего оттенка (+20 S, -10 B).
- Для продвинутого результата корректируй ещё и Hue: сдвигай его в сторону синего/фиолетового на ~20 пунктов для тёмных оттенков (синие и фиолетовые тона воспринимаются темнее, жёлтые и красные — светлее).

## 4. Продуманная компоновка карточек

Не размещай информацию плоским списком с подписями. Вместо этого:

- Убери избыточные лейблы — если интерфейс не говорит сам за себя без подписей, значит он плохо спроектирован.
- Группируй связанную информацию (название + локация, цена + рейтинг, даты заезда/выезда).
- Ранжируй по важности: самое значимое — крупнее и выше, второстепенное — меньше и ниже.
- Используй иконки вместо текстовых подписей для деталей.
- Стэкай (располагай вертикально) пары связанных элементов, а однотипные детали — в горизонтальный ряд.

## 5. Убирай разделительные линии

Линии-разделители чаще всего избыточны и захламляют интерфейс. Вместо них:

- Используй достаточные отступы между элементами, чтобы визуально разделить их.
- Если отступы приходится делать маленькими, применяй чередующийся фон строк (zebra striping) вместо линий.
- Чем меньше визуальных элементов ты используешь для передачи структуры — тем лучше.
- Исключение: линии допустимы, если это осознанный стилевой приём всего дизайна.

## 6. Единообразные отступы по сетке

Используй базовую сетку 4px или 8px для всех отступов и размеров, особенно на мелких элементах. Это даёт заметно более организованный и аккуратный результат.

- Для мелких элементов: строго кратно 4 или 8 пикселям.
- Для крупных размеров: округляй до ближайшего 5 или 10 (разница между 120 и 128 не имеет значения на больших масштабах).
- В Figma: установи nudge amount = 8 (вместо дефолтных 10), чтобы все мелкие подвижки оставались на сетке.

## 7. Цветные фоны вместо чистого чёрного/белого

Не используй чисто чёрный (#000) или чисто белый (#FFF) как фон. Вместо этого подмешивай акцентный цвет:

- Светлая тема: очень светлый оттенок акцентного цвета как фон. По Tailwind CSS: фон = оттенок 50, акцент = оттенок 500.
- Тёмная тема: очень тёмный оттенок акцентного цвета вместо чёрного. По Tailwind CSS: основной цвет = оттенок 300, фон = оттенок 950.

Это добавляет теплоты и характера интерфейсу при нулевом риске ошибки.

## 8. Глубина в тёмной теме через вариации фона

В тёмном режиме тени почти не работают. Создавай глубину с помощью карточек с разными оттенками фона:

- Бери тёмный базовый фон.
- Для каждого следующего слоя: увеличивай Brightness на 4-6, уменьшай Saturation на 10-20.
- Повторяй для каждого уровня вложенности (фон страницы -> карточка -> вложенный элемент).
- Это создаёт чёткую визуальную иерархию и ощущение глубины без теней.
