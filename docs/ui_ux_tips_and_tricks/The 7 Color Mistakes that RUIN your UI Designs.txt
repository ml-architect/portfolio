Видео разбирает 7 самых распространённых ошибок в работе с цветом в UI-дизайне на примере редизайна приложения для управления файлами. Автор показывает конкретные приёмы: от сокращения палитры и нейтрального баланса до правильного построения тёмной темы и работы с состояниями элементов.

## 1. Правило 60-30-10: ограничивай количество цветов
Не раскрашивай каждый элемент в свой цвет — «меньше почти всегда лучше». Придерживайся правила 60-30-10:
- 60% — доминирующий нейтральный цвет (фон, основные поверхности);
- 30% — вторичный цвет (карточки, панели, дополнительные блоки);
- 10% — акцентный цвет (кнопки действий, ключевые элементы).
Пример: Lemon Squeezy — 60% фиолетового, 30% жёлтого, 10% белого.
Если у тебя на экране пять разных акцентных цветов, конкурирующих за внимание, — замени их одним брендовым цветом. Но следи за тем, чтобы яркий брендовый цвет не доминировал на элементах, которые не заслуживают такого внимания: лучше осветлить его, чтобы он меньше «кричал», и переключить текст на чёрный для лучшей читаемости.

## 2. Иконки не нуждаются в цвете — используй цвет только для обозначения статуса
Иконки — это узнаваемые символы, и в большинстве случаев им не нужен цвет. Цвет должен быть зарезервирован для передачи статуса и состояния, например: подсветка активного таба в навигации. Убери декоративный цвет с иконок, которые используются «просто для красоты», — это создаёт визуальный шум и конкурирует с действительно важными элементами.

## 3. Нейтральный баланс (Neutral Balance): фоны должны оставаться на заднём плане
Так же как мы используем белое пространство для создания иерархии и «воздуха» между элементами, цвет нуждается в пространстве нейтральных тонов. Правила для фонов:
- Почти никогда не используй яркие цвета для фона — фон должен оставаться «на заднём плане».
- Начинай с нейтрально-серого фона и очень светлого / белого переднего плана.
- Для тёмной темы: более тёмные цвета — на задний план, чуть более светлые — на передний.
- Чтобы интегрировать цвет в нейтральный фон, возьми нейтральный серый и добавь в него лёгкий оттенок (hint) брендового цвета (например, чуть-чуть фиолетового). Пример: Headspace берёт оттенок своего яркого оранжевого и использует его на фоне карточек — всплеск цвета без перегрузки визуала.
- В светлой теме можно инвертировать фон и передний план (тёмный фон, светлый контент) — особенно для небольших UI-элементов вроде строки поиска, для лучшей accessibility. Пример: строка поиска Dribbble.
- Иногда лучшее решение — убрать фон карточек совсем и оставить только border (обводку). Это убирает визуальный «мусор» и добавляет лёгкости.

## 4. Расширяй палитру от брендового цвета: аналоговые и комплементарные цвета
Когда у тебя всего один брендовый цвет и нужно больше оттенков (например, для прогресс-бара хранилища):
- Поверни оттенок (hue) на цветовом круге немного в обе стороны, чтобы получить аналоговые цвета (например, из фиолетового — голубой и розоватый).
- Выбери комплементарный цвет с противоположной стороны цветового круга (например, для фиолетового — ярко-жёлтый).
- Все эти цвета хорошо сочетаются, потому что они аналоговые и комплементарные.
Если брендовый цвет не проходит WCAG-проверку контраста с белым текстом (например, ярко-оранжевый):
- Затемни его — тёмная версия пройдёт проверку.
- Или используй комплементарный цвет (например, для оранжевого — синий), который тоже пройдёт WCAG.
Примеры из реальных брендов:
- Mailchimp: яркий жёлтый (primary) + комплементарный бирюзовый (secondary).
- Airbnb: яркий розовый + глубокий тёмный розовый в паре.
Не бойся адаптировать брендовые цвета ради хорошего дизайна и accessibility.

## 5. Избегай чисто чёрного и чисто белого — используй оттенки серого для иерархии
Чисто белый (#FFFFFF) и чисто чёрный (#000000) — допустимые цвета, но часто есть варианты лучше. Суть — в иерархии:
- Для второстепенного текста (размер файла, тип, метки) используй тёмно-серый вместо чёрного — это визуально понижает приоритет информации.
- Для border'ов: если элементы хорошо сгруппированы, border можно сделать ещё светлее.
- На практике большая часть текста в качественном UI — светло-серый или тёмно-фиолетовый, а не чёрный.
- В тёмной теме можно быть ещё агрессивнее: понижай яркость серых тонов, а чисто белый (#FFFFFF) оставляй только для самых важных действий / текста.
Умение работать с серым (а не с чёрным и белым) — это то, что отличает посредственных дизайнеров от профессионалов.

## 6. Тёмная тема — это НЕ простая инверсия светлой темы
Если просто инвертировать все цвета из light mode, результат будет неудовлетворительным. Правила построения тёмной темы:
- Border'ы и карточки в тёмной теме нужно делать ярче, чем кажется: в тёмных тонах разница между оттенками хуже заметна, чем в светлых — поэтому контраст между элементами должен быть выше.
- Для основного текста используй светло-серые тона, а не чисто белый — это снижает нагрузку на глаза.
- Чисто белый (#FFFFFF) резервируй для самых важных элементов: логотип, ключевые числа, главные действия.
- Логотипы и брендовые элементы в тёмной теме часто нужно десатурировать (снизить насыщенность), чтобы они не «кричали» на тёмном фоне.
Итого: палитра для тёмной темы должна строиться отдельно, с учётом задач тёмного режима (снижение нагрузки на глаза, читаемость), а не механически инвертироваться.

## 7. Семантические цвета: красный для деструктивных действий, цвет для статусов
Красный и зелёный часто входят в палитру бренда, даже если они не являются брендовыми цветами. Причина: они несут семантику.
- Красный для деструктивных действий (удалить, отменить): кнопка «Delete» должна быть красной, а не брендового цвета — красный передаёт опасность и привлекает внимание к последствиям действия. Фиолетовая кнопка «Delete» будет более «on-brand», но не донесёт деструктивную природу действия.
- Красный для уведомлений — это более гибко; здесь допустимо использовать и брендовый цвет вместо красного.
- Зелёный — для подтверждений и успешных действий.
Не жертвуй юзабилити ради следования бренду: пользователь должен мгновенно считывать намерение элемента по его цвету.

## 8. Цветовые состояния элементов (States): hover, active, disabled
Состояния элементов можно передавать одним только цветом, без дополнительных визуальных подсказок:
- Disabled (отключённый): десатурируй цвет (если элемент не и так серый). Светло-серый фон + белый текст — очевидно считывается как «неактивно».
- Hover (наведение): используй чуть более светлую / яркую версию базового цвета.
- Active / Pressed (нажатие): используй чуть более тёмную версию базового цвета.
- На мобильных устройствах hover-эффектов нет — используй только эффект нажатия (press). Добавление чуть более тёмного серого при нажатии создаёт физическое ощущение «вдавливания».
Эти правила одинаково работают и в светлой, и в тёмной теме.
