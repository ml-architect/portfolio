Видео описывает 11 конкретных микроанимаций и взаимодействий, которые делают интерфейс более живым, приятным и профессиональным. Каждый приём показан на реальных примерах (Linear, Obsidian, Apple, Dub.co, Vercel) и реализован в Figma.

## 1. Hover-эффект кнопки через сдвиг текста

Вместо того чтобы подбирать случайные цвета для состояний hover и active, используй текстовую анимацию. Создай маску с двумя одинаковыми текстовыми слоями внутри кнопки. При наведении (hover) текст сдвигается вверх, показывая второй слой — это создаёт эффект «прокрутки» текста. Для клика (while pressing) уменьшай масштаб кнопки — она становится визуально меньше, давая тактильную обратную связь. Преимущество: не нужно возиться с подбором цветов для разных состояний. Можно варьировать направление движения: горизонтальный или диагональный сдвиг текста вместо вертикального.

## 2. Анимация клавиатурных сочетаний

Если интерфейс использует keyboard shortcuts, визуализируй их нажатие с помощью микроанимации. При нажатии комбинации клавиш каждая клавиша визуально «утапливается» на экране, а при успешном выполнении появляется анимированное сообщение об успехе. Это помогает пользователю запомнить горячие клавиши — визуальная обратная связь закрепляет их в памяти значительно лучше, чем статический список. Реализация: привяжи каждую клавишу к своему триггеру, а финальное сообщение об успехе показывай, когда все клавиши комбинации нажаты одновременно.

## 3. Toast-уведомления с многоэтапной анимацией

Toast-уведомления должны быть не просто статичными карточками, а интерактивными элементами с несколькими этапами анимации:

- Появление: карточка плавно выезжает снизу вверх.
- Загрузка: внутри карточки показывай анимацию загрузки (спиннер, progress bar).
- Успех: после завершения загрузки показывай анимированное сообщение об успехе с декоративными частицами (particle animations).

Реализация: используй цепочку delay-триггеров (задержки), запускаемых одним кликом. Каждый этап последовательно сменяет предыдущий.

## 4. Всплывающий name tag при наведении на аватар

Если в интерфейсе есть маленькие аватары (headshots) — например, команда на сайте агентства или участники чата — при hover на аватар показывай маленький тег с именем.

Реализация в Figma:
- Первый фрейм: аватар + name tag смещён вниз, opacity = 0%.
- Второй фрейм: name tag виден, с небольшим наклоном (tilt) для живости.
- Name tag: используй Auto Layout (Shift+A), скругли углы до максимума (pill-форма), настрой внутренние отступы (padding), чёрная заливка фона, белый текст.
- Анимация: hover-триггер + Smart Animate + custom spring easing: длительность 500ms, stiffness 636, damping 24.

Эти конкретные параметры пружинной анимации (stiffness 636, damping 24) создают приятный «подпрыгивающий» эффект. Без них анимация выглядит плоско и механически.

## 5. Shimmer stroke (мерцающая обводка)

Анимированная градиентная обводка — современный и эффектный приём. Суть:

1. Создай контур обводки с помощью операции Subtract: внешний прямоугольник минус внутренний (чуть меньший) = тонкая рамка.
2. Создай круг с angular gradient (угловой градиент) — от прозрачного к цветному.
3. Наложи круг на рамку через маску (Mask) — градиент виден только через контур обводки.
4. Анимируй вращение круга с angular gradient — получается эффект «бегущего блика» по обводке.

Особенность: из-за геометрии (круг вписан в прямоугольник) блик движется быстрее по коротким сторонам и медленнее по длинным. Если это критично — можно добавить пользователю кнопку паузы/воспроизведения.

## 6. Тултипы с задержкой (delayed tooltips)

Если в интерфейсе много иконок без подписей, показывай tooltip (всплывающую подсказку) только если пользователь задержал курсор на иконке дольше 1 секунды. Это предотвращает мельтешение подсказок при быстром перемещении мыши.

Реализация:
- Триггер: Mouse Enter (не Hover!) с задержкой 1000ms.
- Возврат: триггер Mouse Leave — немедленный возврат к исходному фрейму.
- Анимация появления: используй custom spring easing для эффекта упругого «выскакивания» тултипа.

Ключевое отличие: задержка в 1000ms означает, что подсказка появится только при намеренном наведении, а не при случайном проведении мышью мимо иконки.

## 7. Всплывающее превью при наведении на текст (text hover pop-out)

При наведении на текстовую ссылку или ключевое слово показывай всплывающее изображение или превью, иллюстрирующее то, о чём говорит текст. Это способ показать контент без дополнительных слов или постоянно отображаемых изображений — экономит место и удерживает внимание.

Реализация: для каждого интерактивного текстового элемента создай hover-триггер, по которому рядом с текстом появляется фрейм с изображением/превью. Изображения можно делать «игривыми» — с наклоном, тенями, скруглёнными углами.

## 8. Анимированный progress bar для форм

Формы скучны, но анимированный прогресс-бар удерживает пользователя. Техника создания плавной анимации:

1. Создай полную обводку (stroke) прогресс-бара — контур вокруг всех шагов.
2. Создай цветной прямоугольник (заливка акцентным цветом, например фиолетовым).
3. Наложи обводку как маску на прямоугольник.
4. Анимируй горизонтальное перемещение прямоугольника — он «проявляет» обводку по мере продвижения, создавая эффект «рисования» прогресса.

Круглые элементы (точки на шкале) красиво заполняются благодаря маскированию. Остальные элементы формы (текст, поля) появляются/исчезают через fade in/out при переходах между шагами.

## 9. Свайп карточек (card swipe)

Компактный способ показать несколько уведомлений или элементов в ограниченном пространстве (например, нижний правый угол как центр уведомлений):

1. Расположи карточки стопкой друг на друге.
2. При свайпе/клике верхняя карточка поворачивается (rotate) в сторону и исчезает (opacity -> 0%).
3. Карточки позади одновременно: увеличиваются в масштабе (scale up) и сдвигаются вниз, заполняя освободившееся место.

Критически важная деталь: при уходе верхней карточки остальные карточки должны сдвинуться вниз, занимая освободившееся пространство. Именно это движение «вперёд» создаёт ощущение настоящей стопки, а не статичного списка.

## 10. Раскрытие строки поиска из иконки (search bar expansion)

Вместо постоянно отображаемой громоздкой строки поиска показывай только иконку лупы. При клике на лупу:

1. Первый фрейм: иконка лупы (компактное состояние).
2. Второй фрейм (по клику): лупа начинает «растворяться» в расширяющемся круге/поле.
3. Третий фрейм (по задержке): полностью раскрытая строка поиска с полем ввода.

Реализация: три фрейма, клик-триггер на первом, delay-триггер для перехода ко второму и третьему. Эффект «исчезновения лупы в круге» добавляет элегантности переходу.

## 11. Hover для деталей апгрейда

Когда в интерфейсе отображаются лимиты текущего плана (например, «25 ссылок»), при наведении на этот элемент плавно показывай, что даст апгрейд до Pro. Вместо зачёркивания текста (что выглядит неаккуратно) используй slide-in эффект: текущее значение плавно уходит, а новое — заезжает на его место. Это простое, но продуманное взаимодействие, которое ненавязчиво продвигает премиум-план в контексте использования.
