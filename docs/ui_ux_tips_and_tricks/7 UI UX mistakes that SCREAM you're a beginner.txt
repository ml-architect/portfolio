Видео разбирает 7 (+1 бонусная) типичных ошибок начинающих дизайнеров интерфейсов на примере редизайна приложения рецептов. Каждая ошибка показана в формате «до/после» с конкретными рекомендациями по исправлению.

## 1. Пользовательский поток (User Flow)
Прежде чем приступать к визуальному дизайну, продумай весь путь пользователя от начала до конца. Даже простой набросок экранов в виде прямоугольников на бумаге помогает обнаружить пробелы на ранней стадии.

Что именно проверять:
- На каждом экране с выбором вариантов обязательно предусмотри поле поиска — пользователь должен иметь возможность ввести своё значение, если его нет в пресетах.
- Предусмотри кнопку «Пропустить» (Skip) — не у каждого пользователя есть данные для заполнения на текущем шаге.
- Добавляй вспомогательные элементы: иконка фильтра в строке поиска, кнопка «Сохранить» в видимом месте (например, в верхней части экрана).
- Наиболее часто упускаемые элементы: навигационные ссылки, скрытые состояния (hidden states), динамические взаимодействия — наведение на кнопки, микро-анимации.

## 2. Злоупотребление визуальными эффектами
Тени, свечения, градиенты — всё это перегружает интерфейс и делает его «любительским». Правило: чем меньше визуального шума, тем лучше дизайн.

Градиенты:
- Если градиент действительно нужен — используй вариации одного цвета (например, более тёмный и более светлый оттенок зелёного), а не контрастные цвета вроде синего + зелёного.
- В большинстве случаев без градиента выглядит чище — убери его полностью и оцени результат.

Тени (drop shadow):
- Никогда не используй дефолтные тени Figma — они слишком жёсткие.
- Вместо того чтобы просто снижать opacity, смени цвет тени на светло-серый и значительно увеличь blur.
- Если тень не несёт функциональной нагрузки — удали её полностью.

## 3. Отступы и пространство (Spacing)
Интерфейсы новичков почти всегда «слишком плотные». Используй два ключевых инструмента: сетки (Grids) и Auto Layout.

Сетки:
- Настрой сетку для каждого экрана: например, 3-колоночная сетка для одного экрана, 2-колоночная для другого — в зависимости от контента.
- Выравнивай элементы по сетке как можно точнее. Если элемент слегка выходит за сетку, но визуально смотрится сбалансированно — это допустимо.

Вертикальные отступы:
- Увеличивай вертикальные расстояния между блоками, чтобы контент «дышал» и группировался естественно.
- На мобильных экранах пространства нужно больше, чем кажется на первый взгляд.

Auto Layout:
- Применяй Auto Layout к карточкам, чипам и другим повторяющимся элементам.
- Отключи Vertical Trim для попиксельного контроля.
- Настраивай внутренние отступы (padding) и расстояния между элементами (gap) через Auto Layout, а не вручную — это гарантирует консистентность.

## 4. Неконсистентные компоненты
Разные corner radius, разный стиль одинаковых по функции элементов — всё это моментально выдаёт непрофессиональный дизайн. Это одна из самых простых ошибок для исправления.

Конкретные правила:
- Одинаковые по функции элементы (например, два поля поиска) должны быть идентичными визуально — различаться только текстом-подсказкой (placeholder).
- Установи единый corner-radius для всех мелких компонентов — рекомендуемое значение: 10px. Это правило, которое большинство дизайнеров упускают.
- Кнопки одного уровня (например, «Пропустить» и «Назад») должны совпадать по размеру, corner-radius и стилю.
- Используй Styles для цветов, Variables для числовых значений (отступы, радиусы), Components для UI-элементов — это обеспечит единообразие всего дизайна.

## 5. Иконки
Хорошие иконки значительно улучшают дизайн, а плохие — серьёзно вредят. Это часто недооценивают.

Правила работы с иконками:
- Добавляй иконки к карточкам — без них пользователь вынужден больше читать, что замедляет процесс просмотра. Иконка может заменить текстовую кнопку (например, иконка сохранения вместо текста «Save»).
- Все иконки на одном экране должны быть из одной библиотеки и в одном стиле: одинаковый fill/stroke, одинаковая толщина линий (line width), единый визуальный стиль.
- Рекомендуемые библиотеки: Flat Icons (на сайте выбрать interface icons, отсортировать по stroke, width, corner), Feather Icons и Phosphor Icons (плагины для Figma).
- При скачивании иконок всегда выбирай формат SVG.
- Если иконка общепризнанная (дом, закладка, профиль пользователя) — подпись не нужна. Для неочевидных иконок добавь tooltip при наведении — это упрощает онбординг.
- Избегай «странных» иконок, значение которых невозможно угадать.

Продвинутый совет:
- Допустимо использовать иконки разных стилей в одном дизайне, но только если они визуально разделены — находятся в разных зонах интерфейса и служат для разных целей. Например: навигационные иконки в навбаре, иконки категорий еды в основном контенте и иконки рецептов на карточках — могут отличаться по стилю, если они пространственно разнесены.

## 6. Избыточные элементы (Redundant Elements)
Тренд на «заполнение каждого пикселя» приводит к визуальному мусору. Убирай всё, что не несёт функциональной ценности.

Что удалять:
- Стрелки навигации, если на мобильном устройстве доступен свайп — стрелка избыточна.
- Декоративные стрелки, которые не ведут никуда — это чистый визуальный шум. Если стрелка ведёт к действию, но второстепенна — приглуши её цвет.
- Обводки (stroke) элементов, если они не нужны для разделения контента. Если беспокоит контрастность — значительно уменьши opacity обводки вместо полного удаления, но в большинстве случаев можно убрать совсем.

Принцип: при сомнениях — удаляй. Если после удаления элемент не вспоминается — он не был нужен.

## 7. Интерактивная обратная связь (Interactive Feedback)
Если при злоупотреблении эффектами ошибка в «слишком много», то здесь — в «слишком мало». Пользователь должен всегда понимать, что его действие зафиксировано.

Конкретные приёмы:
- При нажатии кнопки, если следующий экран загружается не мгновенно — делай кнопку серой (grayed out) сразу после нажатия. Это показывает пользователю, что клик зарегистрирован и следующий экран загружается.
- Если загрузка занимает заметное время — добавь индикатор загрузки (loading spinner).
- При нажатии на иконку «Сохранить» — заполняй её (из outline в filled). Дополнительно: добавь красную точку-индикатор (badge) на вкладку «Сохранённое» — так пользователь видит, что рецепт действительно добавлен.
- Продумывай состояния для каждого интерактивного элемента: default, hover, pressed, disabled, loading.

## 8. Графики и диаграммы (Charts) — бонус
Графики часто портят «передизайном» ради эстетики в ущерб читаемости.

Ошибки, которых следует избегать:
- Не убирай вертикальную ось — без неё невозможно считать значения.
- Не используй скруглённые верхушки столбцов (rounded top) — из-за них трудно определить, где именно заканчивается столбец.
- Количество столбцов должно соответствовать данным (например, 7 столбцов для 7 дней недели, а не 16).
- Простой, «менее эстетичный» график, который легко читается, всегда лучше красивого, но непонятного.

Принцип: основная функция графика — передать информацию. Дизайн графика должен помогать считывать данные, а не мешать.
