## О проекте

Security Event Manager (SEM) — полнофункциональная платформа для SOC-команд (Security Operations Center). Автоматически обнаруживает, классифицирует и эскалирует инциденты ИБ. Анализирует события SIEM, применяет ML-модели для детекции атак, сопоставляет с Sigma-правилами и создаёт алерты в TheHive.

## Задача

Создать единый интерфейс для SOC-команды, который автоматизирует рутину: анализ тысяч SIEM-событий, матчинг с правилами детекции, классификацию атак и эскалацию в TheHive. С AI-ассистентом, работающим локально без отправки данных за периметр.

## Архитектура

### Двухконтейнерная система

**Web Container** — FastAPI + Vue 3 SPA. 17 роутеров, async SQLite для конфигурации, интеграция с PostgreSQL SIEM.

**Back Container** — автономный процессор. CatBoost ML-модели, Sigma Rules Engine, циклическая обработка SIEM-событий (30-40 мин).

**Ollama** — локальный AI-сервер (Qwen3, DeepSeek-R1) для чат-ассистента.

### Sigma Rules Engine

Написан с нуля — recursive descent парсер с поддержкой AND/OR/NOT, wildcard, кванторов ("1 of", "all of"), модификаторов (contains, cidr, re). 11 встроенных тестов.

### ML-детекция

Двухуровневая система CatBoost: первая модель определяет факт атаки, вторая классифицирует тип. Применяется в каждом цикле обработки.

### Аналитика правил

Автоматическое выявление проблемных правил: dead (0 срабатываний), noisy (> avg * 3), declining (падение > 50%).

## Стек технологий

**Backend:** Python 3.11, FastAPI 0.109, aiosqlite, psycopg2, Alembic, HMAC-SHA256, bcrypt

**Frontend:** Vue 3.4, TypeScript 5.3, Vite, Pinia, Tailwind CSS 3.4, Radix Vue, shadcn/vue

**ML/AI:** CatBoost 1.2.7, Ollama (Qwen3, DeepSeek-R1), pySigma

**Инфраструктура:** Docker (multi-stage), Docker Compose, GitLab CI/CD

**Интеграции:** TheHive, SIEM PostgreSQL

## Ключевые достижения

- **Sigma Rules Engine с нуля** — полноценный recursive descent парсер, покрытый 11 тестами
- **ML-детекция атак** — двухуровневый CatBoost для обнаружения и классификации в реальном времени
- **Аналитика эффективности правил** — автоматическое выявление dead/noisy/declining правил для поддержания качества rule-базы
- **Локальный AI** — данные не покидают периметр организации, Git2Markdown конвертер для загрузки кода в контекст
- **Эволюция архитектуры** — миграция с Streamlit на FastAPI + Vue 3 с сохранением обратной совместимости данных
- **RBAC** — гранулярные права по разделам, HMAC-SHA256 (timing-attack safe), bcrypt
