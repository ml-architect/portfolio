## О проекте

Palatine Cloud — production-ready облачная платформа, предоставляющая API-сервисы для обработки речи и текста на базе ML-моделей. Полный цикл: от распознавания речи (ASR) и диаризации спикеров до анализа тональности, LLM-суммаризации и синтеза речи (TTS).

Все сервисы объединены единым API Gateway с аутентификацией, биллингом, мониторингом и развёрнуты в микросервисной архитектуре на выделенных серверах с GPU.

## Задача

Создать облачную платформу уровня production, которая позволяет клиентам через единый API обрабатывать речь — транскрибировать аудио в текст, определять спикеров, анализировать тональность и генерировать суммаризацию. Платформа должна поддерживать real-time стриминг, систему биллинга с тарифами и полную наблюдаемость всех сервисов.

## Архитектура

Микросервисная архитектура из **10 независимых сервисов** на нескольких серверах:

### Cloud Transcription & Diarization API
Основной ML-сервис платформы. Двухуровневая архитектура: FastAPI (оркестрация, аутентификация, биллинг) + Ray Serve (GPU inference). Стриминговая транскрипция через WebSocket → Triton gRPC pipeline с поддержкой 15+ аудиокодеков и sample rates 8–48 kHz. OpenAI-совместимый API для транскрипции и chat completions.

### Payment Processing Gateway
Платёжный шлюз с **Clean Architecture** — Protocols, Use Cases, Entities, Factories, Adapters. Мультимерчант-маршрутизация платежей, интеграция YooKassa с webhook-обработкой, фискализация по 54-ФЗ, промокоды. Идемпотентная обработка webhook-событий.

### Super Star Gate (API Gateway)
Динамический API Gateway с декларативными маршрутами из YAML, 4 типами аутентификации (token, user, dev_token, dev_user), request composition — цепочки запросов с подстановкой результатов через плейсхолдеры.

### Supabase (Self-Hosted)
Self-hosted деплой Supabase из 12+ контейнеров — Auth, Realtime, Storage, Edge Functions, PostgREST, Kong, Supavisor. Кастомные SQL-миграции с RLS-политиками.

### Monitoring Stack
Grafana + Prometheus + Loki + Alloy с мультинодной архитектурой. Сбор метрик Ray Serve, Docker-логов со всех серверов.

### TTS Server
Контейнеризированный Fish Speech с GPU inference, клонированием голоса и OpenAI-совместимым API.

### UI Backend, Billing UI, Docs
Бэкенд личного кабинета с Redis rate limiting, административная панель биллинга, публичная API-документация на Mintlify.

## Стек технологий

**Backend:** Python 3.10–3.11, FastAPI, Uvicorn, Pydantic, httpx, WebSocket

**ML/AI:** NVIDIA Triton Inference Server (gRPC, sequence batching), Ray Serve, CUDA 12.6, Fish Speech (TTS), Ollama, CTC Decoder

**Данные:** PostgreSQL 15, Supabase, Redis, Alembic

**Инфраструктура:** Docker / Docker Compose, GitLab CI/CD (lint → test → deploy), Traefik (TLS, HSTS), Grafana, Prometheus, Loki, Alloy

**Интеграции:** YooKassa (54-ФЗ), Supabase Auth (JWT), OpenAI-совместимый API, Mintlify

## Ключевые достижения

- **Real-time стриминговая транскрипция** — WebSocket → decode → resample → Triton gRPC → CTC decode. 15+ аудиокодеков, stateful inference с sequence batching
- **OpenAI-совместимый API** — drop-in замена OpenAI для `/audio/transcriptions` и `/v1/chat/completions`, совместимость с OpenAI SDK
- **Clean Architecture в биллинге** — полноценное разделение на слои с Protocol-based интерфейсами и мультимерчант-маршрутизацией
- **Полный стек мониторинга** — Grafana + Prometheus + Loki с мультинодным сбором метрик и логов
- **CI/CD с покрытием** — GitLab CI пайплайны с lint (flake8/black), unit/integration/E2E тестами, coverage-отчётами, JUnit-артефактами
- **Нейросетевой TTS** — синтез речи с клонированием голоса, OpenAI-совместимый API
