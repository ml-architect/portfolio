## About the Project

Palatine Cloud is a production-ready cloud platform providing API services for speech and text processing powered by ML models. Full cycle: from automatic speech recognition (ASR) and speaker diarization to sentiment analysis, LLM summarization, and text-to-speech (TTS).

All services are unified by a single API Gateway with authentication, billing, monitoring, and deployed in a microservices architecture on dedicated GPU servers.

## Objective

Build a production-grade cloud platform that enables clients to process speech through a unified API — transcribe audio to text, identify speakers, analyze sentiment, and generate summaries. The platform must support real-time streaming, a billing system with pricing tiers, and full observability across all services.

## Architecture

Microservices architecture comprising **10 independent services** across multiple servers:

### Cloud Transcription & Diarization API
The platform's core ML service. Two-tier architecture: FastAPI (orchestration, authentication, billing) + Ray Serve (GPU inference). Streaming transcription via WebSocket → Triton gRPC pipeline with support for 15+ audio codecs and sample rates from 8 to 48 kHz. OpenAI-compatible API for transcription and chat completions.

### Payment Processing Gateway
Payment gateway built with **Clean Architecture** — Protocols, Use Cases, Entities, Factories, Adapters. Multi-merchant payment routing, YooKassa integration with webhook handling, fiscal compliance per Russian Federal Law 54-FZ, promo codes. Idempotent webhook event processing.

### Super Star Gate (API Gateway)
Dynamic API Gateway with declarative routes from YAML, 4 authentication types (token, user, dev_token, dev_user), request composition — chained requests with result substitution via placeholders.

### Supabase (Self-Hosted)
Self-hosted Supabase deployment with 12+ containers — Auth, Realtime, Storage, Edge Functions, PostgREST, Kong, Supavisor. Custom SQL migrations with RLS policies.

### Monitoring Stack
Grafana + Prometheus + Loki + Alloy with multi-node architecture. Collection of Ray Serve metrics and Docker logs from all servers.

### TTS Server
Containerized Fish Speech with GPU inference, voice cloning, and OpenAI-compatible API.

### UI Backend, Billing UI, Docs
Personal dashboard backend with Redis rate limiting, billing admin panel, and public API documentation on Mintlify.

## Tech Stack

**Backend:** Python 3.10–3.11, FastAPI, Uvicorn, Pydantic, httpx, WebSocket

**ML/AI:** NVIDIA Triton Inference Server (gRPC, sequence batching), Ray Serve, CUDA 12.6, Fish Speech (TTS), Ollama, CTC Decoder

**Data:** PostgreSQL 15, Supabase, Redis, Alembic

**Infrastructure:** Docker / Docker Compose, GitLab CI/CD (lint → test → deploy), Traefik (TLS, HSTS), Grafana, Prometheus, Loki, Alloy

**Integrations:** YooKassa (54-FZ), Supabase Auth (JWT), OpenAI-compatible API, Mintlify

## Key Achievements

- **Real-time streaming transcription** — WebSocket → decode → resample → Triton gRPC → CTC decode. 15+ audio codecs, stateful inference with sequence batching
- **OpenAI-compatible API** — drop-in replacement for OpenAI's `/audio/transcriptions` and `/v1/chat/completions`, compatible with the OpenAI SDK
- **Clean Architecture in billing** — full layer separation with Protocol-based interfaces and multi-merchant routing
- **Full monitoring stack** — Grafana + Prometheus + Loki with multi-node metrics and log collection
- **CI/CD with coverage** — GitLab CI pipelines with lint (flake8/black), unit/integration/E2E tests, coverage reports, JUnit artifacts
- **Neural TTS** — speech synthesis with voice cloning, OpenAI-compatible API
