## About the Project

Palatine Astro is a product AI platform for personalized astrological consultations via Telegram. A user asks a question in the chat, the bot routes the request to an LLM through a chain of n8n workflows, enriches the context from a vector knowledge base (pgvector) and astrological data from the MCP server, and displays the response in real time with a "typing" message effect.

The platform targets a B2C audience with subscription-based monetization. The architecture consists of three components: a Telegram bot, the n8n orchestrator, and an MCP server for generating astrological charts.

## Objective

Build an AI platform where the LLM does not merely answer questions but autonomously decides when to invoke tools — constructing natal and synastry charts. Responses must be enriched with context from a knowledge base (RAG) and displayed in real time. Administrators must be able to update the knowledge base directly through the bot.

## Architecture

### Telegram Bot (astro-tg-bot)

Asynchronous bot on aiogram 3 with streaming LLM response output. Sends a placeholder message and edits it as chunks arrive at 200 ms intervals. A custom incremental JSON parser built on `json.JSONDecoder.raw_decode()` extracts completed JSON objects from the stream buffer. A recursive parser `_extract_text_from_data` traverses arbitrary nested JSON structures, ensuring compatibility with any LLM response format.

Administrative module: role model based on `ADMIN_IDS`, stateful file upload mechanism for updating the knowledge base via bot → multipart/form-data → n8n → pgvector.

### MCP Server (Model Context Protocol)

Microservice on **FastMCP** providing the LLM with two tools:
- `birth_chart` — natal chart by date, time, and place of birth
- `synastry_chart` — compatibility chart for two people

Uses **Kerykeion** for astronomical calculations and SVG diagram generation. Parameters are described via `Pydantic Annotated + Field(description=...)` — the LLM "sees" the descriptions and autonomously decides when to invoke a tool. Transport: SSE on port 8000.

### n8n Orchestration

Middleware between the bot and the LLM. `EXECUTIONS_MODE=queue` mode with a Redis broker and a dedicated worker process. pgvector on PostgreSQL 17 stores embeddings for RAG — semantic search for relevant fragments on every request. 6 Docker containers with health checks, dependency ordering, and persistent volumes.

## Tech Stack

**Backend:** Python 3.11, aiogram 3.22, aiohttp (streaming), FastMCP 2.10+, Kerykeion (astronomy + SVG), Pydantic

**AI/ML:** n8n (AI orchestration), pgvector (RAG, semantic search), MCP (Model Context Protocol)

**Data:** PostgreSQL 15, pgvector (PostgreSQL 17), Redis 6

**Infrastructure:** Docker Compose (6 services), GitLab CI/CD (dev/prod), uv (Astral), non-root Docker runtime

## Key Achievements

- **Model Context Protocol (MCP)** — LLM autonomously invokes astrological chart tools through typed Pydantic annotations. A modern approach to AI agents that goes beyond simple prompt chains
- **RAG with dynamic updates** — pgvector for semantic search across the knowledge base with the ability to update via Telegram bot without restarting services
- **Streaming LLM output in Telegram** — custom incremental JSON parser for processing chunk streams in real time with API rate limit control
- **Recursive universal parsing** — traversal of arbitrary nested JSON structures using a prioritized list of 7 keys, compatible with any LLM response format
- **Queue-based n8n** — distributed processing with a Redis broker and dedicated worker, ensuring reliability under load
- **Production Docker infrastructure** — 6 containers with health checks, multi-layer caching (uv), bytecode compilation, non-root runtime, two-environment CI/CD with selective triggering
