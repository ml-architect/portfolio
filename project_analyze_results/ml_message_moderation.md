# ML Message Moderation — Интерактивная лаборатория модерации сообщений с ML-классификацией

**Репозиторий:** https://github.com/versus666jzx/ml_message_moderation
**Демо (Hugging Face Spaces):** https://huggingface.co/spaces/versus666/ml_message_moderation
**Заказчик:** РАНХиГС — Лаборатория интеллектуального анализа (https://lia.ranepa.ru/)

## Обзор проекта

ML Message Moderation — это интерактивное веб-приложение, разработанное для Лаборатории интеллектуального анализа (ЛИА) РАНХиГС. Проект представляет собой образовательную лабораторию, в которой пользователи могут в реальном времени тестировать и сравнивать три принципиально различных подхода к модерации текстовых сообщений:

1. **Регулярные выражения (Regex)** — rule-based детекция контактных данных по паттернам (URL, email, телефон)
2. **TF-IDF + CatBoost** — классический ML-подход на основе частотного анализа текста и градиентного бустинга
3. **BERT + CatBoost** — deep learning подход с использованием предобученной русскоязычной нейросети DeepPavlov/rubert-base-cased-conversational

Приложение решает конкретную бизнес-задачу: **выявление скрытых контактных данных в сообщениях пользователей на торговых площадках**. Пользователи онлайн-площадок по продаже товаров и услуг стремятся обойти комиссию сервиса, передавая контактные данные в замаскированном виде — заменяя буквы цифрами, вставляя пробелы, записывая номера словами и т.д. Система демонстрирует, как разные алгоритмы справляются с этими обходными методами.

Помимо интерактивного тестирования моделей, приложение включает **образовательный компонент**: подробные теоретические блоки с описанием каждого алгоритма (Regex, TF-IDF, BERT) и **интерактивный квиз** из 8 вопросов для проверки усвоения материала — что соответствует формату лабораторной работы для образовательного учреждения.

## Функционал

### Интерактивное тестирование моделей
- Ввод произвольного текстового сообщения пользователем через `st.text_area`
- Генерация случайных сообщений из реального датасета для тестирования (`st.checkbox`)
- **Одновременная классификация** сообщения тремя алгоритмами с визуальным отображением результатов:
  - Зелёный индикатор (`st.success`) — контактная информация обнаружена
  - Красный индикатор (`st.error`) — контактная информация не обнаружена
- Примеры сообщений с замаскированными контактными данными (блок-подсказка с `st.expander`)

### Regex-модуль детекции
- Детекция URL-адресов (HTTP/HTTPS, IPv4, IPv6, домены с портами) через комплексный regex-паттерн
- Детекция email-адресов по шаблону `[\w\.-]+@[\w\.-]+(\.[\w]+)+`
- Детекция телефонных номеров в различных форматах (с пробелами, дефисами, скобками)
- Классификация найденного типа: `url`, `mail`, `phone`

### TF-IDF + CatBoost классификатор
- Полный пайплайн предобработки текста:
  - Удаление спецсимволов (`/*,;-`)
  - Удаление пунктуации (`string.punctuation`)
  - Нормализация пробелов
  - Лемматизация через NLTK WordNetLemmatizer
- Трансформация текста через обученный TfidfVectorizer с загруженным словарём
- Классификация через CatBoost-модель, обученную на TF-IDF-фичах
- Обработка edge-case: слова вне вокабуляра TF-IDF

### BERT + CatBoost классификатор
- Токенизация входного текста через AutoTokenizer из Hugging Face (rubert-base-cased-conversational)
- Padding до 512 токенов с нулевыми значениями
- Генерация attention mask (1 для реальных токенов, 0 для padding)
- Извлечение CLS-эмбеддинга (первый токен `[0][:, 0, :]`) через `AutoModel` в режиме `torch.no_grad()`
- Классификация эмбеддинга через CatBoost-модель

### Образовательный контент
- **Теория Regex**: визуальная аналогия (лист картона с вырезами), области применения (парсинг, валидация, тестирование)
- **Теория TF-IDF**: пошаговое объяснение формул TF, IDF и TF-IDF с иллюстрациями формул (`images/tf_formula.jpg`, `images/idf_formula.jpg`, `images/tf_idf_formula.jpg`)
- **Теория BERT**: объяснение character-level vs word-level, концепция word embeddings (пример "король - мужчина + женщина = королева"), механизм обучения через маскирование 15% слов (`[MASK]`), предсказание связности предложений
- Каждый теоретический блок раскрывается по запросу через `st.expander`

### Интерактивный квиз
- 8 вопросов с вариантами ответов (`st.radio`) в `st.form`:
  - Определение регулярных выражений
  - Способы обхода модерации пользователями
  - Определение TF-IDF и что она оценивает
  - Определение BERT и принцип его обучения
  - Формат входных данных для BERT
  - Учёт контекста в BERT
- Подсчёт правильных ответов с порогом прохождения (7 из 8)
- Визуальная обратная связь: `st.warning` при неудаче, `st.success` при прохождении

### Разделение ответственности
- **`app.py`** (377 строк, 22.1 KB) — Streamlit-интерфейс: весь UI, образовательный контент, квиз, отображение результатов классификации. Не содержит бизнес-логики ML.
- **`tools.py`** — вся ML-логика: загрузка данных и моделей при старте (с `st.spinner`), три функции-предикторa (`get_re_pred`, `get_tf_idf_pred`, `get_bert_prediction`), предобработка текста, утилита генерации случайного сообщения.

### Стратегия загрузки моделей
При инициализации приложения (`with st.spinner('Загружаю данные...')`) выполняется:
1. Проверка и загрузка NLTK-ресурсов (`omw-1.4`, `wordnet`) — с автоматическим fallback при отсутствии
2. Загрузка 4 CSV-файлов с данными: основной датасет, эмбеддинги, предикты, словарь TF-IDF
3. Загрузка 2 CatBoost-моделей (`catboost.cbm`, `tf_idf_catboost.cbm`) с фиксированным `random_state=25`
4. Загрузка BERT-токенизатора и модели из Hugging Face Hub (`DeepPavlov/rubert-base-cased-conversational`)
5. Десериализация обученного TfidfVectorizer из pickle-файла при каждом вызове TF-IDF предиктора

### Пайплайн BERT-классификации
```
Текст → AutoTokenizer.encode (max_length=512, add_special_tokens=True, truncation=True)
      → Zero-padding до 512 токенов
      → Attention mask (1 для реальных, 0 для padding)
      → AutoModel forward pass (torch.no_grad)
      → CLS-embedding [batch, 0, :] → numpy
      → CatBoost.predict → {0: "Нет контактов", 1: "Есть контакты"}
```

### Пайплайн TF-IDF классификации
```
Текст → remove_symbols → remove_punctuation → white_space → lemmatization
      → TfidfVectorizer.transform (загруженный vocabulary)
      → Sparse matrix → CatBoost.predict → {0: "Нет контактов", 1: "Есть контакты"}
```

### CI/CD Pipeline
GitHub Actions workflow (`main.yaml`) реализует двухэтапный деплой:
1. **check_files** — проверка размера файлов через `ActionsDesk/lfs-warning@v2.0` (лимит 10 MB для совместимости с HF Spaces)
2. **sync-to-hub** — синхронизация репозитория с Hugging Face Spaces через `git push --force` с Git LFS fetch, используя секрет `HF_TOKEN`

Триггеры: push в main, pull request в main, ручной запуск (workflow_dispatch).

## Ключевые достижения и фишки

### ML и Data Science
- **Сравнительный анализ трёх подходов**: rule-based (Regex), классический ML (TF-IDF + CatBoost), deep learning (BERT + CatBoost) — на одной и той же задаче, что позволяет наглядно продемонстрировать сильные и слабые стороны каждого
- **Русскоязычная NLP-модель**: использование DeepPavlov/rubert-base-cased-conversational — специализированной BERT-модели для русского языка, обученной на разговорных данных, что оптимально для задачи модерации пользовательских сообщений
- **Каскад моделей**: Regex для rule-based детекции (URL, email, телефон), TF-IDF + CatBoost для частотного анализа, BERT + CatBoost для семантического понимания — три уровня глубины анализа
- **CatBoost как классификатор поверх BERT-эмбеддингов**: вместо дообучения (fine-tuning) всей BERT-модели используется подход feature extraction + CatBoost, что значительно снижает требования к вычислительным ресурсам при сохранении качества

### Предобработка текста
- **Полноценный NLP-пайплайн** для TF-IDF: удаление спецсимволов, пунктуации, нормализация пробелов, лемматизация через NLTK WordNetLemmatizer
- **Комплексный regex-паттерн** для URL: поддержка HTTP/HTTPS, IPv4, IPv6, доменных имён с портами, путей — покрытие широкого спектра форматов контактных данных
- **Учёт реальных паттернов обхода модерации**: замена символов цифрами (`n1gga`, `b0ll0cks`), вставка пробелов (`i d i o t`), удаление пробелов (`dieyoustupid`), запись чисел словами

### Образовательная ценность
- **Формат лабораторной работы**: теория + практика + проверка знаний — полный образовательный цикл в одном приложении
- **Интерактивность**: пользователь видит результаты классификации в реальном времени, может экспериментировать с разными формулировками
- **Визуальные объяснения**: формулы TF, IDF, TF-IDF с иллюстрациями; аналогия листа картона для Regex; пример "король - мужчина + женщина = королева" для word embeddings
- **Квиз с порогом прохождения**: 8 вопросов, минимум 7 правильных для прохождения блока

### Инфраструктурные решения
- **Git LFS** для хранения ML-артефактов: модели CatBoost и данные хранятся через Git Large File Storage, что решает проблему размера файлов в Git-репозитории
- **Автоматический CI/CD**: push в main автоматически синхронизирует репозиторий с Hugging Face Spaces через GitHub Actions с проверкой размера файлов (лимит 10 MB)
- **Hugging Face Spaces как платформа деплоя**: zero-infrastructure подход — приложение доступно по URL без необходимости настройки серверов
- **Ленивая загрузка NLTK-данных**: проверка наличия корпусов с автоматической загрузкой при отсутствии, обеспечивая работоспособность при первом запуске на чистом окружении
