# Uplift Lab -- Интерактивная лаборатория uplift-моделирования

**Репозиторий:** https://github.com/versus666jzx/uplift_lab
**Заказчик:** РАНХиГС -- Лаборатория интеллектуального анализа (https://lia.ranepa.ru/)
**Деплой:** Hugging Face Spaces (Streamlit SDK)

## Обзор проекта

Uplift Lab -- интерактивное веб-приложение для демонстрации и сравнения подходов к uplift-моделированию в задачах маркетинговой аналитики. Проект разработан для Лаборатории интеллектуального анализа (ЛИА) при РАНХиГС и представляет собой образовательно-исследовательский инструмент, позволяющий:

1. **Исследовать данные** рекламной кампании (email-маркетинг) с интерактивной визуализацией.
2. **Самостоятельно сегментировать аудиторию** через систему фильтров и сравнить результат ручной аналитики с ML-моделями.
3. **Сравнить три ML-алгоритма** (CatBoost, Random Forest, XGBoost) в четырех подходах к uplift-моделированию (SoloModel, TwoModels vanilla, TwoModels DDR-control, ClassTransformation).
4. **Визуализировать метрики качества** uplift-моделей (Qini AUC, uplift@k, weighted average uplift, Qini curve, Uplift curve).

### Бизнес-контекст

Uplift-моделирование решает ключевую задачу маркетинга: определить, на каких клиентов рекламное воздействие окажет максимальный положительный эффект (увеличение конверсии), а на каких -- нет. Это позволяет оптимизировать бюджет рекламных кампаний, избегая рассылок клиентам, которые и без того купят (sure things) или будут раздражены рекламой (sleeping dogs).

### Датасет

Используется классический бенчмарк-датасет **MineThatData E-Mail Analytics And Data Mining Challenge** (Kevin Hillstrom), загружаемый через `sklift.datasets.fetch_hillstrom`. Содержит 42 693 записи клиентов, из которых отбираются два сегмента: получившие email с рекламой женских товаров (treatment) и не получившие email (control). Целевая переменная -- факт визита на сайт (`visit`).

**Признаки клиентов:**
| Признак | Описание |
|---------|----------|
| `recency` | Месяцев с момента последней покупки |
| `history_segment` | Классификация клиентов по объему трат ($0-$100, $100-$200, ..., $1000+) |
| `history` | Фактическая сумма трат за прошлый год (в долларах) |
| `mens` | Флаг покупки мужских товаров |
| `womens` | Флаг покупки женских товаров |
| `zip_code` | Тип местоположения (Urban, Suburban, Rural) |
| `newbie` | Флаг нового клиента |
| `channel` | Канал покупки (Phone, Web, Multichannel) |

## Стек технологий

### Фреймворк веб-приложения
- **Streamlit 1.11.0** -- фреймворк для создания интерактивных data-приложений на Python. Используется для построения UI с фильтрами, графиками, метриками и формами.

### Machine Learning -- базовые модели
- **CatBoost 1.0.6** -- градиентный бустинг с нативной поддержкой категориальных признаков. Обучение на GPU (`task_type="GPU"`), 500 итераций. Лучший результат среди всех моделей (uplift@30% = 0.0776 для SoloModel).
- **XGBoost 1.6.1** -- градиентный бустинг с поддержкой категориальных фичей (`enable_categorical=True`), обучение на GPU (`tree_method='gpu_hist'`), 100 estimators, max_depth=3.
- **scikit-learn 1.1.1** -- Random Forest Classifier (100 estimators, max_depth=5), LabelEncoder для кодирования категориальных признаков, OrdinalEncoder в pipeline, train_test_split со стратификацией.

### Machine Learning -- uplift-подходы
- **scikit-uplift (sklift) 0.4.1** -- специализированная библиотека для uplift-моделирования. Используются:
  - **SoloModel** -- один классификатор, обученный на объединенной выборке treatment + control с treatment-флагом как дополнительным признаком.
  - **TwoModels (vanilla)** -- два независимых классификатора: один на treatment-группе, другой на control-группе; uplift = разность предсказаний.
  - **TwoModels (ddr_control)** -- два зависимых классификатора с методом DDR (Dependent Data Representation) на control-группе.
  - **ClassTransformation** -- преобразование задачи uplift в стандартную задачу классификации через трансформацию целевой переменной.
  - **Метрики:** `uplift_at_k`, `uplift_by_percentile`, `qini_auc_score`, `weighted_average_uplift`, `qini_curve`, `uplift_curve`.
  - **Визуализация:** `plot_qini_curve`, `plot_uplift_curve`, `plot_uplift_preds`, `plot_uplift_by_percentile`.

### Визуализация
- **Plotly 5.9.0** -- интерактивные графики EDA (гистограммы распределений по newbie, zip_code, channel, history_segment, recency, history). Все графики с кастомизированными осями, легендами, hover-шаблонами.
- **Matplotlib 3.5.2** -- графики Qini curve и Uplift curve (через sklift.viz), интеграция с Streamlit через `st.pyplot()`.

### Обработка данных
- **pandas 1.4.3** -- загрузка, фильтрация, трансформация данных. Работа с индексами для воспроизводимости (фиксированные train/test split через сохраненные индексы).
- **NumPy 1.23.1** -- числовые вычисления.
- **category-encoders 2.5.0** -- CatBoostEncoder для кодирования категориальных признаков.
- **SciPy 1.8.1** -- статистические вычисления.
- **statsmodels 0.13.2** -- статистические модели.

## Функционал

### 1. Исследовательский анализ данных (EDA)
- **Распределение по типу клиента (newbie):** интерактивная гистограмма с подсчетом соотношения новых/старых клиентов.
- **Распределение по почтовым индексам:** гистограмма с разбивкой по newbie-флагу, отсортированная по убыванию.
- **Распределение по каналам покупки:** Phone / Web / Multichannel с процентными долями.
- **Распределение по history_segment:** 7 ценовых сегментов ($0-$100 до $1000+) с процентными долями.
- **Распределение по recency:** месяцев с последней покупки, с разбивкой по newbie.
- **Распределение по сумме покупок (history):** с обнаружением паттерна -- все покупки >$500 совершают только новые клиенты.

Все графики EDA собраны в раскрывающийся блок (`st.expander`), не загромождая основной интерфейс.

### 2. Интерактивная система фильтрации клиентов
Пользователь может самостоятельно выбрать целевую аудиторию для рекламной кампании через систему фильтров:

- **Канал покупки:** Все / Phone / Web / Multichannel (radio buttons).
- **Тип клиента:** Все / Только новые / Только старые (radio buttons).
- **Тип товаров:** Любые / Мужские / Женские (radio buttons).
- **Ценовые сегменты:** 7 чекбоксов ($0-$100 ... $1000+), все включены по умолчанию.
- **Почтовый индекс:** Suburban / Urban / Rural (чекбоксы).
- **Recency:** слайдер для диапазона месяцев с последней покупки.
- **Процент аудитории (k):** слайдер 1-100% для метрики uplift@k.

Валидация: проверка на пустой результат фильтрации (по каждому шагу), проверка выбора хотя бы одного сегмента и zip_code.

### 3. Расчет метрик для ручной аналитики
После применения фильтров вычисляются метрики для «ручного» подхода (все отфильтрованные клиенты получают uplift = 1):
- **Uplift@k%** -- uplift для заданного процента аудитории.
- **Qini AUC score** -- площадь под Qini-кривой.
- **Weighted average uplift** -- взвешенный средний uplift.
- **Uplift by percentile** -- таблица uplift по процентилям.

Метрики отображаются в 3-колоночном layout через `st.metric()`.

### 4. Сравнение с ML-моделями
Для каждой из трех ML-моделей (CatBoost, Random Forest, XGBoost) отображаются:
- **Метрики с delta** -- абсолютная разница между ML-моделью и ручной аналитикой (со стрелками вверх/вниз через `st.metric(delta=...)`).
- **Таблица uplift by percentile.**
- **Qini curve** -- с опциональной идеальной кривой (perfect=True), с точкой пользовательской метрики (красный маркер).
- **Uplift curve** -- аналогично, с точкой пользователя на графике.

Каждая модель в отдельном раскрывающемся блоке с формой для обновления графиков.

### 5. Предсохраненные предикты моделей
Вместо обучения моделей в runtime (что было бы ресурсоемко и медленно для веб-приложения), используются предсохраненные предсказания в CSV-файлах:
- `catboost/sm_cbc.csv` -- SoloModel CatBoost
- `catboost/ct_cbc.csv` -- ClassTransformation CatBoost
- `catboost/tm_independend_cbc.csv` -- TwoModels (vanilla) CatBoost
- `catboost/tm_dependend_cbc.csv` -- TwoModels (DDR-control) CatBoost
- `random_forest/tm_rfc.csv` -- TwoModels (vanilla) Random Forest
- `xgboost/sm_xgb.csv` -- SoloModel XGBoost

## Архитектура

### Разделение ответственности
Проект разделен на два модуля:
- **`app.py`** (~400 строк) -- основной модуль Streamlit-приложения: UI, загрузка данных и предиктов, вычисление метрик, визуализация Qini/Uplift curves. Содержит всю логику отображения и оркестрации.
- **`tools.py`** (~330 строк) -- утилитарный модуль: загрузка и предобработка датасета, функции фильтрации данных (6 специализированных фильтров + агрегирующая функция), все функции генерации Plotly-графиков для EDA, обертка для метрики weighted_average_uplift.

### Воспроизводимость ML-экспериментов
- **Фиксированные индексы train/test split** сохранены в `data/` (6 CSV-файлов: data, target, treatment для train и test). Это гарантирует идентичные выборки при каждом запуске, независимо от случайности в `train_test_split`.
- **Предсохраненные предсказания моделей** в `src/model_predictions/` обеспечивают детерминированные результаты без необходимости повторного обучения.
- **Random state** зафиксирован во всех моделях (42 для CatBoost, 25 для Random Forest, 42 для sklearn TwoModels).

### Pipeline обучения моделей (Jupyter Notebook)
Ноутбук `src/test.ipynb` содержит полный pipeline обучения всех моделей:

1. **Random Forest (sklearn):**
   - LabelEncoder для категориальных признаков (history_segment, zip_code, channel).
   - 4 подхода: SoloModel, ClassTransformation, TwoModels (vanilla), TwoModels (ddr_control).
   - Параметры: 100 estimators, max_depth=5, random_state=25.

2. **XGBoost:**
   - Категориальные признаки через `astype('category')` + `enable_categorical=True`.
   - GPU-ускорение (`tree_method='gpu_hist'`).
   - 4 подхода: SoloModel, ClassTransformation, TwoModels (vanilla), TwoModels (ddr_control).
   - Параметры: 100 estimators, max_depth=3.

3. **CatBoost:**
   - Нативная поддержка категориальных признаков (`cat_features` передаются при обучении).
   - GPU-ускорение (`task_type="GPU"`).
   - 500 итераций, silent=True.
   - 4 подхода: SoloModel, ClassTransformation, TwoModels (vanilla), TwoModels (ddr_control).
   - Сохранение моделей через `.save_model()` в формате .cbm.

**Итоговые результаты (CatBoost, uplift@30%):**

| Подход | uplift@30% |
|--------|-----------|
| SoloModel | 0.0776 |
| TwoModels (ddr_control) | 0.0734 |
| TwoModels (vanilla) | 0.0732 |
| ClassTransformation | 0.0669 |

### CI/CD Pipeline
GitHub Actions workflow выполняет два job:
1. **check_files** -- проверка размеров файлов через `ActionsDesk/lfs-warning@v2.0` (лимит 10 MB) для совместимости с Hugging Face Spaces.
2. **sync-to-hub** -- автоматический push в Hugging Face Space (`spaces/versus666/uplift_lab`) через HF_TOKEN. Зависит от check_files (sequential execution).

Триггеры: push в main, PR в main, ручной запуск (workflow_dispatch).

## Ключевые достижения и фишки

### Data Science
- **Комплексное сравнение 4 подходов к uplift-моделированию** на 3 различных ML-алгоритмах (итого 12 комбинаций), с систематическим бенчмаркингом по метрике uplift@30%.
- **Использование специализированной библиотеки scikit-uplift** -- одной из ведущих open-source библиотек для uplift-моделирования, что обеспечивает корректную реализацию всех подходов (SoloModel, TwoModels, ClassTransformation) и метрик (Qini AUC, uplift@k).
- **Визуализация Qini и Uplift curves** с наложением пользовательской метрики (красная точка) на ML-кривую -- наглядная демонстрация преимущества ML-подхода перед ручной сегментацией.
- **GPU-ускорение обучения** для CatBoost и XGBoost.

### Продуктовый подход
- **Интерактивная демонстрация "Аналитик vs ML"** -- пользователь сначала вручную сегментирует аудиторию, затем видит, насколько лучше справляется ML-модель. Delta-метрики наглядно показывают разницу.
- **Образовательная ценность** -- приложение пошагово вводит пользователя в предметную область: от описания данных и EDA к фильтрации, метрикам и ML.
- **Интерактивный EDA** -- все графики на Plotly с hover-подсказками, кастомными осями и легендами; возможность обновления выборки.
- **Валидация ввода** -- проверка корректности фильтров с информативными сообщениями об ошибках.

### Инженерные решения
- **Предсохраненные предикты** вместо обучения в runtime -- мгновенная загрузка результатов без GPU-зависимости на сервере.
- **Фиксированные индексы train/test split** -- полная воспроизводимость экспериментов.
- **CI/CD с автодеплоем** на Hugging Face Spaces через GitHub Actions.
- **Git LFS** для управления большими файлами (модели, бинарные данные).
- **Streamlit кэширование** для оптимизации производительности при интерактивном использовании.

### Визуализация
- **6 типов Plotly-графиков** для EDA -- каждый с кастомной настройкой осей, легенд, hover-шаблонов, цветовой дифференциацией по newbie.
- **Qini curve и Uplift curve** с опциональными идеальными кривыми и точкой пользователя.
- **3-колоночный layout метрик** с delta-индикаторами.
