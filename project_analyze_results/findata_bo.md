# Findata BO — Платформа мониторинга корпоративных раскрытий и облигационного рынка РФ

**Локальный путь:** `/mnt/c/Users/Andrew/PycharmProjects/MY_PROJECTS/findata-bo/`

## Обзор проекта

Findata BO — это production-ready финтех-система, построенная как Telegram-бот для профессиональных инвесторов на российском облигационном рынке. Система агрегирует данные из 6+ официальных источников раскрытия корпоративной информации (e-disclosure.ru, НРД, АКРА, НРА, Эксперт РА, isin.ru), обрабатывает их в реальном времени и доставляет персонализированные уведомления через Telegram — с учётом ISIN-кодов облигаций, триггер-фраз и тарифных планов пользователя.

Проект решает критическую проблему: инвестору в облигации необходимо оперативно отслеживать десятки компаний-эмитентов на нескольких разрозненных площадках. Система автоматизирует этот процесс, превращая ручной мониторинг в автоматическую доставку релевантных уведомлений — от событий раскрытия до появления новых облигаций на бирже.

Реализована полная монетизация: тарифные планы (FREE / BASIC / PREMIUM / TRIAL), интеграция платежей через YooKassa с фискализацией по 54-ФЗ, система заявок и администрирования.

## Стек технологий

### Backend (Python 3.11)
- **aiogram 3** — асинхронный фреймворк для Telegram Bot API, FSM (Finite State Machine) для управления диалогами
- **asyncpg** — высокопроизводительный асинхронный PostgreSQL-драйвер (raw queries для runtime)
- **SQLAlchemy 2.0** — ORM для описания моделей (используется только Alembic-миграциями)
- **Alembic** — 20 миграций БД с комментариями на русском языке ко всем таблицам/колонкам
- **httpx** — асинхронный HTTP-клиент с поддержкой прокси
- **Pydantic** — валидация конфигурации и переменных окружения
- **loguru** — структурированное логирование с ротацией по расписанию (ZIP-компрессия, 7-дневный retention)

### Парсинг и скрапинг
- **BeautifulSoup4 + lxml** — парсинг HTML-страниц корпоративных раскрытий
- **Selenium + Chromium** — headless-браузер для источников с динамическим контентом (НРА)
- **fake-useragent** — ротация User-Agent для обхода ограничений

### Финтех-интеграции
- **t-tech-investments (T-Bank Invest SDK)** — gRPC-клиент для получения портфеля облигаций, счетов и эмитентов через T-Bank Invest API (custom PyPI-репозиторий: opensource.tbank.ru)
- **yookassa SDK** — приём платежей, создание чеков по 54-ФЗ, автоматическая проверка статусов
- **cryptography (Fernet)** — шифрование API-токенов T-Bank и контактных данных пользователей в БД

### Инфраструктура
- **Docker + Docker Compose** — трёхсервисная архитектура (bot, worker, postgres)
- **PostgreSQL 15** — основное хранилище (16 таблиц, JSONB, partial indexes)
- **uv** — менеджер зависимостей (lockfile, deterministic builds)

## Функционал

### Мониторинг корпоративных раскрытий
- Подписка на компании-эмитенты по ссылке на e-disclosure.ru
- Автоматическое отслеживание новостей с фильтрацией по ISIN-кодам и триггер-фразам
- Мониторинг 4 типов отчётности: устав, годовые отчёты, бухгалтерская отчётность, МСФО
- Предупреждения о критичных событиях (неисполнение обязательств)

### Broadcast-уведомления из рейтинговых агентств
- **НРД (nsd_data)** — события Национального расчётного депозитария по ISIN
- **АКРА (acra_ratings)** — рейтинговые действия по ISIN
- **НРА (ra_national)** — рейтинговые события (через Selenium)
- **Эксперт РА (expert_ra)** — рейтинговые действия
- Два режима: по совпадению ISIN/фраз и broadcast всем пользователям

### Мониторинг новых облигаций (T-Bank ISIN)
- Сбор новых ISIN с isin.ru (скользящее окно 30 дней)
- Отслеживание появления облигации на Т-Банке (SPA-проверка)
- Мгновенное уведомление PREMIUM-пользователей

### Интеграция T-Bank Invest API
- Привязка брокерского счёта через зашифрованный API-токен
- Автоматическая ночная синхронизация портфеля облигаций (delta sync)
- Определение эмитентов через gRPC API (bond -> asset -> borrow_name)
- Автоматическое создание/обновление/удаление подписок на основе портфеля

### Монетизация и платежи
- 4 тарифных плана: FREE (5 компаний), BASIC (99 руб/мес), PREMIUM (199 руб/мес), TRIAL (7 дней)
- Автоматический приём платежей через YooKassa с фискализацией по 54-ФЗ
- Управление подписками: активация, напоминания за 3 дня, автоматический даунгрейд

### Администрирование
- Админ-панель в Telegram с inline-клавиатурами
- Белый список пользователей (полный доступ без оплаты)
- Одобрение/отклонение заявок на подписку
- Уведомления админу о платежах

## Архитектура

### Двухпроцессная модель (Bot + Worker)

Система разделена на два независимых процесса, работающих параллельно:

**Bot Process (`bot.py`)** — обработка Telegram-взаимодействий:
- Dispatcher с MemoryStorage FSM для управления состояниями диалогов
- 6 роутеров: common, company, settings, subscription, admin, data
- Конечные автоматы: CompanyStates, TbankStates, DataStates, SubscriptionStates, PaymentStates, AdminStates
- Inline- и reply-клавиатуры для навигации

**Worker Process (`worker.py`)** — фоновая обработка данных:
- 6 параллельных воркеров через `asyncio.gather()`
- Каждый воркер с независимым интервалом и конфигурацией расписания
- Graceful shutdown через Event-based сигнализацию

### Модульная система воркеров

Все воркеры наследуют `BaseWorker` с общей логикой: цикл обработки, тихие часы (23:00-07:00 МСК), выходные, graceful shutdown:

| Воркер | Интервал | Назначение |
|--------|----------|------------|
| CompanyWorker | 30с | Мониторинг подписок на компании + отчётность каждые ~30 мин |
| BroadcastWorker | 30с | Глобальные уведомления по ISIN/триггер-фразам |
| TbankIsinWorker | Сбор: 1 день, Проверка: 1 час | Мониторинг новых облигаций на Т-Банке |
| SubscriptionWorker | 1 час | Истечение подписок и напоминания |
| TbankSyncWorker | 1 час (окно 01:00-05:00 МСК) | Ночная delta-синхронизация портфеля T-Bank |
| PaymentWorker | 30с, 24/7 | Проверка статусов платежей YooKassa |

### Абстракция источников данных (Source Registry)

Расширяемая архитектура для подключения новых источников:
- `NewsSource` — абстрактный базовый класс (Generic[RawEventT]) с 5 абстрактными методами
- `NewsEvent` — унифицированный dataclass для событий из любого источника
- `SourceRegistry` — синглтон-реестр с методами `get_company_sources()` / `get_broadcast_sources()`
- Каждый источник получает собственный `HttpClient` с конфигурацией прокси

### Двухуровневый Data Access Layer

Осознанное разделение:
- **SQLAlchemy ORM** — только для Alembic-миграций (декларативное описание схемы)
- **asyncpg raw queries** — для всех runtime-операций (максимальная производительность)
- Dataclass-модели (16 моделей) для типизированной работы с данными в Python-коде

### Схема БД (16 таблиц)

Продуманная реляционная схема с JSONB-полями, partial indexes, soft-delete механикой:
- `companies` + `user_subscriptions` — подписки с мягкой деактивацией (`is_active`)
- `company_events` + `global_events` — полная история с дедупликацией по `event_key`
- `subscription_plans` + `user_plans` + `yookassa_payments` — тарификация и платежи
- `user_tbank_tokens` + `user_tbank_accounts` — зашифрованные данные T-Bank
- `company_reports` — мониторинг 4 типов отчётности
- `tbank_isin_monitoring` — скользящее окно новых ISIN (30 дней)

### Безопасность

- Fernet-шифрование (AES-128-CBC) для API-токенов T-Bank и контактных данных (email/phone) в БД
- Валидация всех пользовательских вводов (URL, ISIN, company_id)
- ADMIN_CHAT_ID + whitelist — разграничение прав доступа
- Проверка `has_full_access()` перед любой привилегированной операцией

## Ключевые достижения и фишки

### Финтех-экспертиза
- **Глубокая интеграция с российской финансовой инфраструктурой**: e-disclosure.ru, НРД, АКРА, НРА, Эксперт РА, isin.ru, T-Bank Invest API — охват всех ключевых источников для инвестора в облигации
- **Работа с ISIN-кодами** — международные идентификаторы ценных бумаг как основа персонализации уведомлений
- **Фискализация по 54-ФЗ** — полноценный процесс приёма платежей с отправкой электронных чеков через YooKassa
- **gRPC-интеграция T-Bank Invest API** — портфель, счета, определение эмитентов через asset_uid/borrow_name

### Архитектурные решения
- **Delta sync** для ночной синхронизации портфеля — минимизация API-вызовов, равномерное распределение нагрузки по 4-часовому окну
- **Event-driven дедупликация** — система `event_key` предотвращает дублирование уведомлений при любых сценариях
- **Soft deactivation** подписок — при даунгрейде тарифа подписки не удаляются, а деактивируются с возможностью реактивации
- **Расширяемый Source Registry** — добавление нового источника требует только реализации ABC-класса и регистрации в реестре
- **Round-robin proxy rotation** — per-source конфигурация прокси для обхода rate limiting
- **Структурированное логирование** — раздельные файлы для каждого воркера и источника, ротация с ZIP-компрессией

### Продуктовая зрелость
- **20 Alembic-миграций** — эволюция схемы БД от простого бота до полноценной финтех-платформы
- **Продуманная тарификация** с trial-периодом, автоматическими напоминаниями и graceful даунгрейдом
- **Полная документация** — docs/ с описанием каждого источника, воркера, платёжной системы и подписочной модели
- **Production-ready инфраструктура** — Docker Compose, автоматические миграции при старте, configurable через env-переменные
