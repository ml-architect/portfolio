# GCM Security Event Manager -- платформа автоматизации реагирования на инциденты ИБ

**Локальный путь:** `/mnt/c/Users/Andrew/PycharmProjects/WORKING_PROJECTS/GCM/security_event_manager/`

## Обзор проекта

Security Event Manager (SEM) -- полнофункциональная платформа для автоматического обнаружения, классификации и эскалации инцидентов информационной безопасности. Система анализирует события от SIEM (Security Information and Event Management), применяет ML-модели для детекции компьютерных атак (КА) и компьютерных инцидентов (КИ), сопоставляет события с Sigma-правилами и автоматически создает алерты в TheHive -- платформе управления инцидентами.

Продукт разработан для SOC-команд (Security Operations Center) и предоставляет единый интерфейс для управления правилами детекции, вайтлистами, мониторингом эффективности правил и AI-ассистентом для аналитиков.

## Функционал

### Автоматическое создание инцидентов
- Циклический анализ событий SIEM (30-40 минутные циклы обработки)
- Группировка событий по внешним IP-адресам для формирования инцидентов
- Обнаружение DNS-атак, троянской активности, майнерского ВПО
- ML-детекция компьютерных атак (CatBoost) с автоматической классификацией типа
- Автоматическое присвоение severity на основе сработавших правил (max из всех)

### Sigma Rules Engine (кастомный движок)
- Полноценный движок матчинга Sigma-правил, написанный с нуля
- Поддержка модификаторов: contains, startswith, endswith, re, cidr, all
- Recursive descent парсер условий: AND, OR, NOT, скобки, wildcard (selection*)
- Кванторы: "1 of selection*", "all of them"
- Фильтрация по sensor_id с поддержкой групп СОА (group:N маркеры)
- CRUD для правил через UI: создание, редактирование, включение/выключение
- Импорт/экспорт правил в формате multi-document YAML
- Валидация YAML с предпросмотром метаданных

### Аналитика и мониторинг правил
- Агрегированная статистика срабатываний по каждому правилу
- Фильтрация по периодам: день, неделя, месяц, произвольный диапазон
- Автоматическое выявление проблемных правил трех категорий:
  - **Dead rules** -- включенные правила с 0 срабатываний за период
  - **Noisy rules** -- правила с аномально частыми срабатываниями (> avg * 3)
  - **Declining rules** -- правила с падением эффективности > 50% за неделю
- Дашборд с параллельным сбором данных из 5 источников (asyncio.gather)

### Система правил тегирования
- Одиночные правила: одна сигнатура -> тег + severity
- Составные правила: до 4 уровней вложенности с логикой AND/OR
- IP-правила: тегирование по IP-адресам атакующих / внутренних хостов с поддержкой подсетей
- Проверка дубликатов при создании правил
- Аудит изменений правил (sem_rule_changes.log)

### Вайтлисты (белые списки)
- Три типа: СОА (сенсоры), сигнатуры, IP-адреса/подсети
- Метаданные: автор, описание, дата добавления
- Таймеры истечения (expires_at) с фоновой проверкой каждые 30 секунд
- Уведомления об истекших элементах с привязкой к пользователю-создателю
- Валидация и нормализация IP-адресов и CIDR-подсетей

### Интеграция с TheHive
- Настройка подключения с тестированием связи
- Экспорт разрешенных кейсов с парсингом метаданных (assignee, impact, resolution, class-attack)
- Авто-экспорт по расписанию (ежемесячно, настраиваемый день)
- Bearer-token аутентификация с поддержкой организаций

### AI-ассистент (чат)
- Локальные LLM через Ollama (Qwen3 0.6b/1.7b/4b, DeepSeek-R1 1.5b)
- Загрузка кода из GitHub/GitLab репозиториев в контекст (Git2Markdown конвертер)
- Фильтрация бинарных файлов по расширению и содержимому
- Мультипользовательские чаты с разделением по владельцам
- Администратор может просматривать чаты всех пользователей
- Автоочистка старых чатов (>60 дней)

### Многопользовательская система
- RBAC: администратор + обычные пользователи с гранулярными правами
- Права доступа по разделам: dashboard, sigma, signatures, thehive_export, chat
- bcrypt-хеширование паролей, HMAC-SHA256 токены (7 дней TTL)
- Управление пользователями: создание, блокировка с кастомным сообщением
- Отслеживание последнего входа (last_login)
- Защита администратора от удаления и деактивации

### Система уведомлений
- Уведомления об истекших элементах вайтлистов
- Уведомления об удаленных правилах
- Привязка к пользователю-создателю + администратору
- Статусы прочитано/непрочитано
- Автоочистка уведомлений старше 30 дней

### Паттерны и решения

- **Repository Pattern** -- слой репозиториев (repo.py, sigma_repo.py, users_repo.py) абстрагирует доступ к данным
- **Factory Pattern** -- `create_app()` для FastAPI с конфигурируемыми middleware и роутерами
- **SPA Fallback** -- кастомный exception handler для клиентской маршрутизации Vue Router
- **Multi-stage Docker Build** -- Node.js (frontend) -> Python 3.11 (API) -> Python 3.10 (processor)
- **Config via API** -- backend processor получает конфигурацию через HTTP API web-контейнера, а не напрямую из файлов
- **Retry with Backoff** -- backend ожидает готовности web-контейнера (30 попыток по 10 секунд)
- **Async/Sync Bridge** -- async aiosqlite для API, синхронный psycopg2 для SIEM (через await в endpoint)
- **Background Tasks** -- asyncio.create_task для периодической проверки истечения вайтлистов
- **Parallel Data Fetching** -- asyncio.gather для параллельного сбора данных дашборда из 5 источников
- **Alembic Migrations** -- программный API (не CLI) для автоматического применения миграций при старте
- **Queue-based Processing** -- Python Queue для управления очередями создания/обновления инцидентов

### Безопасность

- HMAC-SHA256 подпись токенов с проверкой через `hmac.compare_digest` (timing-attack safe)
- bcrypt хеширование паролей с автоматическим salt
- CORS middleware с настраиваемыми origins
- Разделение эндпоинтов: backend endpoints без авторизации (Docker-сеть), UI endpoints с авторизацией
- Токены TheHive не отображаются в UI из соображений безопасности
- Защита от удаления/деактивации администратора на уровне бизнес-логики
- Предупреждения пользователям о конфиденциальности данных в AI-чате

## Ключевые достижения и фишки

### Кастомный Sigma Rules Engine
Разработан с нуля полноценный движок Sigma-правил с recursive descent парсером, поддержкой всех основных модификаторов (contains, startswith, endswith, re, cidr, all) и расширением для фильтрации по сенсорам. Движок включает встроенные тесты (11 тестовых сценариев) и обеспечивает матчинг в реальном времени при обработке SIEM-событий.

### ML-детекция атак
Интеграция CatBoost ML-моделей для двухуровневой системы обнаружения: первая модель определяет факт атаки (attack_model), вторая классифицирует тип атаки (class_attack_model). Модели загружаются при старте и применяются в каждом цикле обработки к формируемым инцидентам.

### Аналитика эффективности правил
Система автоматически выявляет проблемные правила детекции: мертвые (без срабатываний), шумящие (аномально частые) и деградирующие (падение >50%). Это позволяет SOC-команде поддерживать качество rule-базы и оперативно реагировать на изменения ландшафта угроз.

### Локальный AI-ассистент
Интеграция с Ollama обеспечивает работу AI-ассистента без отправки данных за периметр организации -- критически важно для security-операций. Конвертер Git2Markdown загружает код из репозиториев в контекст LLM для помощи аналитикам.

### Эволюция архитектуры
Проект прошел путь от Streamlit (legacy UI) до полноценного стека FastAPI + Vue 3 SPA с TypeScript, сохранив обратную совместимость. Все таблицы мигрированы на Alembic-управление, унаследованные данные (чаты, пользователи) автоматически мигрируются при обновлении.

### Многоуровневые правила тегирования
Система правил поддерживает до 4 уровней вложенности с логическими операторами AND/OR, позволяя описывать сложные цепочки событий ИБ для точного тегирования и создания алертов в TheHive.

- **ML Engineering** -- интеграция CatBoost моделей для детекции и классификации атак, разработка pipeline обработки SIEM-событий

- **DevOps** -- multi-stage Docker build, Docker Compose оркестрация с Ollama, GitLab CI/CD, развертывание в production
- **Рефакторинг** -- миграция с Streamlit на FastAPI + Vue 3, переход на Alembic миграции, унификация именования модулей, оптимизация (параллельный сбор данных дашборда)
