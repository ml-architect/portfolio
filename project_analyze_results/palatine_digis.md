## Обзор проекта

**Mediaclass API** -- бэкенд-сервис для коммерческого десктопного приложения "МЕДИАКЛАСС ИИ" компании Palatine Vision / DIGIS. Продукт предназначен для автоматического распознавания речи (Speech-to-Text) и диаризации спикеров (Speaker Diarization) в аудио- и видеофайлах. API работает как локальный сервер, интегрированный с Electron-фронтендом, и поставляется конечным клиентам в зашифрованном виде (PyArmor) с системой лицензирования, привязанной к оборудованию.

Продукт ориентирован на корпоративных клиентов: образовательные учреждения, медиакомпании и организации, которым необходимо транскрибировать записи совещаний, лекций и других мероприятий с идентификацией говорящих.

## Стек технологий

### Язык и фреймворк
- **Python 3.10/3.11** -- основной язык разработки
- **FastAPI** -- асинхронный REST API фреймворк
- **Uvicorn** -- ASGI-сервер с поддержкой кастомного event loop
- **Pydantic v2** -- валидация данных и сериализация (response models)
- **ORJSONResponse** -- высокопроизводительная JSON-сериализация ответов

### ML / AI модели
- **OpenAI Whisper** (whisper-timestamped) -- модель распознавания речи с точными временными метками на уровне слов/сегментов
- **Pyannote.audio 4.0** -- ML-пайплайн диаризации спикеров (сегментация + эмбеддинги + VBx-кластеризация + PLDA)
- **PyTorch 2.8 + torchaudio** -- фреймворк глубокого обучения, GPU-ускорение (CUDA)
- **PyTorch Lightning** -- обертка для обучения моделей
- **Scikit-learn, SciPy, NumPy, Pandas** -- научные вычисления и обработка данных

### Обработка медиа
- **FFmpeg** (через ffmpeg-python) -- обрезка, склейка аудио/видео, конвертация форматов
- **Pydub** -- извлечение аудиодорожки из видеофайлов (MP4 -> MP3)
- **MoviePy** -- получение метаданных медиафайлов (длительность, формат)
- **SoundFile, torchaudio** -- загрузка и обработка аудиосигналов

### Инфраструктура и DevOps
- **Docker / Docker Compose** -- контейнеризация сервиса
- **Conda (Anaconda)** -- управление ML-окружением с зависимостями CUDA
- **PyArmor** -- шифрование и обфускация исходного кода для коммерческой поставки
- **Pytest** -- e2e и unit-тесты с mock-провайдерами тяжелых ML-зависимостей

### Системная интеграция
- **WMI COM (pywin32) / PowerShell / wmic** -- сбор hardware-информации для привязки лицензий (Windows)
- **machine-id, sysfs** -- идентификация оборудования на Linux
- **IOPlatformUUID** -- идентификация оборудования на macOS

### Логирование и мониторинг
- **Loguru** -- структурированное логирование с ротацией по размеру и времени, сжатие архивов, перехват стандартного logging (uvicorn, FastAPI)

## Функционал

### Распознавание речи (Speech-to-Text)
- Транскрибация аудио- и видеофайлов (MP3, WAV, OGG, FLAC, MP4, MOV, AVI, MKV, WebM)
- Поддержка двух режимов: **локальные модели** (Whisper на GPU/CPU пользователя) и **облачный сервис** (Palatine Cloud API)
- Сегментация текста с точными временными метками (HH:MM:SS.mmm)
- Автоматическое извлечение аудиодорожки из видео для последующей обработки

### Диаризация спикеров (Speaker Diarization)
- Автоматическое определение количества говорящих и разметка по сегментам
- Локальная диаризация через Pyannote.audio 4.0 (сегментация + эмбеддинги + VBx-кластеризация)
- Облачная диаризация через Palatine Cloud API
- Результат: JSON с сегментами, привязанными к конкретным спикерам (SPEAKER_00, SPEAKER_01, ...)

### Редактирование медиа
- Обрезка медиафайлов: удаление произвольного временного сегмента с автоматической склейкой оставшихся частей (FFmpeg)
- Автоматический перезапуск распознавания после редактирования
- Поддержка как аудио, так и видеоформатов при обрезке

### Управление результатами
- Хранение результатов распознавания в структурированных директориях (UUID-based)
- Пагинация результатов с сортировкой по дате создания
- Автоматическое создание резервных копий медиафайлов и транскрипций перед редактированием
- Восстановление из бэкапа одним запросом
- Валидация целостности директорий с авто-восстановлением из backup при обнаружении повреждений
- Скачивание медиафайлов с поддержкой Range-запросов для потокового воспроизведения

### Система лицензирования
- Активация лицензионного ключа с привязкой к оборудованию ПК (CPU + RAM + GPU hash)
- Валидация лицензии при запуске через сервер лицензирования (mediaclass-license.palatine.ru)
- Кроссплатформенная генерация hardware hash (Windows/Linux/macOS)
- Обратная совместимость с legacy wmic-форматом для ранее активированных лицензий
- Верификационный ключ на основе криптографического сложения hardware hash и лицензии

### Асинхронная обработка задач
- Фоновое выполнение транскрибации и диаризации в отдельных потоках (threading)
- In-memory TaskStore с thread-safe операциями (Lock) для управления очередью задач
- Отслеживание прогресса задач в реальном времени (polling с интервалом 1-2 сек)
- Статусы: pending -> in_progress -> completed/failed, с этапами: transcription -> diarization -> done

### Облачный fallback
- Система управления cloud API токенами с автоматическим переключением при ошибке авторизации (3 токена)
- Retry-стратегия с экспоненциальным backoff для сетевых ошибок
- Автоматический сброс состояния токенов при каждом новом запросе

## Ключевые достижения и фишки

### Гибридная ML-архитектура (Local + Cloud)
Реализован паттерн **Provider** (Protocol-based interfaces) для абстракции ML-моделей: единый пайплайн транскрибации прозрачно работает как с локальными моделями (Whisper + Pyannote на GPU пользователя), так и с облачными сервисами Palatine Cloud. Переключение между провайдерами -- один параметр в запросе. Это позволяет клиентам без мощного GPU использовать облачную обработку, а клиентам с GPU -- работать автономно.

### Промышленный ML-пайплайн диаризации
Локальная диаризация спикеров реализована на уровне production-grade: Pyannote.audio 4.0 с предобученными моделями сегментации, speaker embeddings и VBx-кластеризацией с PLDA. Модели (pytorch_model.bin) поставляются вместе с приложением. Динамическая конфигурация YAML с автоматическим обновлением путей к моделям при запуске.

### Коммерческая защита кода
Полный цикл защиты интеллектуальной собственности: шифрование исходного кода через **PyArmor** с коммерческой лицензией, привязка лицензии продукта к hardware-хешу ПК, серверная валидация через выделенный сервер лицензирования. Кроссплатформенная идентификация оборудования (WMI COM + PowerShell + wmic legacy на Windows, machine-id на Linux, IOPlatformUUID на macOS).

### Отказоустойчивость и самовосстановление
- Автоматическая валидация и восстановление поврежденных директорий результатов из backup
- Cloud Token Manager с fallback-цепочкой из 3 токенов и автоматическим переключением
- HTTP retry с экспоненциальным backoff и обработкой 502/503/504
- Graceful degradation: автоопределение GPU/CPU, fallback на PowerShell при недоступности WMI, legacy wmic для обратной совместимости лицензий

### GPU-ускорение с автоопределением
Автоматическое определение доступности CUDA при старте, возможность принудительного выбора устройства (--device cpu/gpu/auto). Логирование информации о GPU (название, объём VRAM) при старте сервиса. Lazy-загрузка тяжелых ML-моделей (загружаются при первом запросе, а не при старте API).

### E2E тестирование с mock ML-провайдерами
Покрытие e2e-тестами полного цикла: загрузка файла -> транскрибация -> диаризация -> получение результатов -> скачивание -> удаление. Реализованы легковесные stub-классы для torch, numpy, moviepy, что позволяет запускать тесты без GPU и тяжелых ML-зависимостей.

### Продуманное логирование
Трехуровневая система логов на Loguru: основной лог с ротацией по размеру, ежедневный лог, отдельный файл ошибок. Перехват логов uvicorn/FastAPI через кастомный InterceptHandler. Структурированные сообщения с контекстом (task_id, имя файла, этап, время выполнения).
