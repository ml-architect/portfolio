## CommentsManagementAPI

**Репозиторий:** https://github.com/versus666jzx/CommentsManagementAPI
**Язык:** Python | **Создан:** август 2023 | **Размер:** 114 KB

### Обзор

Полноценный REST API для управления статьями, комментариями и авторами. Это наиболее зрелый и архитектурно проработанный проект в портфолио -- демонстрирует навыки проектирования бэкенд-систем промышленного уровня. API предоставляет CRUD-операции для статей и комментариев с полнотекстовым поиском, поддержкой пагинации, импортом из Excel, а также мультиязычным анализом текста (русский + английский).

### Стек

- **FastAPI** -- асинхронный Python-фреймворк для REST API
- **Uvicorn** -- ASGI-сервер
- **Elasticsearch 8.9** -- полнотекстовый поиск с мультиязычными анализаторами (russian, english)
- **PostgreSQL 16** -- реляционное хранилище данных с индексами
- **psycopg2** -- драйвер PostgreSQL для Python
- **Pydantic v2** -- валидация данных и сериализация моделей
- **pydantic-settings** -- управление конфигурацией через переменные окружения
- **pandas + openpyxl** -- импорт данных из Excel-файлов
- **Docker + Docker Compose** -- контейнеризация всего стека (3 сервиса: api, elastic, postgres)
- **HTTP Basic Auth** -- аутентификация для защищённых эндпоинтов
- **CORS Middleware** -- поддержка кросс-доменных запросов

### Архитектура и структура

    │   ├── comment.py             # 6 эндпоинтов (CRUD + поиск)

### API-эндпоинты (18 штук)

**Статьи (Articles) -- 10 эндпоинтов:**
- `POST /create_article` -- создание статьи с дедупликацией
- `POST /create_article_from_excel` -- массовый импорт статей и комментариев из Excel
- `POST /edit_article_content` -- редактирование содержимого статьи
- `POST /delete_article` -- каскадное удаление статьи и связанных комментариев из ES + PG
- `POST /update_article_content_by_row` -- обновление конкретной строки статьи
- `GET /search_article` -- полнотекстовый поиск по заголовкам и содержимому
- `GET /search_rows_in_articles` -- PostgreSQL full-text search по строкам
- `GET /get_all_articles` -- пагинированный список всех статей
- `GET /get_article_by_id` -- получение статьи по ID
- `GET /get_article_by_rows` -- получение статьи с фильтрацией по диапазону строк

**Комментарии (Comments) -- 6 эндпоинтов:**
- `POST /add_comment` -- добавление комментария (ES + PG)
- `POST /edit_comment` -- редактирование текста и HTML комментария
- `POST /delete_comment` -- удаление из обоих хранилищ
- `POST /update_comment_in_row` -- обновление комментария в контексте строки
- `GET /search_comments` -- поиск через Elasticsearch multi_match с сортировкой по дате
- `GET /get_comments_by_rows` -- получение комментариев с пагинацией по строкам

**Авторы (Authors) -- 2 эндпоинта:**
- `GET /authors/get_all_articles_authors` -- агрегация уникальных авторов через ES terms aggregation
- `GET /authors/get_articles_by_author` -- статьи конкретного автора с пагинацией

### Ключевые фишки

- **Двойное хранилище данных (Dual Storage):** синхронная запись в Elasticsearch (для полнотекстового поиска) и PostgreSQL (для реляционных запросов) -- грамотная гибридная архитектура
- **Мультиязычный поиск:** маппинги Elasticsearch настроены с анализаторами `russian` и `english` для корректной морфологической обработки текста
- **Автогенерация документации:** FastAPI автоматически генерирует Swagger UI (`/docs`) с описанием всех эндпоинтов, моделей и параметров
- **Полная контейнеризация:** Docker Compose разворачивает весь стек (API + Elasticsearch + PostgreSQL) одной командой
- **Импорт из Excel:** возможность массовой загрузки статей с комментариями из .xlsx файлов через pandas
- **Унифицированный формат ответа:** все эндпоинты возвращают `ApiResult` с полями `status`, `message`, `result`
- **Модульная архитектура:** чёткое разделение на слои (routes, classes, tools, db_tools)
- **Управление конфигурацией:** pydantic-settings для type-safe чтения env-переменных (12-factor app)
- **Graceful shutdown:** корректное закрытие соединений ES и PG при остановке приложения

### Демонстрируемые навыки

- Проектирование REST API с нуля
- Работа с Elasticsearch (маппинги, поиск, агрегации)
- Работа с PostgreSQL (DDL, индексы, SQL-запросы)
- Контейнеризация Docker / Docker Compose
- Асинхронное программирование (FastAPI/asyncio)
- Аутентификация и безопасность API
- Обработка данных (pandas, Excel)

## anmi_web_app

**Репозиторий:** https://github.com/versus666jzx/anmi_web_app
**Язык:** Python | **Создан:** май 2020 | **Размер:** 7 KB

### Обзор

Веб-приложение для сбора и анализа статистики Instagram-аккаунтов. Развёрнуто на реальном хостинге (`anmi.space`). Приложение авторизуется в Instagram через сессии, парсит профили пользователей, собирает данные о публикациях за указанный период и вычисляет аналитические метрики -- количество лайков, комментариев, плотность публикаций по времени суток.

### Стек

- **Python 3**
- **requests** -- HTTP-клиент для работы с Instagram API/web
- **BeautifulSoup (lxml)** -- HTML-парсинг страниц Instagram
- **JSON** -- работа с данными Instagram GraphQL API
- **Прокси-сервер** -- обход блокировок Instagram

### Структура

```
anmi_web_app/
├── __init__.py
├── instagram_tools.py       # Основной модуль: авторизация, сбор статистики, утилиты
├── publication_destiny.py   # Анализ плотности публикаций по времени суток
└── settings.json            # Конфигурация сессии, CSRF-токены
```

### Функционал

1. **Авторизация в Instagram** -- полноценная сессия с CSRF-токенами, User-Agent, прокси
2. **Парсинг профиля** -- извлечение user_id, аватара, статуса верификации, количества подписчиков из HTML через BeautifulSoup + `window._sharedData`
3. **Сбор статистики за период** -- постраничная загрузка публикаций через Instagram GraphQL API (`query_id: 17888483320005918`) с фильтрацией по min/max timestamp
4. **Аналитические метрики:**
   - Общее количество медиа
   - Медиа за выбранный период
   - Суммарные лайки и комментарии за период
   - Плотность публикаций по времени суток (ночь/утро/день/вечер)
5. **Утилиты:** конвертация media_id в shortcode (Base64-like алгоритм), форматирование чисел с порядками (тыс/млн), склонение русских слов по числительным

### Ключевые фишки

- **Обход защиты Instagram:** использование прокси, пользовательских заголовков, CSRF-токенов, динамическое обновление `x-csrftoken` и `rollout_hash`
- **Пагинация через GraphQL:** итеративная загрузка всех постов через `end_cursor` до достижения нужного временного диапазона
- **Аналитика времени публикаций:** оригинальный алгоритм анализа плотности постов по 4 периодам суток
- **Продакшн-деплой:** приложение работало на реальном хостинге anmi.space
- **Конвертация media_id в shortcode:** самописная реализация Base64-подобного кодирования для Instagram

### Демонстрируемые навыки

- Web scraping и парсинг HTML
- Работа с неофициальными API (Instagram GraphQL)
- Управление HTTP-сессиями и аутентификацией
- Обход anti-bot защиты
- Аналитика и обработка данных

## instagram_stat

**Репозиторий:** https://github.com/versus666jzx/instagram_stat
**Язык:** Python | **Создан:** сентябрь 2019 | **Размер:** 1 KB

### Обзор

Скрипт для автоматического сбора ежемесячной статистики Instagram-аккаунтов. Ранний проект, предшественник `anmi_web_app`. Использует неофициальную библиотеку `InstagramAPI` для авторизации и получения данных о фиде аккаунта. Собирает количество подписчиков, постов, лайков и комментариев за текущий месяц.

### Стек

- **Python 3**
- **InstagramAPI** -- неофициальная Python-библиотека для Instagram Private API
- **requests** -- HTTP-запросы для получения user_id
- **JSON** -- сериализация данных фида

### Структура

```
instagram_stat/
├── main_script.py   # Основной скрипт: сбор статистики за месяц
└── tools.py         # Утилиты: вычисление timestamp, получение user_id
```

### Функционал

1. **Авторизация** через InstagramAPI (Private API, эмуляция мобильного клиента)
2. **Получение user_id** по username через endpoint `/?__a=1`
3. **Получение фида аккаунта** за текущий месяц с вычислением unix-timestamp начала месяца
4. **Подсчёт метрик:** количество подписчиков, постов, суммарные лайки и комментарии
5. **Сохранение данных** в JSON-файл для дальнейшего анализа

### Ключевые фишки

- **Автоматический расчёт временного окна:** оригинальный алгоритм вычисления unix-timestamp начала текущего месяца через разложение текущего времени на компоненты
- **Работа с Instagram Private API:** эмуляция мобильного клиента для доступа к закрытым эндпоинтам
- **Предшественник anmi_web_app:** виден путь эволюции от простого скрипта к полноценному веб-приложению

### Демонстрируемые навыки

- Работа с Private API социальных сетей
- Автоматизация сбора данных
- Работа с unix-timestamp и временными интервалами

## insta_bot

**Репозиторий:** https://github.com/versus666jzx/insta_bot
**Язык:** Python / Jupyter Notebook | **Создан:** сентябрь 2019 | **Обновлён:** август 2022 | **Размер:** ~2 MB

### Обзор

Бот для автоматизации взаимодействий в Instagram -- автоматический лайкинг публикаций целевого аккаунта с использованием пула аккаунтов. Реализует стратегию накрутки активности: несколько аккаунтов поочерёдно лайкают посты указанного профиля с рандомизированными задержками для имитации человеческого поведения.

### Стек

- **Python 3**
- **InstagramAPI** -- неофициальная Python-библиотека (Private API)
- **requests** -- HTTP-запросы для получения user_id и media_id
- **configparser** -- управление конфигурацией бота через INI-файлы
- **Jupyter Notebook** -- прототипирование и тестирование

### Структура

```
insta_bot/
├── bot_main.py          # Основная логика бота: авторизация, цикл лайков
├── instagram_tools.py   # Утилиты: конфигурация, получение ID
└── solution.ipynb       # Jupyter-ноутбук с экспериментами
```

### Функционал

1. **Мультиаккаунтная авторизация** -- поддержка пула аккаунтов для распределённого лайкинга
2. **Фильтрация по времени** -- лайки только новых постов (за последние 2 дня или после последнего запуска)
3. **Рандомизированные задержки** -- имитация человеческого поведения (3-10 секунд между действиями)
4. **Персистентная конфигурация** -- сохранение `min_timestamp` последнего обработанного поста в `config.ini` для возобновления работы между запусками
5. **Получение метаданных** -- извлечение user_id по username и media_id по URL

### Ключевые фишки

- **Anti-detection:** рандомизированные задержки между лайками для обхода rate-limiting Instagram
- **Stateful-бот:** сохранение состояния (timestamp последнего поста) между запусками через config.ini
- **Мультиаккаунтный пул:** последовательная авторизация и работа с несколькими аккаунтами
- **Фильтр свежести контента:** бот лайкает только новые посты (2 дня или с момента последнего запуска)
- **Graceful exit:** корректное завершение при отсутствии новых постов

### Демонстрируемые навыки

- Автоматизация социальных сетей
- Работа с Private API
- Проектирование stateful-ботов
- Anti-detection стратегии
- Управление конфигурацией
