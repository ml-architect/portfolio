## Обзор проекта

**Explainer** — это интеллектуальная RAG-система (Retrieval-Augmented Generation), разработанная по заказу МФТИ для объяснения учебного контента школьникам. Сервис представляет собой полноценную AI-платформу, которая принимает вопросы детей и родителей и генерирует понятные, адаптированные под возраст ученика ответы, опираясь исключительно на проиндексированные учебные материалы.

Ключевая идея проекта — создание «объяснительного AI», который не заменяет учителя, а помогает ученику разобраться в материале: система намеренно настроена не решать задачи за ученика, а подводить к пониманию, используя детский язык и строго следуя загруженным учебникам.

Проект реализован в двух итерациях: прототип (`Explainer/`) и финальная production-версия (`explainer-official/`), с добавлением web-панели мониторинга, CI/CD через GitLab, расширенной документации API и Python-клиента.

### Контекст

- **Заказчик**: МФТИ (Московский физико-технический институт)
- **Подрядчик**: Palatine
- **Название по ТЗ**: «Разъяснитель»
- **Целевая аудитория**: школьники 1–11 классов, родители, образовательные платформы
- **Критическое требование**: в production должен использоваться исключительно российский LLM-провайдер; архитектура позволяет замену без изменения внешнего API

## Функционал

### RAG-запросы (ядро системы)
- Приём вопросов от пользователей через HTTP API (POST `/webhook/ask`)
- Семантический поиск релевантных фрагментов учебных материалов с фильтрацией по предмету, классу и теме
- Двухэтапная стратегия поиска: сначала по предмету + классу, затем fallback только по предмету
- Генерация ответов с помощью LLM, адаптированных под возраст ученика
- Поддержка истории диалога (`chat_history`) с настраиваемым лимитом сообщений (до 50)
- Настраиваемые параметры генерации: `top_k`, `max_tokens`, `temperature`
- Трассировка запросов через `X-Trace-Id` для отладки
- Возврат источников с оценкой релевантности (`relevance_score`)
- Статистика использования токенов (OpenAI-совместимый формат: `prompt_tokens`, `completion_tokens`, `total_tokens`)

### Индексация учебных материалов
- Два режима: загрузка бинарных файлов (multipart/form-data) и текстового контента (JSON)
- Поддержка форматов: PDF, DOCX, TXT, CSV, JSON, ePUB
- Автоматический парсинг документов с разбиением на чанки
- Генерация эмбеддингов через мультиязычную модель (1024-dim)
- Хранение с метаданными: предмет, класс, тема, имя файла, номер страницы
- Инкрементальная индексация с отслеживанием по хешу файла
- Дедупликация документов
- Организация материалов по иерархии: предмет / класс / тема

### Служебные операции
- Health-check с проверкой состояния каждого сервиса (Ollama, PGVector)
- Детальная статистика индекса: количество векторов, документов, предметов, классов, разбивка по предметам и классам
- Полная и частичная переиндексация (по предмету/классу или полная очистка)

### Web-панель мониторинга (production)
- Dashboard с real-time мониторингом состояния Ollama и PGVector
- Визуализация статистики индекса: векторы, документы, предметы, классы
- Разбивка по предметам и классам с интерактивным раскрытием
- Автоматическое обновление каждые 30 секунд
- Индикация статуса системы: healthy / degraded / unhealthy
- Отображение загруженных моделей Ollama
- Адаптивный дизайн, тёмная тема
- Runtime-конфигурация через `window.__EXPLAINER_CONFIG__`

### Python SDK (клиентская библиотека)
- Полнофункциональный Python-клиент с типизированными моделями данных (dataclasses)
- Методы: `health()`, `stats()`, `ask()`, `ingest()`, `ingest_file()`, `reindex()`
- Поддержка контекстного менеджера (`with` statement)
- Иерархия исключений: `AuthenticationError`, `ValidationError`, `DuplicateDocumentError`, `ConnectionError`, `ServerError`
- Автоматическое определение MIME-типов файлов
- Session pooling через `requests.Session`
- Конфигурация через переменные окружения (`API_AUTH_TOKEN`)

## Ключевые достижения и фишки

### Научно-образовательная составляющая
- **Педагогический AI-промпт**: система настроена на объяснение материала понятным ребёнку языком, с отказом от решения задач за ученика — это принципиально отличает её от обычных чат-ботов
- **Адаптация под возраст**: ответы генерируются с учётом класса ученика, обеспечивая адекватный уровень сложности языка
- **Строгое следование контексту**: LLM отвечает исключительно на основе проиндексированных учебных материалов, а не своих собственных «знаний» — это критично для корректности образовательного контента
- **Корректная обработка отсутствия информации**: при отсутствии релевантных материалов система честно сообщает об этом, не «выдумывая» ответы

### Архитектурные решения
- **Pluggable LLM Provider**: архитектура спроектирована с возможностью замены LLM-провайдера (Ollama -> российский API) без изменения внешнего API — критическое требование для production-размещения на серверах заказчика
- **Двухэтапный fallback-поиск**: если точный поиск по предмету + классу не дал результатов, система автоматически расширяет поиск только по предмету
- **Queue-based обработка**: n8n работает в режиме очередей с отдельным worker-сервисом и Redis, обеспечивая надёжную обработку параллельных запросов
- **Мультиязычные эмбеддинги**: использование модели `intfloat/multilingual-e5-large-instruct` (f16, 1024-dim) обеспечивает качественную работу с русскоязычным контентом
- **Оптимизированное векторное хранилище**: IVFFlat индекс с cosine distance, GIN-индекс для метаданных, отдельные B-tree индексы для фильтрации по предмету и классу
- **Инкрементальная индексация**: отслеживание проиндексированных документов по хешу файла позволяет избежать повторной обработки
- **Логирование без персональных данных**: таблица `rag_request_logs` хранит только метаданные запросов (длину вопроса, latency, предмет/класс), но не сами вопросы и не персональные данные

### Инфраструктура
- **Полная контейнеризация**: 7+ Docker-контейнеров с healthcheck-ами, dependency ordering и автоматической инициализацией моделей
- **Автоматический pull моделей**: скрипт `init-models-in-ollama.sh` + файл `models-pull.txt` обеспечивают автоматическую загрузку нужных моделей при первом запуске
- **GitLab CI/CD pipeline**: автоматический деплой при пуше в main с предварительной очисткой dangling Docker images
- **Production-ready мониторинг**: Vue 3 web-панель с Nginx reverse proxy, runtime-конфигурацией через environment variables и gzip-сжатием
- **Оптимизация Ollama**: Flash Attention, KV Cache (q8_0), ограничение загруженных моделей, контекст 8192 токенов
- **Поддержка GPU**: опциональное подключение NVIDIA GPU через Docker NVIDIA Container Toolkit

### Качество кода и документация
- **Полная документация API**: детальная спецификация всех 5 endpoints с примерами запросов/ответов, форматом ошибок и стратегией поиска
- **Типизированный Python SDK**: dataclass-модели для всех запросов/ответов, иерархия исключений, docstrings на русском языке
- **Документация Python-клиента**: отдельные markdown-файлы: quickstart, client API, models, exceptions
- **Схема БД с комментариями**: SQL-миграция с триггерами автообновления `updated_at`

- Реализация 3 n8n workflow-ов: ingestion, RAG query, service operations

- Реализация двухэтапного fallback-поиска с семантической фильтрацией
