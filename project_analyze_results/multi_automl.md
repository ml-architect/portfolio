# MultiAutoML — Веб-платформа для автоматического машинного обучения с поддержкой множественных моделей

**Репозиторий:** https://github.com/versus666jzx/MultiAutoML

## Обзор проекта

MultiAutoML — это внутренний корпоративный инструмент (internal tool), предоставляющий веб-интерфейс для автоматизированного обучения моделей машинного обучения без написания кода. Платформа построена на базе фреймворка LightAutoML (разработка Сбера) и обёрнута в интерактивный веб-интерфейс Streamlit, позволяя аналитикам данных и бизнес-пользователям загружать табличные данные, выбирать тип задачи, настраивать параметры обучения и получать обученную модель с визуализацией метрик качества — всё через браузер.

Проект решает ключевую проблему внутри организации: снижение порога входа в машинное обучение для сотрудников, не являющихся ML-инженерами. Вместо написания Python-кода, настройки пайплайнов предобработки и подбора гиперпараметров пользователь просто загружает CSV-файл, указывает целевую переменную и нажимает кнопку — система автоматически выполняет feature engineering, выбор алгоритмов, обучение и оценку модели.

Архитектурная особенность — паттерн MultiAutoML: система спроектирована для одновременного обучения и сравнения нескольких независимых AutoML-моделей через единый интерфейс, что позволяет сравнивать различные конфигурации и выбирать лучшую модель.

**Период разработки:** ноябрь — декабрь 2022 года (25 коммитов, активная разработка в течение ~6 недель).

## Стек технологий

### Фреймворк машинного обучения
- **LightAutoML v0.3.7.3 [all]** — AutoML-фреймворк от Сбера (SberBank AI Lab), установка с полным набором зависимостей (`[all]`), включающим все доступные модели и пайплайны. Используется `TabularAutoML` пресет для работы с табличными данными и класс `Task` для конфигурации типа задачи. LightAutoML автоматически выполняет: feature engineering, отбор признаков, подбор гиперпараметров, построение ансамблей моделей (LightGBM, CatBoost, линейные модели).

### Веб-интерфейс
- **Streamlit v1.15.1** — фреймворк для создания интерактивных data-приложений на Python. Используется для: боковой панели с параметрами (sidebar), загрузки файлов (`file_uploader`), отображения таблиц и графиков, прогресс-бара обучения, интерактивных виджетов выбора параметров (`selectbox`, `multiselect`, `radio`, `number_input`), встроенного руководства пользователя, скачивания демонстрационных датасетов (`download_button`).

### Обработка и анализ данных
- **pandas v1.4.3** — загрузка CSV-данных, манипуляции с DataFrame, разделение на train/test по именам файлов, преобразование типов целевой переменной (бинарные классы 1/2 -> 0/1, int-приведение для мультикласса).
- **NumPy v1.23.5** — математические операции: округление вероятностей до классов (`np.round`), выбор класса через `np.argmax` для мультиклассификации (OHE -> метки), реализация кастомной метрики SMAPE.
- **scikit-learn v1.1.3** — используется для:
  - Разделения данных: `train_test_split` с фиксированным `random_state=25`
  - Метрик классификации: `accuracy_score`, `f1_score`, `roc_auc_score`, `precision_score`, `recall_score`, `average_precision_score`, `classification_report`, `confusion_matrix`
  - Метрик регрессии: `mean_absolute_error`, `mean_squared_error`, `r2_score`, `mean_absolute_percentage_error`
  - Кривых: `roc_curve`, `precision_recall_curve`

### Визуализация
- **Plotly v5.11.0** (`plotly.express`) — интерактивные графики: confusion matrix (heatmap с цветовой схемой `Darkmint_r`), ROC-AUC кривая (area chart с diagonal baseline), Precision-Recall кривая. Все графики рендерятся через `st.plotly_chart()`.

### Контейнеризация и деплой

- **Docker Compose v3.9** — оркестрация единственного сервиса `client_app`, контейнер `multiautoml`, проброс порта `80:80`.
- **.dockerignore** — оптимизация контекста сборки: в контейнер попадают только `src/`, `data/` и `requirements.txt`.

## Функционал

### Поддержка трёх типов задач ML
- **Бинарная классификация** — полная поддержка с автоматическим преобразованием меток (классы 1/2 конвертируются в 0/1 для корректной работы скореров). Доступные метрики: F1, Accuracy, ROC-AUC, Precision, Recall, PR-AUC. Визуализация: confusion matrix, ROC-AUC curve, Precision-Recall curve.
- **Мультиклассовая классификация** — поддержка с приведением целевой переменной к `int`, валидацией минимального количества записей на класс (не менее 5). Доступные метрики: F1, Accuracy, Precision, Recall (все с `average='weighted'` для мультикласса). Предсказания извлекаются из OHE-формата через `np.argmax`.
- **Регрессия** — поддержка с набором регрессионных метрик: MAE, MSE, R2, MAPE, SMAPE (кастомная реализация).

### Гибкая загрузка данных
- **Два режима загрузки:** единый файл (с автоматическим разделением на train/test по указанному проценту 5-40%) или раздельные файлы train/test (автоматическое определение по имени файла).
- **Валидация данных:** проверка количества загруженных файлов (ровно 2 для раздельного режима), проверка корректности имён файлов (должны содержать "train" и "test"), обработка ошибок чтения CSV.
- **Формат:** CSV с разделителем-запятой.

### Настройка параметров обучения
- **Выбор целевой переменной** — динамический `selectbox` на основе столбцов загруженного DataFrame.
- **Исключение столбцов** — `multiselect` для выбора колонок, которые нужно исключить из обучения, с валидацией (нельзя исключить target).
- **Выбор метрик** — динамический набор метрик в зависимости от типа задачи.
- **Таймаут обучения** — настраиваемый лимит времени от 10 секунд до 2 часов (по умолчанию 3600 сек / 1 час).

### Визуализация результатов
- **Таблица метрик** — сводная таблица оценок для каждой модели по выбранным метрикам (`evals_to_clear_table`).
- **Confusion Matrix** — интерактивная тепловая карта через Plotly (для бинарной классификации).
- **ROC-AUC Curve** — площадной график с диагональной линией случайной модели (для бинарной классификации).
- **Precision-Recall Curve** — площадной график (для бинарной классификации).
- **Семпл данных** — отображение 7 случайных записей из обучающей выборки для быстрого ознакомления.

### Встроенная документация и демо-данные
- **Руководство пользователя** — пошаговая инструкция, встроенная прямо в интерфейс приложения (`manual.py`).
- **Демонстрационные датасеты** — три скачиваемых примера для каждого типа задачи:
  - Бинарная классификация: Titanic dataset (891 запись, 12 столбцов, target: `Survived`)
  - Регрессия: Body Fat dataset (189 записей, 15 столбцов, target: `target`)
  - Мультиклассификация: синтетический датасет (11250 записей, 49 столбцов, target: `target`)

## Архитектура

### Паттерн "Multi-Model Orchestrator"
Ключевое архитектурное решение — класс `MultiAutoML`, реализующий паттерн оркестрации нескольких AutoML-моделей через единый интерфейс. Класс принимает список экземпляров `BaseModel`, последовательно обучает каждый из них с отображением прогресс-бара, собирает предсказания и метрики в словари с ключами-именами моделей. Это позволяет:
- Обучать несколько моделей с разными конфигурациями параллельно (в рамках одного запуска)
- Сравнивать результаты на единых данных
- Масштабировать систему добавлением новых AutoML-бэкендов

### Абстракция BaseModel
Класс `BaseModel` обеспечивает унифицированный интерфейс (`fit`, `predict`, `predict_proba`, `fit_predict`, `evaluate`) поверх LightAutoML, скрывая его особенности:
- LightAutoML не имеет отдельного метода `fit` — `BaseModel` оборачивает `fit_predict` и выбрасывает предсказания
- LightAutoML возвращает вероятности для бинарной классификации — `BaseModel` автоматически округляет до классов
- LightAutoML возвращает OHE-формат для мультикласса — `BaseModel` применяет `argmax`
- Поддержка ролей LightAutoML: `target` и `drop` передаются как словарь ролей при обучении

### Обработка данных через стратегию
Функция `preprocess_data` реализует ветвление обработки по типу входных данных:
- Если `data` — список (два файла): итерация по файлам, определение train/test по имени, раздельное чтение в DataFrame
- Если `data` — один файл: чтение, затем `train_test_split` с фиксированным `random_state=25` для воспроизводимости

### Маппинг метрик
Двухуровневая система маппинга: русскоязычные названия задач и метрик из UI-виджетов транслируются в технические идентификаторы через словари `task_type_mapper` (русское название -> строка для LightAutoML: `'binary'`, `'multiclass'`, `'reg'`) и `metric_mapper` (человекочитаемое название -> функция из sklearn.metrics).

### Кастомная метрика SMAPE
Реализована собственная функция Symmetric Mean Absolute Percentage Error, отсутствующая в scikit-learn: `100/n * sum(2 * |y_pred - y_true| / (|y_true| + |y_pred|))`.

### Контейнеризация
Полностью Docker-ready архитектура: приложение запускается в контейнере на порту 80, что обеспечивает одинаковое окружение для всех пользователей внутри организации. Docker Compose упрощает развёртывание до одной команды `docker-compose up -d`.

### Модульная структура проекта
```
src/
├── app.py                          # Точка входа — Streamlit-приложение, UI-логика
└── utils/
    ├── __init__.py                  # Пустой инициализатор пакета
    ├── multi_light_auto_ml.py       # Классы MultiAutoML и BaseModel, фабрика моделей
    ├── data_utils.py                # Загрузка данных, валидация, метрики, визуализация
    └── manual.py                    # Встроенное руководство пользователя с демо-данными
```

Разделение ответственности:
- `app.py` — исключительно UI-логика Streamlit (виджеты, layout, flow управления)
- `multi_light_auto_ml.py` — ML-логика (обучение, предсказание, оценка)
- `data_utils.py` — утилиты данных (загрузка, валидация, маппинг, визуализация)
- `manual.py` — документация и демо-данные

## Ключевые достижения и фишки

### Инженерные решения
- **Абстракция поверх LightAutoML** — `BaseModel` нивелирует особенности API LightAutoML (отсутствие чистого `fit`, формат вывода для разных типов задач), предоставляя стандартный scikit-learn-подобный интерфейс.
- **Расширяемая мультимодельная архитектура** — `MultiAutoML` позволяет добавлять новые AutoML-бэкенды без изменения UI-кода. В закомментированных участках кода видна заготовка для FastAPI-эндпоинтов и запуска через uvicorn, что указывает на планы по расширению до REST API.
- **Воспроизводимость экспериментов** — фиксированный `GLOBAL_RS = 25` для `train_test_split` гарантирует идентичные результаты при повторных запусках на одних и тех же данных.
- **Прогресс-бар обучения** — динамическое отображение хода обучения через Streamlit-плейсхолдеры с автоматической очисткой после завершения.

### Адаптация под корпоративные нужды
- **Русскоязычный интерфейс** — все элементы UI, сообщения об ошибках и руководство пользователя на русском языке, что важно для внутреннего корпоративного инструмента.
- **Встроенная документация с демо-данными** — три скачиваемых примера для каждого типа задачи позволяют пользователям быстро освоить инструмент без внешней документации.
- **Валидация на каждом шаге** — проверка корректности загруженных файлов, запрет исключения target-колонки, проверка минимального числа записей на класс (>=5 для мультикласса), обработка отсутствия target в тестовой выборке.
- **Однокомандное развёртывание** — `docker-compose up -d` запускает полностью готовое к работе приложение на порту 80.

### Визуализация и интерпретируемость
- **Интерактивные Plotly-графики** — confusion matrix, ROC-AUC и PR-кривые с возможностью масштабирования и наведения для детального анализа.
- **Сравнительная таблица метрик** — единообразное представление результатов всех моделей в DataFrame для быстрого визуального сравнения.
- **Адаптивный набор метрик** — динамическая подборка метрик в зависимости от типа задачи (6 метрик для бинарной классификации, 4 для мультикласса, 5 для регрессии).

### Production-подход
- **Docker-контейнеризация** — полная изоляция окружения, обновление pip/setuptools/wheel перед установкой зависимостей, оптимизированный `.dockerignore`.
- **Type Hints** — использование typing (`List`, `Dict`, `Optional`, `Tuple`, `Union`, `Any`) для аннотации типов во всех ключевых функциях.
- **Обработка ошибок** — try/except блоки при чтении CSV и вычислении метрик с информативными сообщениями через `st.error()`.
- **Пиннинг зависимостей** — все библиотеки зафиксированы до точной версии в `requirements.txt`.
